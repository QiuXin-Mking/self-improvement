# q
什么是分布式系统的可扩展性（Scalability）？
# a
可扩展性是分布式系统应对需求增长（如用户量、数据量、业务负载增加）的核心能力，指系统通过高效、低成本的方式（如添加资源、调整架构）提升性能或容量，且不显著影响现有服务质量（如响应时间、吞吐量）。它是解决“增长带来的瓶颈”问题的关键指标。

# q
垂直扩展和水平扩展有什么区别？
# a
- 垂直扩展（向上扩展）：通过增强单个节点的硬件性能（CPU、内存等）提升系统容量。
- 水平扩展（向外扩展）：在分布式集群中新增服务器节点来提升整体能力。

# q
可扩展性设计需要满足哪些关键原则？
# a
1. 成本可控
2. 运维复杂度低
3. 可用性不中断
