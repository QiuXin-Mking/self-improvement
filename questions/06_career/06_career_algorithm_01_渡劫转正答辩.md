# q
GDB工具如何在多线程情况下抓取到指定线程进行调试？
# a
使用 top 命令查找对应线程的线程名和线程号，然后通过 `gdb attach` 到该线程号。

# q
nd app 和 cluster app 的差别与联系是什么？
# a
nd app 是节点自带服务（类似 C++ 中的 private 类），cluster app 是集群自带服务（类似 public 类）。如果需要在本节点修改或查询其他节点的服务或状态，需要先通过 cluster app 再调用 nd app。

# q
服务放在 engine 内部和 engine 外部有什么区别？
# a
engine 是一个进程，内部每个服务是一个线程，能共享机制（如无锁队列）；外部服务则是独立线程，无法共享这些内部资源。

# q
如何提升代码修改后的信心，确保验证充分？
# a
先对修改做充分的 ut 测试并保证全部通过；再对整个流程行为做 st 测试，编写 st 用例并保证通过；或者手动构造异常情况，观察系统在异常下的行为是否符合预期。

# q
性能波动剧烈的原因分析及后续工作方向是什么？
# a
通过性能回环测试发现性能下降梯度最大的位置，怀疑原因是热点 block 被频繁替换出去或替换算法的 size 不够大，后续研究替换算法（如 2Q、LRU）并针对 block size 做修改，以降低性能震荡。
