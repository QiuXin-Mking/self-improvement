# q
LRU 缓存机制需要支持哪些核心操作？
# a
需要通过 key 随机访问 value（get），将访问过的元素移到头部（表示最近使用），以及在容量满时删除最久未使用的元素（即尾部元素）。

# q
为什么 LRU 缓存常使用哈希表与双向链表组合实现？
# a
哈希表提供 O(1) 的随机访问，双向链表支持 O(1) 的头部/尾部插入和删除，两者配合可实现 get 和 put 均为 O(1) 时间复杂度的 LRU 缓存。

# q
在 Python 中哪个内置数据结构可以便捷地实现 LRU 缓存？
# a
`collections.OrderedDict`，它内部使用哈希表与双向链表维护插入顺序，提供了 `move_to_end` 和 `popitem(last=False)` 等方法，适合实现 LRU 淘汰策略。

# q
C++ 实现 LRU 缓存时，双向链表节点应包含哪些基本字段？
# a
节点应包含 `key`、`value`、`prev` 指针和 `next` 指针，以便通过哈希表映射 key 到节点，并在链表中进行快速插入和删除。
