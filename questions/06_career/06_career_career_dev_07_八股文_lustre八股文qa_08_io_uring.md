# q
io_uring 是什么？
# a
io_uring 是 Linux 内核自 5.1 版本引入的高性能异步 I/O 框架，旨在提高传统系统调用（如 epoll、aio）的并发性能并降低调用开销。

# q
io_uring 的轮询模式（polling mode）是如何工作的？
# a
轮询模式让内核专门分配一个线程，在后台不断扫描用户空间的 I/O 提交队列。应用提交 I/O 时无需频繁进行系统调用，从而获得极致的并发与低延迟，适合高性能高并发场景。

# q
传统 I/O 与 io_uring 的核心区别是什么？
# a
传统 I/O 中，用户进程每提交一个 I/O 请求都需要陷入内核（系统调用），内核处理完后再唤醒用户线程。io_uring 通过共享内存中的提交/完成队列，以及可选的轮询模式，大幅减少系统调用次数，降低延迟并提升并发性能。
