# q
什么是文件描述符（file descriptor）？它是如何产生的？
# a
文件描述符是内核为每个打开的文件分配的一个整数标识。当内核打开一个文件时，会创建对应的文件描述符，之后进程便可通过该描述符对文件进行读写操作。

# q
如何将标准输出重定向到文件？
# a
使用 shell 的输出重定向符号 `>`，例如 `ls > file.list`，该命令会将 `ls` 的标准输出写入 `file.list` 文件中。

# q
不带缓冲的 I/O 函数（如 read、write）与带缓冲的 I/O 函数（如 fread、fwrite）的主要区别是什么？
# a
不带缓冲的 I/O 直接通过系统调用读写内核缓冲区，没有额外的用户空间缓冲区，保证了数据的即时性但可能频繁访问磁盘；带缓冲的 I/O 在用户空间中维护一个缓冲区，可以减少系统调用的次数，从而提高 I/O 效率，但数据可能会暂存于用户缓冲区中。

# q
什么是进程？
# a
进程是程序执行的实例，在 UNIX 系统中常被称为任务（task）。
