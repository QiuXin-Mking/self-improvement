# q
TCP 连接的理论最大数量是如何基于四元组计算的？
# a
理论最大连接数由源 IP、源端口、目标 IP、目标端口构成的四元组决定。IPv4 环境下，源 IP 和目标 IP 各有 2^32 种可能，源端口和目标端口各有 2^16 种可能。因此：
- 单端口理论最大连接数：2^32 * 2^16 = 2^48
- 所有端口总和理论最大值：2^48 * 2^16 = 2^64

# q
单台服务器实际可支持的 TCP 并发连接数受到哪些资源限制？如何优化到百万级？
# a
单台服务器通过以下优化可支持百万级并发：
- 文件描述符限制：调整 Linux 的 `ulimit` 和 `sysctl` 参数
- 内存分配：每个连接大约消耗 3-4 KB 内存
- 端口复用技术：使用 `SO_REUSEADDR` 等
实际连接数无绝对固定值，需综合这些资源与优化策略动态平衡。
