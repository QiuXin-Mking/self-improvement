# q
什么是数据密集型（data-intensive）应用？
# a
随着数据规模或查询速率增长，数据必须分布到多台机器上，你可能需要组合多个具备不同能力的存储与处理系统，这样的应用称为数据密集型应用。与之相对的是计算密集型（compute-intensive）系统。

# q
事务（transaction）是什么？
# a
事务指的是形成逻辑单元的一组读取和写入。OLTP（联机事务处理）系统以事务为基本单位，面向大量短小、读写频繁、并发高的事务，强调一致性、可用性和响应速度。

# q
OLTP 和 OLAP 的主要区别是什么？
# a
OLTP（联机事务处理）面向实时交易，大量短小事务，读写频繁，强调响应速度与一致性，数据结构偏向规范化。OLAP（联机分析处理）面向分析型查询，数据量大（TB~PB 级），读多写少，查询复杂（多表关联、聚合），可接受秒级到分钟级延迟，结构偏向冗余（宽表、星型/雪花模型），常用于数据仓库。

# q
分布式系统主要用于解决单机系统的哪些问题？其关键特点是什么？
# a
分布式系统主要解决单机系统的局限性，其关键特点包括：容错/高可用性（单机或网络故障时继续工作）、可伸缩性（负载分散到多台机器）、低延迟（地理分布就近服务）、弹性（根据需求动态伸缩）。云计算常用于在线服务和高可用业务系统，而 HPC 常用于计算密集型科学计算。

# q
软件性能的评判标准有哪些？为什么要关注响应时间的分布？
# a
主要评判标准包括响应时间（时延）和吞吐量。由于响应时间会随请求变化，应将其看作一个可测量的分布，而非单一数字。长尾（高百分位点响应时间，也叫尾部延迟）非常重要，因为它直接影响用户体验。
