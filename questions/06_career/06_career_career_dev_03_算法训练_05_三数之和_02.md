# q
在 `for` 循环体内直接使用 `while` 进行去重（如 `while(fir>0 && nums[fir]==nums[fir-1]){ fir++; }`）为什么是灾难性写法？
# a
因为当 `while` 条件不再满足并退出后，循环体内的后续代码仍会继续执行，而 `fir` 已经被错误地递增跳过了本应处理的元素，导致逻辑错误。正确做法是用 `if` 加 `continue`，仅跳过当前一次迭代。

# q
三数之和中，第一层循环去重的正确写法是什么？
# a
```cpp
for (fir = 0; fir < n; fir++) {
    if (fir > 0 && nums[fir] == nums[fir-1]) {
        continue;
    }
    // ...
}
```

# q
三数之和双指针解法中，如何在第二层循环正确处理 `sec` 与 `thr` 相遇的情况？
# a
当 `sec == thr` 时直接 `break` 跳出内层循环，因为指针相遇后无需继续查找，且必须避免重复使用同一个元素。

# q
双指针模板的要点包含哪些？
# a
1. 定义遍历变量类型与作用域。  
2. 明确定义遍历边界，如 `sec < thr`，防止指针越界或重叠。  
3. 在移动指针时始终检查边界条件，避免死循环或错误。
