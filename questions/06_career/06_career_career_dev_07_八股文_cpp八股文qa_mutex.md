# q
std::mutex 的核心语义是什么？
# a
核心语义是“独占所有权”：同一时间只能有一个线程成功锁定它，其他试图锁定的线程会被阻塞，直到当前持有锁的线程解锁。

# q
std::mutex 提供了哪些基础操作？各自作用是什么？
# a
- `void lock()`：阻塞当前线程，直到成功获取锁。
- `bool try_lock()`：尝试获取锁，成功返回 true，失败返回 false，不阻塞。
- `void unlock()`：释放锁，必须由当前持有锁的线程调用，否则行为未定义。

# q
调用 std::mutex 的 unlock 有什么关键注意事项？
# a
必须由当前持有锁的线程调用，否则行为未定义。
