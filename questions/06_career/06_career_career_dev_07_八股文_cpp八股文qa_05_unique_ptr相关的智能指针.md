# q
使用智能指针解决哪些传统指针的常见问题？
# a
1. 忘记释放内存，造成内存泄漏。
2. 尚有指针引用内存的情况就释放了它，产生引用非法内存的指针。

# q
shared_ptr 的核心特性有哪些？
# a
- **共享性**：多个 shared_ptr 可共享同一资源，通过引用计数管理生命周期。
- **引用计数**：内部维护原子计数器，拷贝时 +1，析构时 -1，计数为 0 时释放资源。
- **线程安全**：引用计数操作为原子操作，支持多线程。

# q
unique_ptr 的核心特性与使用限制是什么？
# a
- **独占性**：同一时间只能有一个 unique_ptr 指向资源，禁止拷贝，仅支持移动语义（std::move）。
- **自动释放**：超出作用域或被重置时，自动调用 delete（或自定义删除器）释放资源。
- **轻量高效**：无额外引用计数开销，性能接近裸指针。

# q
weak_ptr 的正确使用方式及常见误区是什么？
# a
**正确使用**：
- weak_ptr 是对 shared_ptr 管理对象的非拥有性引用，不会增加引用计数。
- 必须通过 `lock()` 转换为 shared_ptr 后才能访问对象。
- `expired()` 可检查对象是否已被销毁。

**常见误区**：
- 误以为 weak_ptr 可以直接访问对象 → 必须通过 `lock()`，否则导致未定义行为。
- 误以为 weak_ptr 会自动释放对象 → weak_ptr 不影响对象生命周期，释放完全由 shared_ptr 决定。
