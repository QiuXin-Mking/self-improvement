# q
多线程中，如何实现线程之间的交互？
# a
共享变量：多个线程通过读写同一变量实现通信，需配合锁保证原子性。  
条件变量：当某个线程需要等待特定条件满足时，主动释放锁并进入阻塞状态；其他线程修改条件后，通过通知机制唤醒等待线程。例如在生产者-消费者模型中，生产者生产数据后通过条件变量通知消费者，消费者等待数据就绪后被通知消费数据。

# q
条件变量在生产者-消费者模型中的作用是什么？
# a
避免其他线程因锁被占用而无法修改共享条件：消费者在等待数据时通过 `wait(lock)` 自动释放互斥锁，进入阻塞状态；生产者生产数据后调用 `notify_one` 或 `notify_all` 唤醒消费者，消费者被唤醒后重新竞争锁，成功获取后继续执行。

# q
线程调用条件变量的 `wait(lock)` 时，会自动执行哪些步骤？
# a
1. 自动释放互斥锁（mutex）；  
2. 线程进入阻塞状态，等待其他线程的通知；  
3. 当被 `notify_one` 或 `notify_all` 唤醒后，线程重新竞争锁；  
4. 成功获取锁后继续执行。整个过程可以概括为：“释放锁 → 阻塞 → 唤醒 → 重获锁”。

# q
条件变量唤醒等待线程时，有哪些通知方式？
# a
`notify_one`：唤醒一个等待的线程；  
`notify_all`：唤醒所有等待的线程。被唤醒的线程会重新竞争互斥锁，获得锁后才能继续执行。
