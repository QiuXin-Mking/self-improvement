# q
mmap实现零拷贝的原理是什么？
# a
通过 mmap 将文件页缓存映射到进程虚拟地址空间，用户态直接读写内核缓冲，省去“内核→用户”的 CPU 拷贝。读文件传输到网络的数据链路变为：磁盘 → DMA → 内核页缓存 → socket 缓冲 → DMA → 网卡，消除了用户缓冲区的参与，减少 CPU 占用与上下文切换。

# q
sendfile 如何减少拷贝和上下文切换？
# a
sendfile 替代 read 和 write，减少一次系统调用和 2 次上下文切换；它直接将内核缓冲区数据拷贝到 socket 缓冲区，不再经过用户态，最终只有 2 次上下文切换和 3 次数据拷贝（磁盘 → 内核 → socket → 网卡）。
