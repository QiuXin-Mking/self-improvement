# q
Lustre实现强一致性的核心锁机制是什么？
# a
Lustre通过字节范围锁（Byte Range Locking）实现强一致性，锁的粒度是文件数据块的偏移范围（如[0,1MB]），而非整个文件。

# q
Lustre字节范围锁支持哪些锁模式？
# a
支持两种锁模式：
- 写锁（EXCL）：独占锁，阻止其他客户端对同一区域的读写。
- 读锁（PR）：共享锁，允许多客户端并发读，但禁止写操作。

# q
Lustre如何处理多个客户端对同一区域申请冲突锁的情况？
# a
通过分布式锁管理器（DLM）触发锁撤销（Revoke）流程，强制持有旧锁的客户端将脏数据刷新到OST，确保后续操作基于最新数据。

# q
Lustre的元数据锁和数据锁分别由哪些组件管理？
# a
- 元数据锁：由MDS（元数据服务器）管理，控制目录树、文件属性等操作（如文件创建需获取父目录的EX锁）。
- 数据锁：由OST（对象存储服务器）管理，控制数据块访问，客户端读写数据前需获取对应OST对象的锁授权。
