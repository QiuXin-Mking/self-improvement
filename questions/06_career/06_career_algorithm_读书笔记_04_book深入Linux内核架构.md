# q
在纯技术层面上，内核的任务是什么？
# a
内核是硬件与软件之间的中间层，作用是将应用程序的请求传递给硬件，并充当底层驱动程序，对系统中的各种设备和组件进行寻址。

# q
微内核与宏内核的主要区别是什么？
# a
微内核只包含最基本的功能，其他功能（如文件系统、内存管理）委托给独立进程，通过明确定义的通信接口与中心内核通信。宏内核则将全部代码（包括所有子系统）打包到一个文件中。

# q
同一时刻系统真正在运行的进程数目最多是多少？
# a
最多不超过 CPU 数目。

# q
UNIX 操作系统中创建新进程的两种机制是什么？分别有何特点？
# a
fork 和 exec。fork 创建当前进程的副本，父进程和子进程只有 PID 不同；exec 将新程序加载到当前内存页中执行。

# q
fork 的写时复制原理是什么？
# a
将内存复制操作延迟到父进程或子进程向某内存页面写入数据之前，在只读访问的情况下父进程和子进程可以共用同一内存页。
