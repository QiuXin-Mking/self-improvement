# q
buffer和cache分别指什么？
# a
buffer（缓冲）是内存中的临时区域，用于在速度/处理节奏不同的两个系统之间暂存数据，强调“先攒后写”，通过合并写、批量提交降低频繁小I/O的开销。cache（缓存）是内存中保存最近或高频访问数据的副本，用于加速后续读取，强调“读多写少、命中优先”，通过命中时免磁盘访问来降低读延迟。

# q
buffer的典型使用场景和主要作用是什么？
# a
典型场景：内核页缓存/块设备缓冲与磁盘、网络之间的数据暂存。主要作用：合并/整批I/O，降低系统调用与寻道成本，例如将多次小写操作合并为一次大块写入。

# q
cache的典型使用场景和主要作用是什么？
# a
典型场景：页缓存（Page Cache）、目录项/索引节点缓存（dentry/inode）。主要作用：加速读取、减少磁盘访问、提升命中率，通过缓存最近/高频访问数据的副本实现。

# q
在Linux `free`命令输出中，buffers和cached分别偏向什么含义？
# a
`buffers`偏向块设备缓冲（如裸块设备的元数据），`cached`偏向文件内容缓存（如页缓存）。两者都属于“占用内存但可回收”的部分。
