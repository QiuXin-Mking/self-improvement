# q
C++20 协程是什么？它和普通函数的主要区别是什么？
# a
C++20 协程是一种可以在执行过程中暂停和恢复的函数，支持协作式多任务。与普通函数一旦开始执行就必须运行到返回不同，协程可以通过 `co_await`、`co_yield` 和 `co_return` 在任意位置暂停并将控制权交还给调用者，之后可从暂停点恢复执行。

# q
C++20 协程的三个核心关键字分别有什么作用？
# a
- `co_await`：用于等待一个异步操作完成，挂起协程并将控制权交还给调用端。
- `co_yield`：产出一个值并暂停协程，适合生成器场景。
- `co_return`：结束协程并返回最终结果。

# q
在 C++20 生成器示例中，`promise_type` 的作用是什么？`yield_value` 函数如何工作？
# a
`promise_type` 是协程的控制块，负责管理协程的状态和传值。其中的 `yield_value(T value)` 函数会在协程执行 `co_yield` 时被调用，将产出的值保存到 `current_value` 成员中，并返回 `std::suspend_always{}` 使协程挂起，让调用者可以读取该值。

# q
协程与线程在调度方式上有何根本区别？
# a
协程是协作式、非抢占的调度方式，协程必须主动暂停（通过 `co_await`/`co_yield` 等）才能让出控制权；而线程通常是抢占式调度，由操作系统决定何时切换。C++20 协程的底层机制与 `std::future`、线程不同，不依赖操作系统线程调度，适合轻量级异步任务和流式处理。
