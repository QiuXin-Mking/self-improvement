# q
设计 LRU 缓存机制时，为什么需要用到哈希表与双向链表组合的数据结构？
# a
因为 LRU 需要支持两种操作：
1. 随机访问 key 对应的值（O(1) 查找）→ 需要哈希表；
2. 插入头部、删除尾部或移动节点到头部（O(1) 修改顺序）→ 需要双向链表。
Python 的 `OrderedDict` 内置结合了哈希表与双向链表；C++ 需手动用 `unordered_map<Key, list::iterator>` + `list` 实现。

# q
Python 中实现 LRU 最便捷的方式是什么？写出核心代码结构。
# a
使用 `collections.OrderedDict`，它结合哈希表与双向链表并保持插入顺序。核心操作：
```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.cache = OrderedDict()
    
    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)   # 移动到末尾表示最近使用
        return self.cache[key]
    
    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.cap:
            self.cache.popitem(last=False)  # 弹出最先插入的（最久未使用）
```

# q
C++ 实现 LRU 时，为什么通常使用 `list`（双向链表）和 `unordered_map` 的组合？对应的类型声明是怎样的？
# a
因为 `list` 支持在任意位置 O(1) 插入/删除，且迭代器在除被删除元素外始终保持有效；`unordered_map` 提供 O(1) 平均查找。常用声明：
```cpp
struct Node {
    int key, val;
};
list<Node> order;                                    // 存储节点，头部为最近使用
unordered_map<int, list<Node>::iterator> cache;      // 映射 key → 链表迭代器
```

# q
LRU 缓存中，当容量超出时应该删除哪个节点？如何用双向链表快速实现？
# a
应删除最久未使用的节点，即链表尾部节点。双向链表可以在 O(1) 时间内找到尾部节点并删除：
1. 通过尾指针或 `list` 的 `back()` 获取尾部节点；
2. 从哈希表中删除该 key；
3. 调用 `pop_back()` 移除尾部节点。
