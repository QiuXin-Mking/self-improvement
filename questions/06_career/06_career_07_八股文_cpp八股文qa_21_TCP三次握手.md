# q
简述 TCP 三次握手的具体过程
# a
- 客户端发送 SYN 报文（seq=x），请求建立连接。
- 服务端收到后，回复 SYN+ACK 报文（ack=x+1, seq=y），表示同意建立连接。
- 客户端再回复 ACK 报文（ack=y+1），双方进入 ESTABLISHED 状态。

# q
为什么 TCP 需要三次握手而不是两次？
# a
两次握手无法可靠地建立连接：
1. 无法有效规避网络中残留的历史错误连接请求。
2. 无法可靠同步双方的初始序列号。
三次握手让双方都确认对方已正确收到自己的初始序列号，是建立可靠连接的最小代价。

# q
为什么 TCP 不使用四次握手来建立连接？
# a
四次握手在功能上可行，但无必要。三次握手已经能同时完成双向的序列号同步，并能防止旧连接干扰，达到了可靠建立连接的最小通信次数。增加握手次数只会增加延迟和开销，不会带来额外的可靠性保障。
