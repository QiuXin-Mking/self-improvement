# q
数据密集型（data-intensive）应用通常需要什么类型的基础设施？
# a
需要组合多个具备不同能力的存储与处理系统，并且随着数据规模或查询速率增长，数据必须分布到多台机器上。

# q
OLTP（联机事务处理）和 OLAP（联机分析处理）的主要区别是什么？
# a
- OLTP：面向实时事务，大量短小事务，读写频繁、并发高，强调一致性、可用性、响应速度，数据结构偏向规范化。
- OLAP：面向分析查询，TB~PB 级历史数据，读多写少，查询复杂（多表关联、聚合、分组等），延迟可接受秒级到分钟级，常采用宽表、星型/雪花模型。

# q
为什么需要分布式系统而不是单机系统？其关键特点有哪些？
# a
- 容错/高可用性：部分机器或网络故障时系统继续工作。
- 可伸缩性：负载增长时可将负载分散到多台机器。
- 延迟：全球多地部署服务器，使每个用户能从地理接近的服务器获得服务。
- 弹性：云部署可根据需求向上或向下伸缩。

# q
构建应用时常有哪些非功能性需求？
# a
性能（响应时间、吞吐量），服务可靠性，可伸缩性，可维护性。

# q
为什么响应时间的高百分位点（长尾延迟）很重要？
# a
因为它直接影响用户体验，响应时间应被视为一个可测量的分布，而非单一数字，高百分位点反映最慢请求的延迟。
