# q
Lustre的渐进式文件布局（PFL）比普通条带化优秀在哪里？
# a
普通条带化虽然简单且对大文件高并发写入优化较好，但对小文件或文件大小变化大的场景容易造成资源浪费或性能不足。  
PFL 则根据文件大小动态调整条带策略：
- 小文件部分使用极窄的条带（如只有1个OST），提高效率，节省资源，避免条带碎片；
- 文件逐渐增大后自动平滑切换到更宽的条带（如4、16、32个OST），充分利用集群高带宽。

PFL 的具体策略示例：
文件区间：0 - 128MB → 条带宽度 1，条带大小 1MB  
文件区间：128MB - 1GB → 条带宽度 4，条带大小 4MB  
文件区间：1GB - 8GB → 条带宽度 16，条带大小 8MB  
文件区间：8GB 以后 → 条带宽度 32，条带大小 8MB

这样在文件创建初期采用保守方案，文件变大时自动切换到高性能宽条带写入，同时兼顾小文件和大文件的最佳性能。

# q
PFL 在不同文件大小区间采用的条带宽度和条带大小具体是多少？
# a
PFL 的典型配置如下：
- 0 – 128MB：条带宽度 1（1个OST），条带大小 1MB
- 128MB – 1GB：条带宽度 4（4个OST），条带大小 4MB
- 1GB – 8GB：条带宽度 16（16个OST），条带大小 8MB
- 8GB 以后：条带宽度 32（32个OST），条带大小 8MB
