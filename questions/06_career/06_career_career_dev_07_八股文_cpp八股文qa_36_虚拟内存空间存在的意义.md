# q
虚拟内存如何扩展可用内存空间？
# a
虚拟内存让每个进程看到的是一个连续、完整的内存空间（通常远大于实际物理内存），即使物理内存不足，操作系统也能通过磁盘空间（如交换区/页面文件）扩展“可用”内存，避免程序因为物理内存不够而无法运行。

# q
虚拟内存如何实现进程隔离与安全性？
# a
每个进程拥有自己独立的虚拟地址空间，不同进程的内存互不可见，从而防止进程间直接访问和修改彼此的数据，提高了系统的稳定性和安全性。

# q
虚拟内存如何简化内存管理和编程？
# a
开发者无需关注物理内存的具体分布和碎片问题，只需按照虚拟地址访问内存，系统自动完成物理地址映射，提高编程效率，简化了程序员的工作。

# q
虚拟内存如何支持内存共享与重定位？
# a
操作系统可以将相同的物理内存区域（比如只读的代码段和共享库）映射到多个进程的虚拟地址空间，提升资源利用效率；同时程序无需固定装载在某块物理内存，便于动态装载和重定位。

# q
虚拟内存的按需加载与内存置换机制有何作用？
# a
虚拟内存使得程序和数据可以按需加载到物理内存，未用到的部分可以在需要时置换到磁盘，这样大程序也能在有限内存下运行。
