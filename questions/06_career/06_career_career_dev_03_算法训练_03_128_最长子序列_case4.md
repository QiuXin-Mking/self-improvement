# q
最长连续序列问题中“状态分析错了”的典型根因是什么？
# a
根因通常是对“序列起点”的判断条件分析错误。标准做法使用 `if (!set_a.count(num-1))` 保证只从没有前驱的数字开始统计长度。如果状态分析错了，可能会：
- 把非起点误判为起点，导致重复计数或遗漏更长的序列；
- 忘记在 `while (set_a.count(cur+1))` 循环中更新 `cur` 和 `lev`，导致计数偏差或死循环；
- 不理解 `unordered_set` 的查找 `O(1)` 特性，错误地认为需要排序或额外状态数组。

# q
如何从代码逻辑快速定位最长连续序列的“状态分析”错误？
# a
1. **验证起点判断**：在 `if (!set_a.count(num-1))` 前后打印 `num`，确认只对真正的起点进入分支。
2. **检查循环逻辑**：在 `while (set_a.count(cur+1))` 内部打印 `cur`、`lev`，确认每次循环均正确递增 `cur` 和 `lev`。
3. **观察 ans 更新**：在 `ans = max(ans, lev)` 处打印当前 `ans` 与 `lev`，检查是否遗漏了某些序列的最大长度。
4. **边界测试**：用包含重复数字、负数、空数组的用例测试，确认集合初始化和遍历均正确处理边界。
