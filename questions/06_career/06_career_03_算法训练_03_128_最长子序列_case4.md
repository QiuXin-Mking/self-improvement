# q
最长连续序列问题中，典型的状态分析错误是什么？
# a
典型错误是误以为需要额外标记已访问元素来避免重复计算。标准解法通过判断 `num - 1` 是否存在于集合中，来识别序列的起点：只有当 `num - 1` 不存在时，才以 `num` 为起点向后扩展。这确保每条连续序列只被计算一次，无需维护额外的访问状态。若忽略这一“前驱检查”，就可能错误地认为算法会重复计算或存在状态遗漏。

# q
如何验证该代码片段在解决最长连续序列问题时的正确性？
# a
验证要点是检查 `if (!set_a.count(num - 1))` 这一条件。该条件保证：
- 仅从连续序列的最小值开始扩展（因最小值没有前驱元素）；
- 序列内的其他元素在遍历时因 `num - 1` 存在而被跳过，避免重复扩展；
- 最终 `ans` 保存所有序列的最大长度，且时间复杂度为 O(n)。只要此逻辑正确实现，即使 `unordered_set` 的遍历顺序不确定，结果也是确定的，无需额外状态记录。
