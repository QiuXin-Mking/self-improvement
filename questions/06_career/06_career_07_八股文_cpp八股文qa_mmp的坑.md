# q
mmap有什么坑，用什么方法可以规避？
# a
mmap 提供高效 I/O，但写入仅存于内存页缓存，掉电即丢失。msync 可将脏页刷至磁盘，确保数据持久，适用于关键数据。合理使用 msync 和 fsync 可在速度与可靠性间平衡。

# q
awk 'BEGIN{s=0}{s+=$9}END{print s}' 如何理解？
# a
BEGIN：只在开始前执行一次，适合做初始化。
{...}：对每一行做的操作（主程序块）。
END：在处理完所有行后执行一次，可用来输出结果。

BEGIN{s=0}：一开始设置变量s为0，表示总和清零。
{s+=$9}：对每一行，把第9列数据加到s里去。$9就是当前行的第9列内容（以空格或Tab分隔），类似于Excel的某一列。
END{print s}：所有行加完后，把s的值输出；即所有行的第9列相加的和。
