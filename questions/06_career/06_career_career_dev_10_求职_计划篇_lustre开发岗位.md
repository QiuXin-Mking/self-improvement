# q
Lustre 体系结构中 MGS、MDS/MDT 和 OSS/OST 分别承担什么角色？
# a
- MGS：管理 Lustre 文件系统的全局配置（如挂载信息、日志位置）。
- MDS/MDT：提供元数据服务，管理命名空间与目录结构，负责 FID 分配、layout、xattr、ACL 和配额。
- OSS/OST：提供对象存储，存放文件数据，通过条带化（striping factor/pattern）分布数据，支持 OSS affinity、对象预取与回写。

# q
LDLM 分布式锁管理器的常见锁模式有哪些？Intent 锁的工作流程是什么？
# a
常见锁模式包括：读取锁、写入锁、范围锁等，并涉及锁依赖与死锁检测。
Intent 流程：客户端先向 MDT 发出带意图的元数据操作请求，MDT 根据操作类型（如 open、create）在返回属性信息的同时授予相应锁或触发 glimpse 回调，从而减少额外的 RPC 交互并维护缓存一致性。

# q
MDT 或 OST 发生故障切换时，客户端 replay 的流程和主要限制是什么？
# a
流程：
1. 检测到节点故障后，服务切换到备用节点。
2. 备用 MDT/OST 回放日志（journal/lfsck-recovery）以恢复元数据或对象状态。
3. 客户端重连后，通过 replay 机制重放未完成的事务（如打开的文件句柄、未提交的修改）。
限制：
- 客户端 replay 仅能重放已经记录在服务端日志中的确定性操作，无法恢复应用层缓冲中丢失的数据。
- 需处理孤儿对象清理，确保文件系统一致性；过长时间未恢复的客户端可能被强制断开。

# q
为 Lustre 选择条带大小（stripe size）和条带计数（stripe count）时应考虑哪些因素？
# a
- 条带大小应匹配后端存储特性（SSD/NVMe 或 HDD）以及网络 MTU，避免跨分片引起不必要的网络 IO 放大。
- 条带计数决定了并发度：较多条带可提升吞吐但会增加锁争用和元数据开销；需根据并发写入线程数、OSS 节点数及网络带宽权衡。
- 同时需考虑文件访问模式（顺序大 I/O 宜用较大条带，随机小 I/O 需适当减小条带或减少条带数）。
