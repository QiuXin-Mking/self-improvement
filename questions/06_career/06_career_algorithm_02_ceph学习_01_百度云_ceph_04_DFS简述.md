# q
单体存储的主要缺点是什么？
# a
1. 存储处理能力不足，传统IDE的IOPS为100次/s，SATA固态500次/s，NVMe 2000-4000次/s，无法承受高峰期数万至亿次并发访问；数据操作能力受限于本地挂载的文件系统。
2. 存储空间不足，单机容量有限。

# q
分布式文件系统（DFS）的核心思想是什么？
# a
操作系统不再局限于本地IO，开始通过远端TCP/IP方式获取数据，相当于增加了一种远距离传输I/O技术。它将数据分散到多台独立设备中，由多个单体系统协同合作，解决传统存储的容量和性能限制。分布式存储系统等同于分布式文件系统。

# q
分布式存储系统按存储数据类型如何分类？
# a
- 非结构化数据 → 分布式文件系统
- 结构化数据 → 分布式数据库
- 半结构化数据 → NoSQL数据库

# q
列举常见的分布式文件系统及其典型适用场景。
# a
- Ceph：随OpenStack流行，通用分布式存储
- Gluster：谷歌文件系统思路
- Ocean Store：华为分布式存储
- Lustre：并行文件系统，高性能计算
- HDFS：大规模数据分析场景（Hadoop生态）
- FastDFS：轻量级文件上传与下载
