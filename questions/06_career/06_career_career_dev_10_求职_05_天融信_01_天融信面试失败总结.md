# q
对于40亿个不重复的unsigned int整数，使用位图（bitmap）进行查重大约需要多少内存？时间复杂度是多少？
# a
每个数字用一个 bit 表示，40 亿位约等于：
```
40 * 100 000 000 / 1024 / 1024 / 8 ≈ 476.83 MB
```
构建位图时遍历一次数据，时间复杂度 O(n)；查找时直接检查对应位的值，时间复杂度 O(1)。

# q
内存有限无法加载 40 亿数据时，如何使用分治思想快速判断一个数是否存在？
# a
将 40 亿个数看作是 32 位二进制数。查找时，按目标数的二进制位逐层划分文件：
1. 根据最高位（第 32 位）为 0 或 1 将全部数据分成两个文件，每个文件约含一半数据。
2. 根据目标数的该位值进入对应文件，再按次高位继续划分，重复此过程。
3. 每次划分数据量减半，直到找到或确认不存在，时间复杂度 O(log n)，且每次只需少量内存。

# q
一台 8G 内存服务器要处理 100GB 的 IP 地址文件，要求统计重复次数最多的前 10 个 IP，应采用什么思路？
# a
采用分治 + 哈希映射 + 堆合并的思路：
1. 设计哈希函数，将 100GB 文件分散到约 1000 个小文件，每个约 100MB，可装入内存。
2. 在每个小文件中使用哈希映射统计 IP 出现次数，并记录出现次数最多的前 10 个 IP。
3. 将所有小文件的 top 10 候选（共约 10000 个 IP）汇总，用最大堆或重新统计找出全局 top 10。

# q
面试中当前薪资为 12k，期望薪资报多少比较合理？过高会有什么风险？
# a
通常按 20%～30% 的涨幅，期望报 15k 左右较为合理。报得过高可能让面试官认为超出岗位预算，即使技术过关，也会因薪资分歧而被直接拒绝。
