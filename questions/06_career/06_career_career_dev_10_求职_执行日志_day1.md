# q
在设计 call_center 系统时，如何定义通话状态和人员角色这类有限集合的字段？
# a
使用枚举类（Enum）。例如，定义 `call_status(Enum)` 表示通话状态，`Rank(Enum)` 表示职级（主任、主管、操作员）。重点掌握枚举类的写法，利用枚举保证状态值的类型安全和可读性。

# q
call_center 练习中，人员和通话动作如何抽象？
# a
将人员类别（主管、操作员、主任）抽象为 `class 主管 操作员 主任` 对应的类，将通话动作抽象为独立的 `class call`，让职责更单一、扩展更容易。

# q
求职准备时，如何系统梳理岗位所需技术并制定学习计划？
# a
1. 搜索目标岗位的技术栈要求，归纳核心技能点。  
2. 依此制定分阶段的学习计划，匹配项目实践经验。  
3. 收集 100 份面试经验，提炼共识问题和高频考察点，针对性地准备。
