# q
内核缓冲区是什么
# a
内核缓冲区（Kernel Buffer）是操作系统内核中用于临时存储数据的内存区域，它是内核与硬件、用户空间或不同子系统之间进行数据传输时的“中间暂存区”。其核心作用是协调速度差异、提高IO效率、保证数据一致性。

# q
内核缓冲区按照作用方式分为哪几大类
# a
1. 文件IO缓冲区（Page Cache）：内核将磁盘数据缓存到内存页中，读可直接命中，写先写入缓存后再异步刷盘。
2. 磁盘IO缓冲区（Disk Buffer/Cache）：磁盘控制器或驱动层维护的硬件缓存，与内核缓冲区协同工作。
3. 网络IO缓冲区：每个Socket的发送缓冲区（Send Buffer）和接收缓冲区（Receive Buffer），用于平衡CPU处理速度与网络带宽，应对拥塞。

# q
文件IO缓冲区（Page Cache）在读写文件时如何工作
# a
读文件：内核先将数据从磁盘加载到Page Cache，后续对同一数据的读取可直接从Cache获取，无需访问磁盘。
写文件：应用程序调用write()时，数据先存入Page Cache，内核在缓冲区满、调用fsync()、close()或定时回写线程时将数据刷到磁盘。作用是大幅减少磁盘直接访问，提升频繁小文件读写性能。

# q
网络IO缓冲区的工作原理是什么
# a
发送数据：应用程序调用send()后，数据先进入内核Send Buffer，网络协议栈（如TCP/IP）再将数据分包、添加协议头，通过网络接口发送。
接收数据：网卡收到数据包后存入内核Receive Buffer，再通过中断通知应用程序，应用程序调用recv()从Buffer读取。作用：平衡CPU与网络速度差异，应对网络拥塞或流量波动。
