# q
Lustre实现强一致性的核心机制是什么？
# a
Lustre通过字节范围锁（Byte Range Locking）和元数据/数据锁分离机制实现强一致性。字节范围锁以文件偏移范围为粒度（如[0,1MB]）申请读锁（PR，共享）或写锁（EXCL，独占），锁冲突时由DLM触发锁撤销流程，强制客户端刷新脏数据到OST，保证后续操作基于最新数据。元数据锁由MDS管理（如文件创建需获取父目录EX锁），数据锁由OST管理，两者解耦可提升并发性能。

# q
Lustre的字节范围锁如何处理并发写冲突？
# a
当多个客户端申请冲突锁时，分布式锁管理器DLM会触发锁撤销流程（Revoke），强制持有旧锁的客户端将脏数据写回OST，然后撤销其锁授权，再授予新锁。写锁（EXCL）为独占锁，会阻止其他客户端对同一区域的任何读写操作，从而避免并发写冲突。例如，客户端A持有区域[0,1MB]的写锁，客户端B申请同一区域的写锁时会触发A被撤销并回刷数据，B获取锁后看到最新数据。

# q
Lustre如何通过元数据锁与数据锁分离保证原子性和性能？
# a
元数据锁由MDS管理，用于控制目录树和文件属性操作，例如文件创建需获取父目录的独占锁（EX锁），确保创建过程的原子性。数据锁由OST管理，用于控制文件数据块的访问。两者分离使得元数据操作和I/O操作可以并发处理不同锁资源，避免无关操作相互阻塞，从而在高并发场景下提升整体性能。
