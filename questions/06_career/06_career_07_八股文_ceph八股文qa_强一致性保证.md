# q
ceph pg 如何实现其强一致性，一个io下来，是先写元数据还是数据，如果写数据过程中，整机断电，如何保证数据一致性
# a
1. Ceph PG 的强一致性是通过Primary-副本原子同步+WAL日志机制实现的。
2. 写IO会把数据和元数据打包进一个事务，原子写入（不是先写元数据或先写数据），而是事务化同步写入所有副本。
3. 如果写数据期间节点或整机断电，OSD重启时用WAL/journal恢复，所有PG保证要么都commit新数据，要么都维持旧数据，不会出现“有的commit，有的没commit”导致不一致，更不会客户端读到乱序/脏数据。

# q
如果事务连本地副本都没有写入，其他副本也没有写入事务，此时中断会发生什么
# a
可以把它看作“一个写操作还没真正放进账本、保险箱，突然断电，恢复后大家都发现，这件事相当于没发生过”。
如果事务既没写入本地副本，也没写入其他副本，突然中断，不会有任何数据不一致或持久化问题。系统恢复后所有副本均为旧数据，写操作不会被“半执行”。这种情况是分布式系统设计里标准的安全场景。

此次写操作“没有被提交”，也没有对任何副本产生副作用。
无论此次崩溃、中断如何，整个分布式系统里所有副本的数据仍然是“写操作之前”的（旧数据）。
外界（客户端、其他OSD）对这次写入结果“不可见”，客户端最多收到超时或失败，没有收到成功确认。
