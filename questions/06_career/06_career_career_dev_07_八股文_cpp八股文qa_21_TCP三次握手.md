# q
TCP三次握手的过程是怎样的？
# a
- 第一次握手：客户端发送 SYN 报文，seq=x，请求建立连接。
- 第二次握手：服务端收到后回复 SYN+ACK，ack=x+1，seq=y，表示收到并同意建立连接。
- 第三次握手：客户端再发送 ACK，ack=y+1，确认收到服务端的同意，连接建立。

# q
TCP为什么需要三次握手？
# a
主要有两个目的：1. 规避网络中延迟的历史错误连接请求；2. 同步双方的初始序列号，保证可靠传输。

# q
为什么TCP握手是三次而不是两次或四次？
# a
- 两次握手不安全：无法防止已失效的连接请求到达服务端时建立错误的连接，缺少对客户端的确认。
- 四次握手没有必要：三次握手已经能在最小代价下既防止旧连接，又完成序列号同步，增加次数只会浪费资源。
