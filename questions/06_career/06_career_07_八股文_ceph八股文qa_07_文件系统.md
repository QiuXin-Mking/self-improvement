# q
LOCAL（局部分配，也叫“内联分配”）的特点是啥
# a
当文件比较小时（通常是很小的文件，比如几 KB 甚至更小），XFS 不会给它单独分配外部的“区段（extent）”或者“B 树节点”，而是把文件的元数据和实际数据直接存在 inode（索引节点）所在的块里，或者说“内联”在 inode 周围的一小片区域。

对小文件（比如配置文件、小图片、短文本等）的存储和访问友好，能显著提升小文件的 CRUD（创建、读取、更新、删除）性能。

# q
XTENTS 区段分配的核心特点是啥
# a
对于中等大小或较大的文件，XFS 会把文件数据分成一段一段连续的“区段（extent）”来存储。每个 extent 记录了「起始块号 + 块数」这样的信息，相当于告诉文件系统：“这个文件的一段数据连续存放在磁盘的这些块上”。

磁盘上“连续存储”的数据读写效率往往更高（减少磁头寻道、旋转延迟等）。用 extents 来组织文件，既能利用“局部连续”的 IO 优势，又不需要像早期文件系统那样把所有块号都塞到 inode 里（inode 空间有限）。

# q
BTREE（B 树索引） 核心特点是啥
# a
当文件变得非常大（比如几个 GB、几十 GB 甚至更大），或者文件的 extents 数量太多（超过一定阈值，inode 里放不下了），XFS 就会启用B+ 树​​ 来管理文件的 extents 信息。

如果文件极大、extents 太多，再用“线性存储 extent 信息”（比如全存在 inode 或一个数组里）就会导致元数据太大、查找太慢。B 树通过“分层索引”把时间复杂度从 O(n) 降到 O(log n)，保证了超大文件下也能快速定位数据块。
