# q
线段树中，区间加法的 lazy 标记（add）在 push_col 或 push_down 时如何传递给子节点？
# a
在 `push_col` 中，如果当前节点 `add[rt] != 0`，则执行：
```c
int m = (l + r) >> 1;
color(lson, add[rt]);  // 将 add[rt] 传给左儿子
color(rson, add[rt]);  // 传给右儿子
add[rt] = 0;
```
其中 `color(l, r, rt, v)` 的作用是：`sum[rt] += (r - l + 1) * v; add[rt] += v;`

# q
线段树同时支持区间加和区间乘时，如何合并两种操作到一个懒标记更新函数中？
# a
定义一个 `color(l, r, rt, a, b)` 函数，表示对区间执行 `sum = sum * a + (r-l+1) * b`。更新时：
```c
sum[rt] = (sum[rt] * a) % P + (long long)(r - l + 1) * b; sum[rt] %= P;
add[rt] = (add[rt] * a + b) % P;
mul[rt] = (mul[rt] * a) % P;
```
其中 `add` 和 `mul` 初始值分别为 0 和 1。`push_col` 判断 `mul[rt] != 1 || add[rt] != 0` 时用当前 `mul`、`add` 值调用 `color` 向下传递。

# q
线段树节点存储左右边界 `l, r` 的结构体方式（如 codevs 1080）中，区间查询的递归终止条件是什么？
# a
如果当前节点区间完全等于查询区间，即 `t[k].l == l && t[k].r == r`，则直接返回 `t[k].sum`，不再递归。否则需要判断查询区间与左右子区间的关系，分别递归或组合求和。

# q
在单点查询的线段树实现中（如 codevs 1081），如何通过遍历路径上的懒标记获取某个位置的真实值？
# a
从根节点开始，沿着查询位置所在的路径递归，累加经过节点的 `d`（即懒标记）。最终答案 = 原始数组值 `a[pos]` + 路径累加的 `ans`。递归函数 `ask(k, count)` 若 `count` 不在当前节点区间内则剪枝返回，否则累加 `tree[k].d` 并继续与左右儿子递归。

# q
线段树递归宏（如 `lson` 和 `rson`）的典型定义是什么？在代码中如何提高可读性？
# a
通常定义为：
```c
#define lson l, m, rt << 1
#define rson m + 1, r, rt << 1 | 1
```
在使用时，`build(lson)` 和 `modify(lson, nowl, nowr, v)` 可以简明地传递区间范围和子节点编号，避免重复书写参数。
