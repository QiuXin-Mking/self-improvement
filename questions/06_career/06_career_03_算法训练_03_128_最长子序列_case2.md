# q
这段代码在LeetCode 128最长连续序列问题中逻辑错误的根本原因是什么？
# a
核心错误在于查找序列起点的条件判断写反了。代码中写成 `if ( 0 != set_a.count(num-1) )`，即当前数字的前驱存在时才进入计数循环，这导致只对非起点元素进行计数，且计数方向是向前递减查找更小的数（`cur--`）。正确逻辑应当是判断前驱不存在（`set_a.count(num-1) == 0`）时才作为起点，并向后递增（`cur++`）计算连续长度。

# q
如何修复这段代码使其正确解决最长连续序列问题？
# a
需要修改两处：
1. 将入口条件改为：`if (set_a.count(num - 1) == 0)`，确保只从连续序列的起点开始计数。
2. 将内层 while 循环的查找方向改为向后递增，即 `while (set_a.count(cur + 1))` 然后 `cur++`，并递增长度 `lev++`。
修正后的核心片段：
```cpp
for (auto num : set_a) {
    if (set_a.count(num - 1) == 0) {  // 确保是起点
        int cur = num;
        int lev = 1;
        while (set_a.count(cur + 1)) {
            cur++;
            lev++;
        }
        ans = max(ans, lev);
    }
}
```

# q
从“第二次优化”的命名和代码逻辑推测，这次改动尝试解决什么问题？又引入了什么新问题？
# a
初步推测“第一次优化”可能是直接对每个元素双向扩展，时间复杂度高。此版本尝试只从序列起点开始计数以减少重复计算。但由于条件 `if ( 0 != set_a.count(num-1) )` 取反，本质上变成了“只有当 num 不是起点时才计数”，导致完全跳过真正的起点；同时计数方向错误（减 1 向前找），不仅无法获得正确长度，还可能对同一个序列的非起点元素重复向前遍历，反而引入了更严重的逻辑错误。
