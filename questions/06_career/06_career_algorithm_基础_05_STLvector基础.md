# q
vector容器的主要特点是什么？包括其存储方式、扩容机制和性能权衡。
# a
vector 是一个可变大小的序列容器，元素使用连续的动态分配存储空间，支持通过指针或偏移量直接访问。当元素数量超过当前容量时会自动扩容，但为了减少重新分配带来的性能开销，vector 会预分配额外空间，使实际容量通常大于所需大小。这种策略以消耗更多内存为代价，换取了高效的动态增长能力。定义于头文件 `<vector>`，声明为 `template<class T, class Allocator = std::allocator<T>> class vector;`。

# q
在 C++ 中，使用 `cin` 和 `getline` 读取 `string` 时，它们的分隔行为有何不同？混用时应注意什么问题及如何解决？
# a
- `cin` 以空格、制表符等空白字符为分隔，遇到空格停止读取，并会自动吃掉（忽略）前面的换行和空格，因此 `cin` 读取的字符串不会为空。
- `getline` 以换行符 `\n` 为分隔，遇到换行停止读取，但它不会自动吃掉前导的换行符。如果混用 `cin` 和 `getline`，前一个 `cin` 留下的换行符可能被 `getline` 立即读取，导致获取到空串。
- 解决方法：在 `cin` 之后添加一个 `getchar()` 或一个多余的 `getline` 用来消耗掉残留的换行符。

# q
`std::string` 的大小比较遵循什么规则？
# a
按照字典序逐个字符比较其 ASCII 码值。遇到第一对不相等的字符时，ASCII 码较大的字符串就更大。如果所有对应字符都相等，则较长的字符串更大（短串是长串的前缀时，长串更大）。

# q
`std::string` 的常用成员函数有哪些？简要说明其功能或时间复杂度。
# a
- `size()`：获取字符串长度，O(1).
- `insert(pos, str)`：在指定位置插入字符串，O(n).
- `append(str)` 或 `+=`：在末尾追加字符串，均摊 O(1).
- `erase(pos, len)`：删除从指定位置开始的若干字符，O(n).
- `substr(pos, len)`：截取从指定位置开始的若干字符构成子串，O(len).
- `back()`：返回最后一个字符引用。
- `pop_back()`：删除最后一个字符，O(1).
- `empty()` 或 `s == ""`：判断字符串是否为空。
- `clear()` 或 `s = ""`：清空字符串。

# q
`std::string` 有哪两种常用的遍历方式？给出示例代码风格。
# a
1. 下标遍历：`for(int i = 0; i < s.size(); i++) { cout << s[i]; }`
2. 范围 for 循环（foreach）：`for(char c : s) { cout << c; }`
