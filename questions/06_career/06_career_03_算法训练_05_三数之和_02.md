# q
在 C++ for 循环内部，为什么使用 `while (fir > 0 && nums[fir] == nums[fir-1]) { fir++; }` 进行去重是灾难性的？
# a
因为在 for 循环体内直接递增循环变量 `fir`，当 while 条件不满足时会继续执行循环体内的后续代码，而不是跳出本次循环。这会导致逻辑混乱（可能跳过应处理的元素或造成死循环）。正确做法是使用：
```cpp
if (fir > 0 && nums[fir] == nums[fir-1]) {
    continue;
}
```

# q
写双指针循环时，检查条件 `nums[sec] + nums[thr] > target && sec < thr` 有什么潜在风险？正确的写法是什么？
# a
这种写法先访问数组元素再判断 `sec < thr`，可能导致 `sec` 或 `thr` 越界，引发未定义行为。必须先检查边界再访问元素，正确写法：
```cpp
while (sec < thr && nums[sec] + nums[thr] > target) {
    thr--;
}
```

# q
三数之和问题中，对第一个数 `fir` 进行去重的正确做法是什么？请给出代码片段。
# a
在 `for` 循环遍历第一个数时，使用 `if` + `continue` 跳过与前一个数相同的值：
```cpp
for (fir = 0; fir < n; fir++) {
    if (fir > 0 && nums[fir] == nums[fir-1]) {
        continue;
    }
    // 后续双指针逻辑
}
```

# q
双指针模板用于三数之和时，三个关键要点是什么？
# a
1. 定义遍历类型：外层固定第一个数，内层使用两个指针相向移动（sec 左向右，thr 右向左）。
2. 定义遍历边界：`fir` 从 `0` 到 `n-1`，`sec` 从 `fir+1` 到 `n-1`，`thr` 初始化为 `n-1`。
3. 始终保证 `sec < thr`，当 `sec == thr` 时内循环终止，避免越界和重复。
