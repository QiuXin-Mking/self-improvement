# q
最长连续子序列问题第二次优化版本中，导致结果错误的典型根因是什么？
# a
根因是错误地选择“有前驱的元素”作为计数起点，且只向左扩展，导致所有孤立元素（连续序列长度为 1）被跳过。同时 `ans` 初始为 0，在只有孤立元素时从未更新，最终错误返回 0。关键代码：
```cpp
if ( 0 != set_a.count(num-1) )   // 仅在前驱存在时进入
{
    int cur = num;
    int lev = 1;
    while ( 0 != set_a.count(cur-1) )  // 只向左扩展
    {
        lev++;
        cur--;
    }
    ans = max(ans,lev);
}
// ans 初始为 0，孤立元素从不更新 ans
```

# q
如何从代码结构快速定位该算法无法正确处理长度为 1 的连续序列？
# a
检查遍历逻辑：`for(auto num : set_a)` 内使用 `if ( 0 != set_a.count(num-1) )` 作为计数入口，直接过滤掉了所有没有前驱的元素。若输入中所有元素的 `num-1` 都不存在（如 `[5]` 或 `[2,7,9]`），则计数循环体永不被执行，`ans` 保持 0 并作为结果返回，导致长度为 1 的连续序列被错误输出为 0。

# q
解决该问题并得到正确实现的标准流程是什么？
# a
1. 将数组全部元素放入 `unordered_set`；
2. 遍历集合，对每个元素检查其前驱 `num - 1` 是否存在；
3. 仅当 `num - 1` 不存在时，以 `num` 为连续序列起点，向右扩展计数：
   ```cpp
   int cur = num;
   int len = 1;
   while (set_a.count(cur + 1)) {
       cur++;
       len++;
   }
   ans = max(ans, len);
   ```
4. 最后返回 `ans`。这样可以正确处理包括长度为 1 在内的所有情况，且避免了重复计算。
