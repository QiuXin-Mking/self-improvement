# q
协程和线程在调度方式上有什么区别？
# a
线程由操作系统内核调度和管理（内核级线程），切换时涉及上下文切换和内核态/用户态转换，代价较高。协程完全在用户态进行调度和切换，切换速度极快，不涉及内核和系统调用，属于协作式调度，需代码主动让出控制权（如 yield/await）。

# q
协程和线程在多核并行能力上有什么区别？
# a
线程在多核CPU上可以真正并行运行。协程同一时刻只有一个协程在运行（单个线程内），需要主动让出控制权，本质上是“伪并发”；除非多个线程各自运行自己的协程调度，否则协程不利用多核并行。

# q
协程和线程在资源消耗上有什么区别？
# a
线程每个需要独立的栈空间（一般几十KB到几MB），大量线程会消耗较多内存和调度资源。协程栈空间很小（几KB），同等资源下可创建的数量远多于线程，调度消耗低。

# q
协程和线程在创建与销毁的代价上有何区别？
# a
创建和销毁线程需要操作系统支持，速度较慢。协程的创建和销毁只涉及用户空间的数据结构，速度非常快。

# q
协程和线程分别适合什么场景？
# a
线程适合需要真正并行的计算密集型任务，以及需要和操作系统/硬件交互的场景。协程适合大量IO密集型任务、高并发但不需要CPU同时并行的业务，如高并发Web服务器、爬虫等。
