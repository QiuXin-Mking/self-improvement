# q
LRU 替换算法的基本思想是什么？
# a
LRU（Least Recently Used，最近最少使用）算法在 cache 组已满且发生访问缺失时，选择最近最久未被使用的地址（物理块）进行替换。通过维护一个时间表记录每个物理块上次访问到当前的时间差，每次访问后更新时间记录，淘汰时间值最大的块。

# q
在 LRU 实现中，如何记录每个物理块的最近使用时间并找到最久未使用的块？
# a
使用一个与物理块数量相同的时间数组 `time[blockN]`，初始为 0。每次访问一个地址时：
- 对非空块（`block[j] != -1`），`time[j]++` 增加未访问时长。
- 若命中（`block[j] == list[i]`），将该块对应 `time[j]` 重置为 0。
- 若未命中且需要替换，遍历 `time` 数组找到值最大的索引 `max`，将该块替换为新地址，并重置 `time[max] = 0`。

# q
LRU 代码中，未命中（失效）次数和替换次数是如何定义的？
# a
- `miss` 计数：每当发生未命中（无论物理块是否已满）时，`miss` 加 1。包括直接存入空块和需要替换的情况。
- `rep` 计数：仅在“未命中且物理块已满”发生替换（即 `j == blockN`）时，`rep` 加 1。
- 因此 `miss` 包含了所有未命中，`rep` 则是其中发生替换的未命中次数。`hit`（命中次数）= 总访问次数 - `miss`。

# q
上述 C 代码中，当访问序列长度和路数固定时，失效率和替换率如何计算？
# a
```c
printf("失效次数:%d   失效率:%d/%d=%.2f\n", miss, miss, listN, (float)miss/(float)listN);
printf("替换次数:%d  替换率:%d/%d=%.2f\n", rep, rep, listN, (float)(listN - miss)/(float)listN);
```
失效率 = 失效次数 / 访问地址总数（`listN`）。  
替换率 = 替换次数 / 访问地址总数，但这里实际打印的是 `(listN - miss)/listN`，即命中率，变量命名或注释可能有偏差。根据输出文本“替换率:”后的公式，实际计算的是命中率。
