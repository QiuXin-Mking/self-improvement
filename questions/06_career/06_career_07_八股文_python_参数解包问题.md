# q
Python中关于参数解包，下列哪个说法是错误的？  
A. 函数fn的参数a和b都是位置参数  
B. 不能使用参数解包将列表或元组中的元素传递给函数  
C. 程序输出3  
D. 程序不会抛出异常

```python
def fn(a, b):
  return a + b

lst = [1, 2]
f = fn(*lst)
print(f)
```
# a
错误选项：B。  
Python完全支持使用 `*` 操作符对列表、元组等可迭代对象进行解包，将其元素作为位置参数传递给函数。`fn(*lst)` 等价于 `fn(1, 2)`。  
其他选项分析：  
- A 正确：`fn(a, b)` 中 a 和 b 都是位置参数。  
- C 正确：`fn(1, 2)` 返回 `1 + 2 = 3`，输出 3。  
- D 正确：语法和逻辑均无异常。

# q
Python中如何使用 `*` 和 `**` 将列表、元组或字典的元素传递给函数？
# a
- 位置参数解包：  
  ```python
  def fn(a, b):
      return a + b
  lst = [1, 2]
  result = fn(*lst)  # 等价于 fn(1, 2)
  ```
- 关键字参数解包：  
  ```python
  def fn(a, b):
      return a + b
  d = {'a': 1, 'b': 2}
  result = fn(**d)  # 等价于 fn(a=1, b=2)
  ```
- 组合使用实例：  
  ```python
  fn(0, *[1, 2], **{'c': 3})
  ```

# q
Python 中 `*` 和 `**` 操作符在函数调用中的核心区别是什么？
# a
- **`*`**：解包可迭代对象（列表、元组等）为 **位置参数**。  
  ```python
  lst = [1, 2, 3]
  fn(*lst)  # → fn(1, 2, 3)
  ```
- **`**`**：解包字典为 **关键字参数**。  
  ```python
  d = {'a': 1, 'b': 2}
  fn(**d)  # → fn(a=1, b=2)
  ```
- 调用时 `*` 和 `**` 可以同时出现，位置参数必须出现在关键字参数之前。

# q
Python 函数参数中的位置参数和关键字参数有什么区别？
# a
- **位置参数**：按顺序传递，由位置决定对应关系。  
  ```python
  def fn(a, b):
      return a + b
  fn(1, 2)  # a=1, b=2
  ```
- **关键字参数**：通过参数名指定，顺序可任意。  
  ```python
  fn(b=2, a=1)  # a=1, b=2
  ```
- 混合使用时，位置参数必须放在关键字参数之前：`fn(1, b=2)` 合法；`fn(a=1, 2)` 会引发 `SyntaxError`。
