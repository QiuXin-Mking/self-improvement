# q
100亿个int型数字（约37.2GB）无法一次性装入内存，如何进行排序？
# a
采用“哈希分片+内部排序+归并”的思路：
1. 用`fopen()`打开大文件（不会全部加载到内存），遍历每个数字，将其对1000取模，得到0~999共1000个分片编号，分别写入对应的小文件中。
2. 每个小文件平均大小约38MB，可以一次性读入内存进行排序（如快排）。
3. 使用归并算法将所有已排序的小文件合并成一个全局有序的大文件。整个过程中可以使用`mmap()`来打开这些小文件和大文件，避免占用用户进程的内存。

# q
在这个排序方案中，`mmap()`系统调用主要解决什么问题？
# a
`mmap()`用于打开1000个小文件和1个大文件，文件映射到进程地址空间，但读取时按需加载，不占用用户进程的私有内存，从而避免因文件过多或过大导致内存压力。

# q
如何将37.2GB的大文件均匀拆分成约38MB的小文件？
# a
通过遍历文件中的每个int数字，计算`数字 % 1000`，根据余数将数字写入对应编号的小文件中。理想情况下，哈希均匀分布，每个小文件约37.2GB / 1000 ≈ 38MB。
