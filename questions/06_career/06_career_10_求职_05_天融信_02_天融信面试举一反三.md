# q
为什么100亿个int数字无法直接全部加载到内存中进行排序？
# a
100亿个int数字约占据37.2GB空间，远超常见服务器的内存容量，无法一次性装入内存处理。

# q
面对100亿个数字的大文件排序，基本的外排序（External Sort）思路是什么？
# a
采用哈希分片 + 归并排序。先将大文件遍历，把每个数字对1000取模，分散写入1000个小文件（理想每个约38MB），每个小文件可单独读入内存排序；最后将所有有序小文件通过归并合并成一个有序的大文件。

# q
在合并1000个已排序小文件时，如何高效且低内存占用完成归并？
# a
可以使用 ```mmap()``` 系统调用，将1000个小文件和最终大文件映射到进程地址空间，直接进行内存操作，文件数据不会全部占用用户进程内存，从而实现低内存开销的多路归并。
