# q
盛水最多的容器算法实现中，传入空数组时程序崩溃的典型根因是什么？
# a
未对空输入进行校验。代码中 `int l=0,r=0,size=height.size()-1,ans=0; r=size;` 当 `height` 为空时，`height.size()` 返回 `0`（`size_t` 类型），减 `1` 发生无符号下溢，得到一个极大的值赋给 `size`，进而 `r=size` 导致后续 `height[l]` 或 `height[r]` 访问越界崩溃。

# q
如何修复上述空数组导致的越界崩溃？
# a
在函数入口增加空数组判断：
```cpp
if (height.empty()) return 0;
```
或者将 `size` 的计算改为 `int size = (int)height.size() - 1;` 但仍需结合空检查，避免无效的指针初始化。

# q
如何通过排查双指针移动逻辑，定位盛水最多容器计算结果错误的问题？
# a
检查三个关键点：
1. 面积计算是否正确：`area = min(height[l], height[r]) * (r - l)`。
2. 指针移动方向是否基于较小高度：`if (height[l] < height[r]) l++; else r--;`，必须移动较矮的一侧才可能找到更大面积。
3. 循环条件是否严格为 `l < r`，防止访问越界或提前结束导致漏算。若出现错误结果，可打印每次迭代的 `l`、`r`、`area`，比对预期值来定位移动逻辑偏差。
