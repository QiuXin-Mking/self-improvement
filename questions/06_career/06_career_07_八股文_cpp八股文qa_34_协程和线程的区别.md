# q
协程和线程在调度机制上有什么区别？
# a
线程：由操作系统内核调度和管理（内核级线程）。线程切换时需要上下文切换，涉及内核态和用户态的转换，代价较高。
协程：完全在用户态对执行权进行调度和切换，切换速度极快，不涉及内核和系统调用。

# q
协程和线程在多核CPU上的并行性有何不同？
# a
线程：在多核CPU上，多个线程可并行运行。
协程：同一时刻只有一个协程运行（除非多个线程各自有自己的协程调度），本质上是“伪并发”，需要主动让出控制权（yield/await等）。

# q
协程和线程的资源消耗及创建销毁代价有何差异？
# a
线程：每个线程需要分配独立的栈空间（一般为几十KB到几MB），大量线程会消耗较多资源。创建和销毁线程需要操作系统的支持，速度较慢。
协程：每个协程的栈空间很小（几KB），同等资源下可创建数量远多于线程，调度消耗低。创建和销毁只涉及用户空间的数据结构，速度非常快。

# q
协程和线程分别适用于什么场景？
# a
线程：适合需要真正并行处理、需要和操作系统/硬件交互的场景，如计算密集型任务。
协程：适合大量IO密集型任务、需要高并发但不需要CPU同时并行的业务，如高并发Web服务器、爬虫等。

# q
线程和协程的调度方式有何不同？
# a
线程：由操作系统调度（抢占式调度）。
协程：完全由程序员或中间件调度（协作式调度，依赖代码主动让出控制权）。
