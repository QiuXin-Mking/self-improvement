# q
mmap零拷贝原理是什么
# a
通过mmap把文件的页缓存 Page Cache 映射到进程的虚拟地址空间，用户态直接读写同一份内核缓冲，省去“内核→用户”的那次CPU拷贝。以读文件传输到网络为例，数据链路变为：磁盘 → DMA → 内核页缓存 → socket缓冲 → DMA → 网卡。相比传统的 read + write 链路（磁盘 → DMA → 内核页缓存 → CPU拷贝 → 用户缓冲区 → CPU拷贝 → 内核socket缓冲区 → DMA → 网卡，共2次CPU拷贝、2次DMA、4次上下文切换），mmap 消除了用户态的CPU拷贝，减少上下文切换，并让用户程序避开操作系统直接访问硬件存储。

# q
sendfile 的原理是什么
# a
sendfile 可替代 read() 和 write() 两个系统调用，减少一次系统调用，从而减少 2 次上下文切换。该系统调用直接把内核缓冲区里的数据拷贝到 socket 缓冲区，不再经过用户态，总共只有 2 次上下文切换和 3 次数据拷贝（包括 DMA 拷贝）。
