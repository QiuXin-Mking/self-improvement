# q
智能指针主要为了解决普通指针的哪两个问题？
# a
1. 忘记释放内存，导致内存泄漏。
2. 尚有指针引用内存时提前释放，导致其他指针指向非法内存（悬空指针）。

# q
`unique_ptr` 的核心特点是什么？
# a
- 独占所有权：同一时间只能有一个 `unique_ptr` 指向资源。
- 禁止拷贝，仅支持移动语义（通过 `std::move` 转移所有权）。
- 轻量高效：无引用计数开销，性能接近裸指针。
- 自动释放：超出作用域或被重置时，自动调用 `delete` 或自定义删除器释放资源。

# q
`weak_ptr` 如何安全访问其观测的对象？
# a
必须通过 `lock()` 方法获取临时的 `shared_ptr` 后再访问，否则会导致未定义行为。
- `lock()`: 若对象存在，返回有效的 `shared_ptr`；若已释放，返回空指针。
- 可使用 `expired()` 检查对象是否已销毁（返回 `true` 表示已失效）。
- `weak_ptr` 不参与引用计数，不影响对象生命周期，对象是否释放完全由 `shared_ptr` 决定。
