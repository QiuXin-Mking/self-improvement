# q
线段树模板中 `#define lson l,m,rt<<1` 和 `#define rson m+1,r,rt<<1|1` 的作用是什么？
# a
这两个宏用于简洁地表示递归调用线段树左右子树的参数：`lson` 代表左子节点的区间范围 `[l, m]` 和节点编号 `rt*2`；`rson` 代表右子节点的区间范围 `[m+1, r]` 和节点编号 `rt*2+1`，通过宏简化 `build(lson)` 等调用。

# q
在区间加、区间求和的线段树（模板 1）中，`push_col` 函数如何将 lazy 标记下传？
# a
`push_col(l, r, rt)` 检查当前节点的 `add[rt]` 是否非零，若存在标记则执行：

```cpp
int m = (l+r)>>1;
color(lson, add[rt]);  // 对左子节点进行区间加操作，更新其sum和add
color(rson, add[rt]);  // 对右子节点做同样操作
add[rt] = 0;           // 清空当前节点标记
```

`color` 函数内部会累加 `sum` 和 `add`，实现延迟标记的正确传递。

# q
同时支持区间加和区间乘的线段树（模板 2）中，`color` 函数如何更新 `sum`、`add` 和 `mul`，以保持乘法的优先级？
# a
`color(l, r, rt, a, b)` 的参数 `a` 为乘因子，`b` 为加数，更新公式遵循“先乘后加”：

```cpp
sum[rt] = (sum[rt] * a) % P + (r - l + 1) * b; sum[rt] %= P;
add[rt] = (add[rt] * a + b) % P;
mul[rt] = (mul[rt] * a) % P;
```

即子区间的和先乘以 `a` 再加上区间长度乘 `b`，原有的加法标记也要先乘后加，乘法标记直接累乘。

# q
对于"区间修改、单点查询"的线段树（如 codevs 1081），如何通过标记累加实现？
# a
不维护 `sum`，每个节点只保存一个 `d`（增量标记）。区间修改时若当前区间完全被覆盖，直接将 `d` 增加 `v` 并返回；否则向下递归左右子节点修改。单点查询时，从根到目标叶子节点的路径上，将经过的所有节点的 `d` 累加，最终结果为 `ans + a[pos]`（原始数组值加上路径上的总增量），无需 push_down。
