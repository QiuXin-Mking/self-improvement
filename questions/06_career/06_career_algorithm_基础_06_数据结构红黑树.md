# q
红黑树的5条特性规则是什么？
# a
1. 节点是红色或者黑色。
2. 根节点是黑色。
3. 叶子节点（外部节点、空节点）都是黑色。
4. 红色节点的所有子节点都是黑色（即不能有两个连续的红色节点）。
5. 任一节点到其叶子节点的路径上包含相同数目的黑色节点。

# q
为什么需要红黑树？它解决了二叉搜索树的什么问题？
# a
二叉搜索树在插入有序数据时可能退化为链表，时间复杂度变为 O(N)。红黑树通过自平衡特性，确保在最坏情况下查找、插入、删除的时间复杂度均为 O(logN)。

# q
红黑树与4阶B树（2-3-4树）的等价关系是什么？
# a
红黑树和4阶B树具有等价性：黑色节点与其红色子节点融合在一起，形成一个B树节点。红黑树的黑色节点个数等于对应4阶B树的节点总个数。在B树节点中，黑色节点为父节点（中间），红色节点为子节点（两侧）。

# q
红黑树与AVL树的性能对比如何？
# a
- AVL树时间复杂度优于红黑树，但现代CPU速度下差异可忽略。
- 红黑树的插入和删除操作更便于控制，旋转次数少于AVL树。
- 红黑树整体性能略优于AVL树。

# q
红黑树右旋操作的步骤是什么？（以节点M右旋为例）
# a
1. 将节点M的左孩子引用指向节点E的右孩子。
2. 将节点E的右孩子引用指向节点M，完成旋转。
