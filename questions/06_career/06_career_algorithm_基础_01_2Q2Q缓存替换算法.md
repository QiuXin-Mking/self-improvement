# q
2Q 缓存替换算法的基本结构由哪两个队列组成？各自采用什么淘汰策略？
# a
2Q 算法包含一个 **FIFO 队列** 和一个 **LRU 队列**。FIFO 队列按先进先出规则淘汰，LRU 队列按最近最少使用规则淘汰。

# q
在原始 2Q 算法中，数据从 FIFO 队列迁移到 LRU 队列的条件是什么？
# a
当数据在 FIFO 队列中被**第二次访问**时，会从 FIFO 队列中移除，并添加到 LRU 队列的头部。

# q
改进后的 2Q 算法将 A1 队列拆分为哪两个队列？它们分别采用什么淘汰策略？
# a
改进后的 2Q 将 A1 拆分为 **A1in**（FIFO 队列，缓存实际数据）和 **A1out**（FIFO 队列，仅存储已淘汰页的标识符，不缓存数据）。

# q
在改进版 2Q 算法中，一个数据页从首次访问到进入 Am 队列需要经历什么完整路径？
# a
1. 首次访问：页被放入 A1in 头部。
2. A1in 满时，淘汰尾部的页（设为 Y），将 Y 的标识符放入 A1out 头部。
3. 若 Y 的标识符在 A1out 中被再次访问（命中 A1out），则调用回收过程为其分配页槽，并将该页放入 Am 头部（进入 LRU 管理）。
4. A1out 满时淘汰尾部标识符。

# q
改进版 2Q 中 A1out 过滤器的 cutoff hotness（访问概率）与哪些参数有关？A1out 的大小 f 通常建议设为多少？
# a
A1out 的 cutoff hotness 与 **A1out 的大小 f** 和 **miss 率 m** 相关，近似为概率约 50% 时对象的访问率满足 `p = f/m`。模拟实验表明，当 m 在 10%~90% 时，A1out 的大小与 Buffer size B 的比例固定为 **1/2**；对于 m 更小的系统可更小。
