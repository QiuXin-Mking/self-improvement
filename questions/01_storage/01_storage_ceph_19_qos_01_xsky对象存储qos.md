# q
XSKY 分布式 QoS 解决方案包含哪些关键组件？
# a
包含三个核心模块：
- **MQ（MasterQoS）**：从所有网关中选出的统一决策者。
- **HQ（HandleQoS）**：由 MQ 选取的一批网关，每个 HQ 负责对应一批桶的 QoS 配额分配。
- **令牌桶（Ratelimiter）**：在各网关上运行的本地令牌桶，提供低延迟的本地 QoS 服务。
此外，还有心跳检测与 MQ 切换机制，保证异常时平稳运行。

# q
分布式 QoS 的 HQ 如何分担压力以减少性能瓶颈？
# a
通过桶分类组成不同的桶队列，每个桶队列对应一个 HandleQoS；同时配置多个可交互的 HQ，分摊业务压力；全内存运行、极少量数据写入数据库，降低数据库压力；各网关依靠 HQ 下发的配额独立提供本地令牌桶服务，降低交互延迟。

# q
XSKY 分布式 QoS 方案如何应对网关拓扑变化？
# a
通过心跳检测与 MasterQoS 切换机制，快速发现网关拓扑变化并自动重新选择 MQ 和调整 HQ 分配，实现 QoS 服务的快速恢复；网关无状态特性下也能动态平衡。

# q
本地令牌桶机制在分布式对象存储中的主要优缺点是什么？
# a
- **优点**：响应快、低延迟、能应对突发流量。
- **缺点**：不具备多网关协调能力，无法满足全局分布式 QoS 的统一管控需求。

# q
XSKY 对象存储 QoS 方案中，普通网关与 HQ 之间的交互流程是什么？
# a
普通网关向其所属的 HQ 上报本地的 QoS 使用信息，并从 HQ 获取最新的配额分配信息，随后更新本地令牌桶配置；HQ 负责对应桶队列的全局配额决策，MQ 负责协调 HQ 分配。这种设计使得网关间交互信息尽可能少，配额分配高效。
