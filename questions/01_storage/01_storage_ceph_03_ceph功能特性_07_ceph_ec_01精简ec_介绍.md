# q
什么是Ceph中的纠删码（EC）技术？其核心原理是什么？
# a
纠删码（EC）是一种数据持久化技术，它将数据分割为 `k` 个数据块，并额外生成 `m` 个校验块，总共 `k+m` 个块。这种配置允许最多丢失 `m` 个块而仍然能够重建原始数据，从而在提供类似复制数据保护级别的同时显著减少存储开销。

# q
Ceph中 `ec4+2` 与 `ec4+2:1` 配置有什么区别？
# a
- `ec4+2`：基本纠删码配置，4个数据块+2个校验块，不涉及故障域，最多容忍2个块丢失。
- `ec4+2:1`：在基本配置基础上增加了故障域冗余，`:1` 表示能够容忍一个故障域（如一个机架）的完全失效，适用于多故障域集群以提供更高可靠性。

# q
如何创建自定义的Ceph纠删码规则？
# a
使用以下命令设置纠删码配置文件：
```
ceph osd erasure-code-profile set <规则名> k=<数据块数> m=<校验块数>
```
例如：
```
ceph osd erasure-code-profile set newecrule k=4 m=2
```
之后可以通过该规则创建纠删码池：
```
ceph osd pool create testec 1 1 erasure newecrule
```

# q
在Ceph CRUSH规则中，`step chooseleaf indep 0 type host` 的含义是什么？
# a
该步骤表示在纠删码数据放置时，独立选择类型为 `host` 的叶节点，确保数据分片分布在不同物理主机上，以实现故障域隔离和高可用性。`indep 0` 表示独立选择模式，避免同一主机被重复选中。
