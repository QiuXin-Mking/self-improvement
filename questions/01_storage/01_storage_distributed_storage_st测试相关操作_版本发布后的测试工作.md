# q
版本发布后数据流性能测试的配置和参数是什么？
# a
测试使用12TB存储池，创建12个卷，每个卷容量1TB。测试覆盖512B到1MB共11个块大小粒度，分别进行随机读、随机写、顺序读、顺序写四种模式，组合为44个任务，每个任务持续运行15分钟。

# q
版本发布后的管理性能测试包含哪些具体操作？
# a
测试包括：界面创建room、row、rack的耗时；创建36个OSD和6个OCache的耗时；创建iSCSI、PG和访问路径的耗时；创建300个IN和300个LUN的耗时，并对比第1-2个LUN与第299-300个LUN的创建耗时差异。

# q
mdbs-scan服务测试的目的是什么？
# a
测试创建卷后整个mdbs-scan元数据扫描服务完成扫描所需的时间，以评估该服务在大规模卷创建场景下的性能表现。
