# q
性能测试中通过回环分析性能瓶颈的目的是什么？
# a
目的是发现各个阶段的性能损耗，分析回环后的性能波动原因，判断瓶颈在于管理流程还是通道（chan send/recv）等环节，最终输出性能损耗报告。

# q
在性能回环测试中，如何判断回环是否成功？
# a
通过查询进程的 CPU 占用率，观察 BUS、IO 等指标，使用 `top -H -p <进程数>` 命令确认回环是否成功。

# q
性能回环测试包含哪些关键组件和操作？
# a
组件包括 spdk 回环、proxy 回环、sio 回环、ds 回环（可选）、osd 回环。操作方式：spdk 回环使用命令行（需咨询罗运）；其他回环如 proxy_rollback 通过执行 `echo 1 > proxy_rollback` 触发；测试时长约十分钟。
