# q
如何利用vdbench的扇区内容判断数据不一致是写不一致还是读不一致？
# a
当用 `./vdbench print /dev/<sd*> <sector_lba> 512` 重新读取扇区数据时：
- 如果读出的数据与右侧“找到的数据”一致，但与左侧“预期数据”不一致，则判定为**写不一致**（数据未被正确写入）。
- 如果读出的数据与左侧“预期数据”一致，而与右侧“找到的数据”不一致，则判定为**读不一致**（写入正确但读取返回错误）。

# q
vdbench数据验证中，扇区前32字节的字节偏移量字段（Byte 0x00-0x07）记录了什么？
# a
记录该磁盘块的**字节偏移量**，即扇区的LBA值。例如预期值为`0xa4e98aa00`，实际找到的值为`0xa2fae20000`，通过比较可以判断是否访问了错误的扇区。

# q
vdbench扇区数据中的Data Validation key（Byte 0x10）如何变化，有什么含义？
# a
该字段取值范围 1 - 126，用于标识数据的写入次序：
- `00` 表示**创建写**（首次写入）。
- `01` 表示**第一次覆盖写**，`02` 表示第二次覆盖写，以此类推。
- 当达到 126 后折返为 `00`，循环使用。

# q
vdbench数据块的时间戳字段（Byte 0x08-0x0f）的具体含义是什么？
# a
记录自 1970 年 1 月 1 日以来的**毫秒数**，表示该数据**最后一次写入时的时间**。例如 `00000187fb762a86`。Byte 0x11 是该时间戳的校验和。
