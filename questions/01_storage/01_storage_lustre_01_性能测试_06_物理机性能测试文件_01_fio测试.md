# q
fio命令中的--direct=1参数的作用是什么？
# a
--direct=1 启用直接I/O，绕过操作系统的页面缓存，直接对存储设备进行读写操作，从而获得设备本身的原始性能数据，避免缓存带来的干扰。

# q
如何理解fio测试结果中的IOPS和BW？
# a
IOPS（每秒输入/输出操作次数）衡量存储设备每秒能处理的I/O请求总数，直接反映设备的随机读写能力；BW（带宽）表示数据吞吐量，通常以MiB/s或MB/s显示，代表连续或随机读写的数据传输速率。例如在汇总行中 `READ: bw=10.1MiB/s, IOPS=2597` 即同时给出这两个指标。

# q
从4k随机读测试结果看，NVMe SSD相比/dev/vdb云盘的性能优势主要体现在哪些方面？
# a
NVMe SSD的4k随机读IOPS达到34.4k，带宽135MiB/s，平均完成延迟（clat）约为115us，99.99百分位延迟仅237us；而/dev/vdb云盘IOPS为2597，带宽10.1MiB/s，平均延迟1536us，99.99百分位延迟为49021us。NVMe的IOPS是云盘的13倍以上，延迟降低了一个数量级，具有明显更高的并发处理能力和更低的响应延迟。
