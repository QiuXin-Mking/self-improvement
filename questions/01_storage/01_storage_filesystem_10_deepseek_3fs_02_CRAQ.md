# q
什么是强一致性？
# a
强一致性保证对于单个对象的所有读和写操作均按一定顺序执行，并且对于单个对象的读始终能看到最新的值。

# q
什么是最终一致性？
# a
最终一致性意味着对单个对象的写入仍按一定顺序应用于所有节点，但读操作可能在一段时间内返回旧数据（在写入被应用到所有节点之前）；一旦所有副本都收到写入请求，读操作不会再返回比最近提交的写入版本更旧的版本。

# q
强一致性与最终一致性的主要区别是什么？
# a
强一致性下读总是立即返回最新写入的值；最终一致性下写入后可能因副本未同步而暂时读到旧数据，但最终所有副本都会达到一致状态。

# q
最终一致性中提到的“单调读一致性”是什么？
# a
如果客户端保持与同一个特定节点的会话，对某个对象的读取将返回相同的先前的值或一个更新的值，但绝不会返回更旧的版本。
