# q
Lustre集群弹性伸缩的目标是什么？
# a
实现Lustre集群的节点数量可以根据负载动态增加或减少，以优化资源利用和成本。

# q
在FLR（File Level Redundancy）特性中，如何实现多副本与精简配置？
# a
在一个pool上启用FLR，通过设置文件布局的冗余级别（如mirror count）来实现多副本存储，同时结合精简配置（thin provisioning）以按需分配空间。

# q
为何需要对比Lustre自带的`lfs find`与Linux `find`命令在处理100万小文件时的性能？
# a
为了验证`lfs find`是否采用递归算法并评估其文件遍历效率，以确定在大规模小文件场景下应选用哪个工具进行高效查找。

# q
在Lustre环境中调研RAID5方案的目的是什么？
# a
评估在后端存储（如OST）上使用RAID5级别冗余的可行性与性能，在容量利用率和数据保护之间寻求平衡。

# q
基于pool的文件配额如何实现租户隔离？
# a
通过将Lustre的文件系统池（pool）与配额（quota）关联，可以为不同租户分配专属的存储池，并对其设置容量或文件数限制，从而实现存储资源的隔离与管控。
