# q
什么是内核中的进程上下文？
# a
当前内核代码属于某个进程的系统调用路径，可以调度、可以休眠阻塞（如内核态的read/write、文件IO等）。例如系统调用sys_read进入内核后即处于进程上下文。

# q
中断上下文的核心特点是什么？
# a
中断上下文中的代码处于中断服务程序（ISR）、软中断或tasklet中，不可调度、不可睡眠，只能执行短小快的操作，不能执行yield、sleep等可能阻塞的操作。

# q
进程上下文与中断上下文的主要区别是什么？
# a
进程上下文可调度、可睡眠阻塞，属于进程的系统调用路径；中断上下文不可调度、不可睡眠，必须快速执行，属于中断处理路径。

# q
什么是内核线程上下文？
# a
由内核调度的kthread所处的上下文，拥有独立的调度和阻塞能力，但不属于任何用户进程。
