# q
IDP是什么？
# a
IDP（Identity Provider）即身份提供商，是一个能够创建、管理和验证用户身份信息的实体。在 AWS 环境中，它允许用户使用外部应用或服务（如 Google、Okta）的已有身份访问 AWS 资源，无需在 AWS IAM 中单独创建用户。

# q
在AWS中使用外部IDP的核心优势是什么？
# a
用户可以使用已有的外部身份凭据登录 AWS，无需记住 AWS 特定凭据；同时减少了在多个系统中维护和管理凭据的需要，管理员只需在 IAM 中建立信任关系，即可为外部用户提供访问能力。

# q
外部IDP在AWS中的典型身份验证工作流程是怎样的？
# a
1. 管理员在 AWS IAM 中为外部 IDP 创建配置并建立信任关系。  
2. 用户输入外部 IDP 的用户名和密码登录。  
3. IDP 验证成功后向 AWS 发送包含用户身份信息的断言（如 SAML 断言）。  
4. AWS 验证断言后生成一组临时安全凭据（访问密钥 ID、秘密访问密钥和会话令牌），用户凭此访问 AWS 资源，权限有限且有效期较短。

# q
AWS 支持哪些主要的 IDP 协议？
# a
AWS 支持 OpenID Connect（OIDC）和 SAML 2.0 两种协议，通过这些协议外部 IDP 可与 AWS IAM 进行交互，验证用户身份并返回临时安全凭据。
