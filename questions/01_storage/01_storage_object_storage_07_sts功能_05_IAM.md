# q
在AWS中，IAM是什么？它的主要目的是什么？
# a
IAM（Identity and Access Management，身份与访问管理）是AWS提供的一项服务，用于安全地控制对AWS资源的访问。它集中管理谁通过了身份验证（登录）以及谁获得了授权（拥有权限）来使用资源。

# q
IAM中的用户（User）、组（Group）、角色（Role）和策略（Policy）分别是什么？
# a
- **用户**：具有唯一身份标识和凭证的个人或实体，用于访问AWS资源。
- **组**：一组具有相同权限的AWS用户的集合，便于统一管理权限。
- **角色**：一种临时身份，可分配给AWS资源或服务，用于访问其他AWS资源或执行特定任务，通常不与人直接关联。
- **策略**：一组权限规则，用JSON文档定义用户、组或角色可以对哪些AWS资源执行哪些操作。

# q
IAM支持哪些安全因素来增强账户保护？
# a
IAM支持多因素身份验证（MFA），可以使用短信、虚拟MFA设备或硬件MFA安全密钥等方式，为账户提供额外的安全保护层。

# q
IAM如何实现精细的访问控制和审计监控？
# a
IAM支持**细粒度访问控制**，管理员可根据实际需求为用户、组和角色精确配置权限。同时提供**审计和监控**功能，可查看用户活动日志、API调用记录等，及时发现异常行为。
