# q
`git commit -a --amend` 命令的核心作用是什么？
# a
该命令用于自动暂存所有已跟踪的已修改文件，并修改最近一次的提交（而非创建新提交）。适用于修复小错误、改进提交信息，或将多个小提交合并为一个，从而保持 Git 历史整洁。用法：
```bash
git commit -a --amend
```

# q
`git commit` 的 `-a` 选项具体包含哪些行为？
# a
`-a`（`--all`）选项告诉 Git 自动暂存所有已被跟踪且已修改的文件，这样就不需要手动执行 `git add`。但它不会暂存新添加的未跟踪文件，新文件仍然需要先用 `git add` 添加。

# q
`--amend` 选项在修改最近一次提交时会触发什么操作？
# a
`--amend` 会打开文本编辑器（默认 vim/nano 或 `GIT_EDITOR` 指定的编辑器），其中包含上一次提交的提交信息。你可以修改该信息，或直接保存退出以保留原信息。同时，它会用当前暂存区的内容更新最近一次提交。

# q
在公共分支上使用 `git commit --amend` 修改已推送的提交会带来什么风险？更推荐的做法是什么？
# a
修改公共分支上已推送的历史可能导致他人拉取更新时遇到冲突或混乱，因为提交哈希已改变。更推荐的做法是使用 `git revert` 来撤销错误的提交，而不是修改历史。
