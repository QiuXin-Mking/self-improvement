# q
HTTP 请求头中的 `Authorization: AWS4-HMAC-SHA256` 表示什么？
# a
表示客户端使用 AWS Signature Version 4（签名版本 4）的签名算法 **AWS4-HMAC-SHA256** 为请求生成签名，用于向 AWS 服务验证请求者的身份和请求完整性。

# q
AWS4-HMAC-SHA256 签名机制的核心作用是什么？
# a
核心作用有两个：一是验证请求确实由持有有效 AWS 访问密钥的实体发出（身份验证），二是保证请求在传输过程中未被篡改（完整性保护）。

# q
AWS Signature Version 4 的签名过程涉及哪些关键信息？
# a
签名过程会对请求的多个组件进行哈希和加密，包括请求方法、主机名、请求路径、请求参数等，并使用 AWS 提供的算法（如 HMAC-SHA256）和秘密访问密钥生成最终签名。
