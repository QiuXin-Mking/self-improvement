# q
对象存储中 list 请求为什么会导致 QOS 问题？
# a
从线上故障场景看，每个节点约 20 个 list 请求时可能引发服务器 QOS 问题，因为 list 请求会遍历大量元数据或对象，消耗较多存储系统资源（如 CPU、内存、磁盘 IO），在高并发下容易触发性能瓶颈，导致服务降级或不可用。

# q
针对 syjw 对象存储 list 请求的 QOS 需求包含哪些目标？
# a
需求包含四个主要目的：
1. 通过暴力请求获取上限值，达到阈值时触发告警（基于已有 QOS 切片方案）。
2. 调研 Ceph 源码是否支持直接限制 list 请求。
3. 调研其他厂商的 QOS 方案（如对象存储的元数据操作限流）。
4. 明确当前已有的 QOS 方案，并分析其适用情况。
