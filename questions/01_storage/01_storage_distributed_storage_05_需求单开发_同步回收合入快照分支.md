# q
同步回收合入快照分支的主要目的是什么？
# a
将原本在主线使用的**同步回收（Sync Recycle）逻辑**完整集成到**快照分支（8k 分支）**中，替代原有的异步回收方案，以减少回收延迟、避免异步回收带来的数据一致性问题，并修复版本升级过程中遇到的读写问题。

# q
在快照分支中，同步回收的任务类型有什么变化？
# a
主线使用 `OMT_RECYCLE`，快照分支新增 `OMT_SNAP_RECYCLE`，用于标识快照场景下的同步回收任务，流程入口和状态机处理逻辑与主线的 `OMT_RECYCLE` 对应但独立实现。

# q
MDLOG 协议为支持同步回收新增了哪些日志类型？
# a
新增了 `SEG_DEPEND_DELETED`、`SEG_DELETED`、`SEG_BLOCK_CHANGE` 等 MDLOG 类型，用于记录快照合并过程中的段依赖删除、段删除和块变更信息，保证崩溃恢复后能正确重放。
