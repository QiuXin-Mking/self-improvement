# q
如何使用 Docker 限制 CPU 核心数和内存来启动一个交互式 Ubuntu 容器？
# a
使用以下命令：
```bash
docker run -it --cpus=4 --memory=4g ubuntu:latest /bin/bash
```
其中 `--cpus=4` 将容器可用 CPU 限制为 4 核，`--memory=4g` 将内存限制为 4GB。

# q
在 stress-ng 测试输出中，`cpu` 压力测试的 `real time` 和 `usr time` 分别代表什么？容器测试中 CPU 压力测试的 `bogo ops/s (real time)` 是多少？
# a
- `real time`：实际运行时间，即压力测试从开始到结束的墙上时钟时间（单位秒）。
- `usr time`：CPU 在用户态执行指令所花费的总时间（单位秒）。
容器测试中 CPU 压力测试的 `bogo ops/s (real time)` 为 7760.07。

# q
stress-ng 的 `vm` 压力器主要测试什么？在容器和 KVM 测试的输出中，`vm` 的 `bogo ops/s (real time)` 分别是多少？
# a
`vm` 压力器主要通过分配和释放内存、触发页错误等操作来测试虚拟内存子系统的压力。容器测试中 `vm` 的 `bogo ops/s (real time)` 为 493418.12，KVM 测试中为 46244.80。

# q
对比容器和 KVM 的 `pipe` 压力测试结果，`bogo ops` 和 `bogo ops/s (real time)` 有何差异？这反映了什么？
# a
容器中 pipe 测试的 `bogo ops` 为 126,865,315，`bogo ops/s (real time)` 为 2,114,495.76；KVM 中 `bogo ops` 为 97,672,015，`bogo ops/s (real time)` 为 1,627,860.43。容器环境下的管道吞吐性能明显高于 KVM 虚拟机，说明轻量级虚拟化在 I/O 密集型操作上可能具有更低的性能开销。
