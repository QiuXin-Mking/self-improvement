# q
Docker 启动容器时，如何限制容器使用的 CPU 核心数和内存大小？
# a
使用 `--cpus` 和 `--memory` 参数，例如：
```bash
docker run -it --cpus=4 --memory=4g ubuntu:latest /bin/bash
```
`--cpus=4` 限制 CPU 核心数为 4，`--memory=4g` 限制内存为 4GB。

# q
stress-ng 压力测试工具主要支持哪些类型的压力测试？
# a
stress-ng 可对多种系统资源进行压力测试，常用类型包括：cpu（CPU）、vm（虚拟内存）、io（输入输出）、fork（进程创建）、pipe（管道通信）等。

# q
如何以后台方式运行一个带资源限制的 Docker 容器并使其保持运行？
# a
使用 `docker run -d` 结合 `--cpus`、`--memory` 限制资源，并执行 `sleep infinity` 保持容器存活。示例：
```bash
docker run -d --name test1 --cpus=4 --memory=4g ubuntu:1.0 sleep infinity
```

# q
在 Docker 容器或 KVM 环境中运行 stress-ng 后，如何解读其输出结果？
# a
输出中会显示每个 stressor（如 cpu、vm 等）的指标，包括：
- `bogo ops`：执行的操作总数
- `real time`：实际运行时间（秒）
- `usr time` / `sys time`：用户态/内核态 CPU 时间
- `bogo ops/s`：按实际时间和 (usr+sys 时间) 计算的每秒操作数
- 若有 `passed: X` 表示该 stressor 全部通过，`failed: 0` 表示无失败
