# q
如何将两个特性分支 `feature/new_feature_A` 和 `feature/new_feature_B` 合并到一个新分支 `feature/new_feature_C`，然后将其合并回 `main`？
# a
```bash
git checkout main
git checkout -b feature/new_feature_C
git merge feature/new_feature_A
git merge feature/new_feature_B
git checkout main
git merge feature/new_feature_C
```

# q
如何合并两个分支并在发生冲突时保留某一特定分支的计算方式？
# a
```bash
git checkout main
git merge feature/new_feature_D
git merge feature/new_feature_E
# 解决冲突后保留 feature/new_feature_E 的奖金计算方式
git add <conflicted_file>
git commit -m "Merge feature/new_feature_E and feature/new_feature_D into main, preserving bonus calculation from D"
```

# q
如何将特性分支上最近的4个提交压缩成一个提交后再合并到 `main`？
# a
```bash
git checkout feature/new_feature_F
git rebase -i HEAD~4
# 在编辑器中，将除第一个提交外的所有提交的 `pick` 改为 `squash` 或 `s`
# 保存后编辑新的提交信息
git checkout main
git merge feature/new_feature_F
```

# q
如何只将某个特定提交（例如消息为 "Added superSecretFeature.c"）应用到 `main` 分支？
# a
```bash
git checkout feature/new_feature_G
git log --grep="Added superSecretFeature.c"
git checkout main
git cherry-pick <查询到的提交哈希>
git push origin main
```
