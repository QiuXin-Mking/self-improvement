# q
在 stress-ng 压力测试中，Docker 相对于 KVM 在哪些性能维度上表现更优？
# a
Docker 在 CPU 计算性能上是 KVM 的 12.4 倍，内存操作性能是 KVM 的 5 倍，进程通信性能是 KVM 的 1.46 倍。

# q
在 systembench 测试中，Docker 相比 KVM 在 CPU 和内存上的具体性能数据差异是多少？
# a
Docker 的 CPU 事件数达 5364.39/s（KVM 为 3997.12/s），平均延迟从 KVM 的 1ms 降至 0.75ms（降低 25%）；内存带宽为 95176.66 MiB/s，较 KVM 的 31874.01 MiB/s 提升近 198.6%。

# q
在哪些测试场景下 KVM 的性能优于 Docker？
# a
stress-ng 测试中，KVM 的存储 I/O 性能是 Docker 的 1.72 倍，进程创建性能是 Docker 的 3.06 倍；systembench 测试中，KVM 的随机读写速率和每秒读写操作数比 Docker 高约 25%。

# q
本文实验使用了哪两个主要压力测试工具，并配套了哪些监控手段？
# a
主要工具为 stress-ng 和 systembench。stress-ng 测试时通过 iostat、vmstat、free 实时监控并记录日志，再用 Python 脚本生成图表；systembench 则测试 CPU 算力、内存搬运速度和磁盘 I/O 吞吐量。
