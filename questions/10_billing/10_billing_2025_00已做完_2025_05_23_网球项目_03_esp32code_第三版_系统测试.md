# q
在这个网球项目测试中，使用 mosquitto_pub 命令向 MQTT 服务器发送控制指令的基本命令格式是什么？
# a
```bash
mosquitto_pub -h 101.201.53.141 -p 1883 -t "wq/v1.0/9167" -m '{"motion":"1", "speed":100, "user_id":"U100", "user_name":"Test"}'
```
其中 `-h` 指定 MQTT 服务器地址，`-p` 指定端口，`-t` 为发布主题，`-m` 为 JSON 格式的消息内容。

# q
在发送的 JSON 控制指令中，`motion` 字段有哪些取值，分别代表什么含义？
# a
`"motion":"1"` 代表正转；`"motion":"2"` 为另一种转动（文档中未明确定义，可能为反转或停止）；`"motion":"3"` 代表转动（从测试看可实现正转）。文档中 `"motion":"1"` 明确标注为“正”，且使用 `"motion":"3"` 也能实现转动。

# q
从测试记录看，`speed` 参数的典型值范围及其对设备运行的影响是什么？
# a
测试中发现 `speed` 设置为 70 时“可以跑得起来”，设置为 80 时“在转动，很慢”，设置为 90 时“快得很”。可见 `speed` 取值范围一般为 70~100，值越高转速越快，80 时速度偏慢。
