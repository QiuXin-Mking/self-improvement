# q
本项目对比的两种虚拟化技术是什么？
# a
KVM + QEMU 虚拟化（基于内核的虚拟机）与 Docker 容器化。KVM+QEMU 提供强隔离性，Docker 提供轻量级应用级隔离与快速启动。

# q
使用哪些工具进行系统压力测试和性能监控？
# a
使用 `stress-ng` 施加 CPU、内存、磁盘 I/O 等多维度负载；使用 `iostat`（sysstat 工具集）实时采集磁盘读写速率、I/O 等待时间和 CPU 统计等指标。典型测试命令如：
```
stress-ng --cpu 4 --io 2 --vm 2 --vm-bytes 4G --timeout 300s --metrics-brief
iostat -xm 1
```

# q
OpenDroneMap 在实验中的主要作用是什么？
# a
作为计算密集型的真实业务负载，模拟边缘计算中无人机影像处理（三维重建、点云生成），用于对比 KVM 虚拟机和 Docker 容器在处理大规模数据集时的资源调度效率、并发能力和系统弹性。

# q
实验硬件与系统环境如何配置？
# a
两台服务器，统一硬件：8 核 CPU、8 GB RAM、SSD 存储，千兆以太网互联（延迟 <1 ms）。操作系统为 Ubuntu 22.04 LTS。服务器 1 部署 KVM+QEMU 虚拟化环境，服务器 2 部署 Docker 引擎与单节点 Kubernetes。
