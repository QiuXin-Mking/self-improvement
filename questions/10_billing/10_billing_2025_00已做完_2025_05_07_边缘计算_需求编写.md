# q
论文对比实验中使用哪些软件工具来构建虚拟化和容器化环境？
# a
- **虚拟化**：KVM + QEMU，操作系统为 Ubuntu 22.04 LTS，支持动态资源分配与快照功能。
- **容器化**：Docker + Kubernetes（单节点集群），部署 OpenDroneMap 模拟复杂应用。
- **压力测试与监控**：Stress-ng（CPU/内存/I/O 负载模拟）、wrk（HTTP 压测）、iostat（磁盘性能监控）。

# q
在静态负载测试中，如何用 Stress-ng 在虚拟机上模拟混合负载？关键参数有哪些？
# a
在单虚拟机（4核4G）上运行命令：
```bash
stress-ng --cpu 4 --io 2 --vm 2 --vm-bytes 4G
```
参数含义：
- `--cpu 4`：启动 4 个 CPU 负载线程；
- `--io 2`：启动 2 个 I/O 负载线程；
- `--vm 2`：启动 2 个内存分配/释放线程；
- `--vm-bytes 4G`：每个内存线程分配 4 GB 虚拟内存。

# q
该实验设计中，虚拟化组与容器化组在 CPU 利用率、内存占用、I/O 吞吐量、启动时间和扩展延迟上的预期差异是什么？
# a
| 指标 | 虚拟化组（VM） | 容器化组（Container） |
|------|----------------|------------------------|
| CPU利用率 | 80%-90%（含 Hypervisor 开销） | 50%-70%（直接调用内核） |
| 内存占用 | 4GB（含虚拟化层） | 2GB（共享内核） |
| I/O吞吐量 | 500 MB/s（模拟延迟） | 800 MB/s（零拷贝优化） |
| 启动时间 | 30秒（冷启动） | 5秒（热启动） |
| 扩展延迟 | 分钟级（需预分配资源） | 秒级（自动调度） |

# q
在测试容器化轻量级任务时，如何执行点云生成任务并记录关键指标？
# a
运行 Docker 单实例容器执行 OpenDroneMap 点云生成任务（数据集 200 MB），使用如下方式记录指标：
- 启动时间：`time docker run ...`
- 资源监控：`docker stats` 获取内存占用峰值
- 完成耗时：统计任务执行总用时
对比时还会通过 `virsh domstats` 获取虚拟机的资源占用曲线。
