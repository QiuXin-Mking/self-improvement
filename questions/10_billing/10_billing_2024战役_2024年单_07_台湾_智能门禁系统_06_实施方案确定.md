# q
在该实施方案中，如何为Python项目安装`requirements.txt`中的依赖，并指定使用清华源进行加速？
# a
使用命令：
```
pip install -r .\requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

# q
如何快速验证yolov5环境是否正确安装并可执行？
# a
执行测试命令：
```
python detect.py --source ./data/images/bus.jpg
```

# q
在Windows上安装`tensorflow-cpu`时遇到`OSError: [Errno 2] No such file or directory`，并提示可能与长路径支持有关，应如何解决？
# a
按照提示启用Windows长路径支持，具体方法可参考文档 `008_win10 长路径支持`。

# q
在树莓派上部署YOLOv5模型时，如果无法成功生成`.pt`文件且TensorFlow安装出现问题，采用了什么替代方案？
# a
放弃生成`.pt`文件，转而将模型导出为ONNX格式，并在树莓派上使用ONNX Runtime进行推理。参考博客：`https://blog.csdn.net/m0_73862068/article/details/134175292`

# q
在树莓派上使用`pip install onnxruntime`时出现`Could not find a version that satisfies the requirement onnxruntime`错误，可能的解决思路是什么？
# a
该问题可能是因为当前系统架构或Python版本不匹配，可尝试切换到32位环境下运行。参考链接：`https://blog.csdn.net/weixin_45233413/article/details/125198496`
