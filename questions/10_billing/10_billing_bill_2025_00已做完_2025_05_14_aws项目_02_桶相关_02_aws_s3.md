# q
如何使用 AWS CLI 创建一个 S3 桶？
# a
使用 `aws s3 mb` 命令，格式为：
```bash
aws s3 mb s3://your-bucket-name
```

# q
如何使用 AWS CLI 删除一个 S3 桶？
# a
使用 `aws s3 rb` 命令，例如：
```bash
aws s3 rb s3://my-test-bucket-20240619
```

# q
如何在 S3 桶中创建一个模拟的文件夹/目录？
# a
用 `put-object` 并指定以 `/` 结尾的键（Key），示例：
```bash
aws s3api put-object --bucket buc-250516 --key thumbnails/
```
成功后返回 ETag 和加密信息。

# q
如何查看当前 AWS CLI 调用者的身份，并判断是否使用临时凭证？
# a
运行 `aws sts get-caller-identity`，输出中若 `Arn` 包含 `assumed-role`，表示当前身份是通过 STS 担任的 IAM 角色（临时安全令牌），例如：
```json
{
    "UserId": "AROAT6F7R6K4IJ64ERMK:user3909028=HAOBO_ZHAO",
    "Account": "270984671928",
    "Arn": "arn:aws:sts::270984671928:assumed-role/voclabs/user3909028=HAOBO_ZHAO"
}
```
`assumed-role` 角色通常权限受平台策略限制，无法执行桶级安全管理操作。

# q
如何为 S3 桶配置对象创建事件通知，触发多个 Lambda 函数？
# a
使用 `aws s3api put-bucket-notification-configuration`，在 `LambdaFunctionConfigurations` 数组中列出多个 Lambda ARN 和监听事件，例如：
```bash
aws s3api put-bucket-notification-configuration \
    --bucket myapp-images-bucket \
    --notification-configuration '{
        "LambdaFunctionConfigurations": [
            {
                "LambdaFunctionArn": "arn:aws:lambda:REGION:ACCOUNT_ID:function:LAMBDA1",
                "Events": ["s3:ObjectCreated:*"]
            },
            {
                "LambdaFunctionArn": "arn:aws:lambda:REGION:ACCOUNT_ID:function:LAMBDA2",
                "Events": ["s3:ObjectCreated:*"]
            }
        ]
    }'
```
需确保 Lambda 已存在并授予 S3 调用权限。
