# q
What are the advantages of TCP over UDP in terms of reliability and data delivery?
# a
TCP provides reliable data transfer with error detection and correction, flow control to prevent overwhelming the receiver, congestion control to adjust transmission rate, ordered delivery of packets, and is connection-oriented (requires an established connection before data transfer). This ensures accurate, in-sequence delivery.

# q
What are the main advantages of using UDP compared to TCP?
# a
UDP offers low latency due to minimal protocol overhead, simplicity and lightweight implementation, native support for broadcasting and multicasting, and no connection establishment or teardown overhead. It is ideal for time-sensitive applications like online gaming, live streaming, and video conferencing where occasional data loss is acceptable.

# q
What are the key differences between TCP and UDP?
# a
- **Reliability**: TCP is reliable with error checking; UDP is unreliable with no delivery guarantees.
- **Connection**: TCP is connection-oriented (uses a handshake); UDP is connectionless.
- **Ordering**: TCP guarantees packet order; UDP does not.
- **Overhead**: TCP has higher overhead due to flow/congestion control; UDP has lower overhead.
- **Use cases**: TCP for reliability-critical applications (web, email); UDP for speed-critical applications (streaming, gaming).

# q
In which layer of the 5-layer network model do TCP and UDP operate?
# a
Both TCP and UDP operate at the Transport Layer (Layer 2 in the 5-layer model: Application, Transport, Network, Data Link, Physical). The Transport Layer provides communication services directly to application processes on different hosts.
