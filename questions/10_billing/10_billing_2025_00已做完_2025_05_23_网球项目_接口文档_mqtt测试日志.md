# q
使用 `mosquitto_pub` 向远程 MQTT broker 发布消息时，如何指定主机地址和端口？
# a
使用 `-h` 指定主机（如 `-h 101.201.53.141`），使用 `-p` 指定端口（如 `-p 1883`）。示例：
```bash
mosquitto_pub -h 101.201.53.141 -p 1883 -t "wq/v1.0/9167" -m '{"motion":"050", "speed":50, "user_id":"U100", "user_name":"Test"}'
```

# q
测试中向 MQTT 主题 `wq/v1.0/9167` 发布的 JSON 消息包含哪些字段？
# a
包含四个字段：`motion`（值 `"050"`）、`speed`（值 `50`）、`user_id`（值 `"U100"`）、`user_name`（值 `"Test"`）。完整消息体为：
```json
{"motion":"050", "speed":50, "user_id":"U100", "user_name":"Test"}
```

# q
在 MQTT 测试中，为什么出现两个分别发送到远程 IP 和 localhost 的相同消息？
# a
用于验证 MQTT broker 在不同网络环境下的连通性：一条直接发布到远程服务器 `101.201.53.141`，另一条发布到本地 broker（`localhost`），可能用于检查本地服务是否正常转发或消费消息。
