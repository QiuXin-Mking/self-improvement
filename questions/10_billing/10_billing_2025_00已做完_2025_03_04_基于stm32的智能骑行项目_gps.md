# q
NMEA 0183 协议中，一个完整的语句的基本格式是怎样的？其中的校验和如何计算？
# a
基本格式：`$<talkerID><sentenceID>,<字段1>,<字段2>,...,<字段n>*<checksum><CR><LF>`
- 以 `$` 开头，`*` 后的 `<checksum>` 是校验和，为两个十六进制字符，计算方式是 `$` 和 `*` 之间所有字符的 XOR（异或）值。
例如：`$GNRMC,081322.000,V,,,,,,,220325,,,M*5E` 中的 `5E` 就是校验和。

# q
从日志中看，`$GNRMC` 语句中定位状态 `V` 和 `$GNGGA` 中 GPS 质量指标 `0` 分别表示什么含义？
# a
- `$GNRMC` 的第三个字段（定位状态）：`V` 表示 **无效/没有定位**（V = Void），`A` 表示有效定位。
- `$GNGGA` 的第七个字段（GPS 质量指示）：`0` 表示 **未定位/无效**，`1` 表示标准 GPS 定位，`2` 表示差分定位，`6` 表示正在估算（推算定位）等。

# q
在 `$GNGGA` 语句中，HDOP（水平精度因子）为 `25.5` 时代表什么？正常有效的定位 HDOP 值通常应在什么范围？
# a
HDOP 为 `25.5` 通常表示 **精度极差或没有有效定位**（很多模块在无定位时输出最大值 25.5）。正常有效定位的 HDOP 值通常小于 2.0，良好的定位可以小于 1.0，值越小精度越好。

# q
GPS 卫星可视信息语句 `$GPGSV` 和 `$BDGSV` 中，每句最多可描述几颗卫星？其语句末尾出现类似 `*7F` 的作用是什么？
# a
- 每个 `$GPGSV` 或 `$BDGSV` 语句最多描述 **4 颗卫星**（每个卫星信息占 4 个字段：PRN号、仰角、方位角、信噪比）。
- 语句末尾的 `*7F` 是 **校验和**，用于验证数据完整性。
