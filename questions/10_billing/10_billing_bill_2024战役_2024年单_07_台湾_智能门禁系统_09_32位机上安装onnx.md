# q
在树莓派上如何查看 CPU 架构和系统位数？
# a
- 查看架构：`uname -a`，例如输出中的 `armv7l` 表示 32 位 ARM 架构。
- 查看系统位数：`getconf LONG_BIT`，输出 `32` 表示 32 位系统。

# q
如何根据树莓派的 `/etc/debian_version` 确定 Debian 版本代号？
# a
```
cat /etc/debian_version
```
根据文件内容对应关系：

| 值   | 代号      |
|------|-----------|
| 11.x | Bullseye  |
| 10.x | Buster    |
| 9.x  | Stretch   |

示例中 `10.11` 对应 Buster。

# q
在 32 位树莓派（armv7l, Python 3.7, Buster）上安装 ONNX Runtime 的完整步骤是什么？
# a
1. 克隆预编译的 ONNX Runtime 仓库：
```bash
git clone https://gitcode.com/nknytk/built-onnxruntime-for-raspberrypi-linux.git
```
2. 进入对应 Debian 版本（如 Buster）的 wheels 目录：
```bash
cd built-onnxruntime-for-raspberrypi-linux/wheels/buster
```
3. 使用 pip 安装匹配 Python 版本的 `.whl` 文件（示例 Python 3.7）：
```bash
pip3 install onnxruntime-1.7.2-cp37-cp37m-linux_armv7l.whl
```
