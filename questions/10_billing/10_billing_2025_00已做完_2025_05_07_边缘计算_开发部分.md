# q
在对比虚拟机和容器的静态负载测试中，使用了哪个 stress-ng 命令来模拟 CPU、I/O 和内存压力？
# a
```bash
stress-ng --cpu 4 --io 2 --vm 2 --vm-bytes 4G --metrics-brief
```

# q
该实验中，分别采用哪些平台来搭建虚拟化组和容器化组？
# a
虚拟化组：在第一台服务器上安装 KVM/QEMU，并配置 libvirt 管理工具；容器化组：在第二台服务器上安装 Docker 引擎，并搭建单节点 Kubernetes 集群（使用 kubeadm 初始化）。

# q
实验开发步骤中列举了哪些用于性能数据采集的工具和命令？
# a
- stress-ng --metrics-brief：收集 CPU、内存、I/O 的性能数据
- docker stats、kubectl top：容器化组的资源监控
- virsh domstats：虚拟化组的资源监控
- iostat、htop、iftop：基础性能分析工具（磁盘 I/O、进程监控、网络流量）
