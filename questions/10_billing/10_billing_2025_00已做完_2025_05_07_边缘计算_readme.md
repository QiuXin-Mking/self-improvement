# q
该项目中用到了哪些工具来对边缘计算集群做性能测试与监控？
# a
- 测试工具：iperf3（网络），sysbench（CPU/内存/磁盘 I/O），stress-ng（施加可重复负载）
- 监控工具：Prometheus + Grafana
- 系统监视：top
- 功耗记录：UM24C 功率计

# q
容器化与虚拟机在边缘计算环境中的主要区别是什么？
# a
- 容器化：与宿主操作系统共享内核，轻量、灵活，部署快，资源利用率高，但隔离性较弱。
- 虚拟机：通过 hypervisor 抽象硬件，每台 VM 运行完整 OS，隔离性强，可精确满足底层需求，但引入较大开销。
- 结论：容器化在能源消耗上更优，更适合资源受限的边缘环境；虚拟机在需要完全隔离、特定 OS 配置时更适用。

# q
论文中 Raspberry Pi 平台上的 stress-ng 测试结果，Kubernetes 和 Docker Compose 分别有什么表现？
# a
- CPU 性能：长时间运行时 Kubernetes 表现更好，短时间测试 Docker Compose 与 Podman 更优。
- I/O 磁盘性能：1 分钟测试 Docker Compose 更好；10 分钟测试 Kubernetes 有显著提升。
- 内存性能：差异不显著，Kubernetes 因自身复杂性消耗更多资源。
