# q
Git中暂存区的作用是什么？如何将修改放入暂存区？
# a
暂存区是工作区和版本库之间的“中间层”，用于临时存放想要记录的修改。使用 `git add 文件名` 添加单个文件，或 `git add .` 添加当前目录下所有修改的文件。

# q
`git status` 命令如何帮助区分文件的三种常见状态？
# a
- 显示 `Untracked files:` 说明文件是新创建的，Git 尚未跟踪。
- 显示 `Changes not staged for commit:` 说明已跟踪的文件被修改但未送入暂存区。
- 显示 `nothing to commit, working tree clean` 说明所有修改都已提交或暂存区为空。

# q
如何创建一个提交并查看提交历史？
# a
先将修改加入暂存区：`git add .`，然后执行 `git commit -m "备注信息"` 创建提交。查看历史用 `git log` 显示详细信息，`git log --oneline` 则只显示提交ID和备注（如 `a1b2c3d 初始化项目`）。
