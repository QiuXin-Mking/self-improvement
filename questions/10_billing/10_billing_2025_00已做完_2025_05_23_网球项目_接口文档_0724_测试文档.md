# q
如何使用 mosquitto_pub 向网球项目设备的 MQTT 主题发送控制指令？
# a
命令格式：
```
mosquitto_pub -h 101.201.53.141 -p 1883 -t "wq/v1.0/9167" -m '{"motion":"1", "speed":100, "user_id":"U100", "user_name":"Test"}'
```
其中 `-h` 指定 Broker 地址，`-p` 指定端口 1883，`-t` 为主题，`-m` 为 JSON 消息，包含 motion、speed、user_id 和 user_name 字段。

# q
主题 wq/v1.0/9167 与 wq/v1.0/9167/x 在使用上有何不同？
# a
主题 `wq/v1.0/9167` 用于常规运动控制；`wq/v1.0/9167/x` 可能是附加子路径，用于发送特定指令（如停止），示例：
```
mosquitto_pub -h ... -t "wq/v1.0/9167/x" -m '{"motion":"1", "speed":0, ...}'
```
（speed=0 可实现停止）

# q
消息中 motion 字段的常见取值及其对应含义是什么？
# a
文档中出现的取值包括：
- `"1"`：注释为“正”（正转）
- `"2"`：无明确注释，可能代表另一种运动模式
- `"3"`：无明确注释，可能与反向或不同模式相关
具体含义需结合设备实际行为确认。

# q
如何调整设备的运动速度？
# a
通过修改 JSON 消息中的 `speed` 字段值，例如 `"speed":70`（“70 可以跑得起来”）、`"speed":80`（“在转动，很慢”）、`"speed":90`（“90 快得很”）、`"speed":100` 等，数值越大速度越快。
