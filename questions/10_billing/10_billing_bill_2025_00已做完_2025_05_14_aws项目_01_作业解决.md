# q
部署一个包含EC2、S3、RDS、Lambda、ALB和ASG的云原生图像标注应用的核心步骤顺序是什么？
# a
顺序：先建S3和RDS → 配置EC2并开发Web上传/查询逻辑 → 搭建ASG和ALB以保证可扩展性 → 编写两个Lambda（AI标注与缩略图生成） → 配置S3事件通知触发Lambda → 联调权限和安全组，确保资源互通。

# q
在AWS上，如何配置安全组以保证Web层、数据库层和无服务器层之间的安全通信？
# a
- ALB安全组：入站允许80/443端口，来源为所需公网IP或全网。
- EC2安全组：入站仅允许来自ALB安全组的流量；出站允许连接到RDS（3306）和必要外部服务。
- RDS安全组：入站允许来自EC2安全组和Lambda安全组的3306端口访问。
- Lambda若需访问RDS，应部署在RDS所在VPC的子网中，并使用允许3306出站的安全组。

# q
Lambda函数如何通过S3上传事件自动触发？
# a
在S3控制台的“属性”>“事件通知”中，针对`images/`前缀的 `s3:ObjectCreated:*` 事件，将目标设置为对应的Lambda函数（标注Lambda和缩略图Lambda各创建一个通知）。上传图片到桶时会自动触发这两个函数。

# q
EC2和Lambda分别需要怎样的IAM角色权限才能操作S3和RDS？
# a
- EC2实例应挂载IAM角色，附加策略如 `AmazonS3FullAccess` 和 `AmazonRDSFullAccess`（生产环境应遵循最小权限原则）。
- 标注Lambda的执行角色需：S3读权限（获取图片）、RDS写权限（写入标注结果）。
- 缩略图Lambda的执行角色需：S3读权限（下载原图）和S3写权限（上传缩略图到`thumbnails/`前缀）。

# q
整个架构中，S3桶、RDS数据库以及Lambda函数的数量要求分别是什么？
# a
- S3桶：1个，用于存储原图和缩略图（建议在桶内创建`thumbnails/`前缀存放缩略图）。
- RDS：1个MySQL实例，存储图片元数据和AI标注。
- Lambda函数：2个，一个负责调用AI生成标注并写入RDS，另一个负责生成缩略图并回传S3。
