# q
如何创建一个新分支 `feature/new_feature_C`，并将 `feature/new_feature_A` 和 `feature/new_feature_B` 合并到该分支中？
# a
先从 `main` 创建并切换到新分支 `feature/new_feature_C`，然后依次合并两个分支：
```bash
git checkout main
git checkout -b feature/new_feature_C
git merge feature/new_feature_A
git merge feature/new_feature_B
```
最后可以将 `feature/new_feature_C` 合并回 `main`：
```bash
git checkout main
git merge feature/new_feature_C
```

# q
在合并 `feature/new_feature_D` 和 `feature/new_feature_E` 到 `main` 时，如何解决冲突并确保保留 `feature/new_feature_E` 的奖金计算方式？
# a
在 `main` 上先合并 `feature/new_feature_D`，再合并 `feature/new_feature_E`，出现冲突后手动编辑冲突文件，选择保留 `feature/new_feature_E` 的修改，然后标记已解决并完成合并：
```bash
git checkout main
git merge feature/new_feature_D
git merge feature/new_feature_E
# 手动解决冲突，保留 feature/new_feature_E 的版本
git add <conflicted_file>
git commit -m "Merge feature/new_feature_E and feature/new_feature_D into main, preserving bonus calculation from D"
```

# q
如何将 `feature/new_feature_F` 分支上最近的 4 个提交压缩成一个提交，再合并到 `main`？
# a
切换到该分支，使用交互式变基压缩最近 4 个提交：
```bash
git checkout feature/new_feature_F
git rebase -i HEAD~4
```
在编辑器中，将除第一个外的 `pick` 改为 `squash`（或 `s`），保存退出后在提交消息编辑器中输入新的提交信息，然后合并到 `main`：
```bash
git checkout main
git merge feature/new_feature_F
```

# q
如何将 `feature/new_feature_G` 分支上提交消息为 "Added superSecretFeature.c" 的特定提交 cherry-pick 到 `main`？
# a
先查找该提交的 hash，然后切换到 `main` 执行 cherry-pick：
```bash
git checkout feature/new_feature_G
git log --grep="Added superSecretFeature.c"
git checkout main
git cherry-pick <查询到的hash>
git push origin main
```
