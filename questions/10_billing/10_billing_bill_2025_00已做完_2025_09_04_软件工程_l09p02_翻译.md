# q
如何使用 `git cherry-pick` 仅将另一个分支上的单个特定提交合并到当前分支？
# a
使用 `git cherry-pick <commit-hash>` 命令，其中 `<commit-hash>` 是要引入的提交的哈希值。例如，若需将消息为 "Added superSecretFeature.c" 的提交合并到主分支，可先切换到主分支，然后执行 `git cherry-pick <该提交的哈希>`，即可仅将该提交的更改应用到主分支。

# q
如何利用 GCC Address Sanitizer 检测 C 程序中的地址安全漏洞？
# a
在编译时添加 `-fsanitize=address` 选项即可启用 Address Sanitizer。例如：
```bash
gcc -fsanitize=address -o employee-manager executeCommand.c
```
或在 Makefile 的编译规则中加入该选项。运行程序时，Address Sanitizer 会自动检测内存错误（如堆缓冲区溢出、栈缓冲区溢出、释放后使用等），并输出详细的错误报告。

# q
如何编写一个基于 GCC 镜像的 Dockerfile 来构建 C 应用程序？
# a
根据规范，Dockerfile 应包含以下关键指令：
- `FROM gcc:latest`：使用 GCC 最新版作为基础镜像。
- `WORKDIR /usr/src/employee-manager`：设置工作目录。
- `COPY . .`：将当前上下文中的应用目录复制到工作目录。
- `RUN make`：执行 Makefile 构建 `employee-manager` 可执行文件。
- `CMD ["./employee-manager"]`：容器启动时运行该程序。

# q
在 Docker Compose 文件中如何定义一个 PostgreSQL 服务并配置环境变量、端口及数据卷？
# a
在 `services` 下定义服务，例如 `employee-manager-postgres`，需包含：
- `image: postgres:latest`
- `container_name: employee-manager-postgres`
- `environment`: 设置 `POSTGRES_DB=employee-manager`、`POSTGRES_USER=admin`、`POSTGRES_PASSWORD=pass`
- `ports: "5432:5432"`
- `volumes: employee-manager-data:/var/lib/postgresql/data`
并确保数据卷 `employee-manager-data` 在顶层 `volumes` 中声明。
