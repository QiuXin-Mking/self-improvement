# q
使用 AWS CLI 创建面向互联网的 Application Load Balancer 的基本命令是什么？
# a
```bash
aws elbv2 create-load-balancer \
  --name webapp-alb \
  --subnets subnet-aaa subnet-bbb \
  --security-groups sg-11112222 \
  --scheme internet-facing \
  --type application \
  --ip-address-type ipv4
```
该命令创建名为 webapp-alb 的 ALB，绑定到指定子网和安全组，采用 internet-facing 模式，类型为 application。

# q
应用 EC2 实例的安全组需要开放哪些端口，分别允许哪些来源访问？
# a
- 80/443 端口：仅允许来自 Application Load Balancer 的流量。
- 22 端口：仅允许来自堡垒机的流量。
（堡垒机自身的安全组只开放指定 IP 的 22 端口。）

# q
EC2 启动脚本（user-data）中如何安装依赖并以后台方式启动 Flask 应用？
# a
启动脚本执行以下关键步骤：
```bash
yum install -y python3-pip mariadb105 unzip awscli
cd /home/ec2-user
aws s3 cp s3://buc-250516/Image-caption-app-V1.1-new.zip .
unzip -o Image-caption-app-V1.1-new.zip -d ./images
pip3 install flask mysql-connector-python -q -U google-generativeai boto3 werkzeug
nohup python3 /home/ec2-user/images/Image-caption-app-V1.1/app.py >app.log 2>&1 &
```
先用 yum 安装系统工具，从 S3 下载应用代码，解压后用 pip 安装依赖，最后通过 nohup 后台启动 Flask 应用。

# q
AWS CLI 如何创建用于 Auto Scaling Group 的目标组，并配置健康检查路径？
# a
```bash
aws elbv2 create-target-group \
  --name webapp-tg \
  --protocol HTTP \
  --port 5000 \
  --vpc-id vpc-002fe9064c70ed42a \
  --target-type instance \
  --health-check-protocol HTTP \
  --health-check-path /
```
此命令创建名为 webapp-tg 的目标组，监听 HTTP 5000 端口，目标类型为 instance，健康检查路径为 `/`。
