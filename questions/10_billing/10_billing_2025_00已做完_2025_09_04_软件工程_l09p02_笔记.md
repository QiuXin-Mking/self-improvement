# q
在交互式 rebase 过程中，修改当前提交后，应使用什么命令将更改合并到当前提交并继续 rebase？
# a
修改后可以使用 `git commit --amend` 修改当前提交，然后执行 `git rebase --continue` 继续 rebase 过程。也可以直接使用 `git commit -m "message"` 创建一个新的提交，之后同样需要 `git rebase --continue`。

# q
`git reset 6d7a018c --hard` 命令执行后会产生什么效果？
# a
会将 HEAD 指针重置到指定的提交 `6d7a018c`，同时工作目录和暂存区也会被重置为该提交的状态，之后的所有提交会被丢弃。在示例中，执行后 `git log` 只显示到该提交，后来的 "modify setup" 提交不再出现。

# q
在 rebase 过程中执行 `git status`，提示 "Next commands to do (4 remaining commands)" 表示什么？
# a
表示在交互式 rebase 中，还有 4 个待执行的提交操作。可以使用 `git rebase --edit-todo` 查看和编辑这些待办命令列表。
