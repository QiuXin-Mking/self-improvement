# q
如何向网球项目的 /tennis_action 接口发送 HTTP POST 请求？
# a
使用 curl 命令：
```
curl -v -X POST "http://<IP>:80/tennis_action" \
  -H "Content-Type: application/json" \
  -d '{"motion":<motion>, "speed":<speed>, "user_id":"<user_id>", "user_name":"<user_name>"}'
```
内部测试使用 IP `127.0.0.1`，外部测试使用 IP `101.201.53.141`。

# q
motion 参数的有效范围是多少？超出范围会怎样？
# a
motion 不能超过 3。如果传入大于 3 的值（如 8），请求会报错，且对应的 MQTT topic 上不会发布消息。

# q
speed 参数的有效范围是多少？超出范围会怎样？
# a
speed 不能超过 100。如果传入大于 100 的值（如 850），请求会报错，且对应的 MQTT topic 上不会产生消息。

# q
如何通过 MQTT 订阅某个用户的网球动作消息？
# a
使用 mosquitto_sub 命令订阅 topic `wq/v1.0/<user_id>`，例如：
```
mosquitto_sub -h localhost -t wq/v1.0/u123
```
