# q
ESP32C3 如何通过 BLE 实现文件传输到连接的 SD 卡？
# a
ESP32C3 作为 BLE 外设，将接收到的文件数据写入 SD 卡。传输过程需要封包（分包），将文件分成多个数据包通过 BLE 发送。接收端收到包后按队列重组，并写入文件系统。APP 端负责发起传输和管理文件。

# q
在 BLE 文件传输中，为什么要对文件进行封包？
# a
BLE 单次传输的数据包大小有限（MTU 通常 23~512 字节），无法一次发送整个文件。封包（分包）可将大文件拆分为多个小数据包依次传输，接收端按顺序重组，保证文件完整性。

# q
APP 如何实现对 TF 卡上文件的管理（增删查改）？
# a
APP 通过 BLE 与设备通信，发送文件管理指令。设备响应该指令，在 TF 卡的文件系统上执行相应操作：获取文件列表（查）、删除指定文件（删）、重命名或修改（改）、写入新文件（增）。所有操作通过自定义 BLE 服务和特征实现。

# q
音频播放的控制方式是什么？
# a
播放控制由设备上的物理按键完成，而非手机 APP 控制。手机只负责文件传输和管理。
