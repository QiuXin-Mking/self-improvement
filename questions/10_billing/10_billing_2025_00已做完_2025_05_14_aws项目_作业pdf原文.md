# q
What are the two main components that the AWS project assignment requires to deploy?
# a
1. **Web Application Component**: Hosted on EC2 instances in an Auto Scaling Group (ASG, max capacity >1) with an Application Load Balancer (ALB) distributing traffic. It must support an HTML upload form storing images in S3 and metadata in RDS (MySQL), as well as a page displaying uploaded images/thumbnails/captions.
2. **Serverless Component (AWS Lambda)**: Two Lambda functions triggered by S3 `ObjectCreated` events:
   - **Annotation Function**: Retrieves image from S3, invokes the Gemini API for a description, and stores the result in RDS.
   - **Thumbnail Generator**: Retrieves image from S3, generates a thumbnail, and stores it in the `thumbnails/` folder of the same S3 bucket.

# q
How are the Lambda functions triggered in the serverless architecture?
# a
Both Lambda functions are triggered directly by S3 `ObjectCreated` events (no SNS/EventBridge unless specified). When a new image is uploaded, the event triggers the Annotation Function and the Thumbnail Generator.

# q
What evidence is required for the Auto Scaling test in the assignment?
# a
- EC2 scaling actions (instance count changes) 
- CPU/memory metrics trends
- ALB request distribution metrics
Tools like Apache Bench or Python concurrent request scripts can be used to generate load.

- Lambda execution: CloudWatch logs, S3 updates, RDS entries.
