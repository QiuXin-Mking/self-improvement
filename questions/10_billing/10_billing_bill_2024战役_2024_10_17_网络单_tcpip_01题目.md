# q
What are the advantages of using TCP?
# a
TCP advantages:
1. Reliable Data Transfer: error detection and correction, accurate and in-order delivery.
2. Flow Control: prevents overwhelming the receiver.
3. Congestion Control: adjusts transmission rate based on network congestion.
4. Connection-Oriented: connection established before transmission.
5. Ordered Delivery: packets delivered in sequence.

# q
What are the advantages of using UDP?
# a
UDP advantages:
1. Low Latency: minimal overhead, faster transmission for time-sensitive applications (e.g., online gaming, live streaming).
2. Simple and Lightweight: easy implementation without connection management overhead.
3. Broadcast and Multicast Support: useful for video conferencing.
4. No Connection Overhead: connectionless, no connection setup/teardown.

# q
What are the key differences between TCP and UDP?
# a
1. Reliability: TCP is reliable with error checking; UDP is unreliable, no delivery guarantee.
2. Connection: TCP is connection-oriented (handshake); UDP is connectionless.
3. Ordering: TCP guarantees packet order; UDP does not.
4. Overhead: TCP has higher overhead (error checking, flow/congestion control); UDP has lower overhead, faster.
5. Use Cases: TCP for reliable applications (web browsing, email); UDP for speed-critical applications tolerating loss (streaming, gaming).

# q
Which layer of the 5-layer model do TCP and UDP belong to?
# a
Both TCP and UDP operate at the Transport Layer (Layer 2 in the 5-layer model: Application, Transport, Network, Data Link, Physical).
