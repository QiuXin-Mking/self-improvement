# q
如何将本地Docker镜像导出为tar文件，以及如何从tar文件导入镜像？
# a
导出镜像：使用 `docker save -o <输出文件名.tar> <镜像名:标签>`
示例：
```bash
docker save -o ubuntu-latest.tar ubuntu:latest
```
导入镜像：使用 `docker load -i <tar文件路径>`
示例：
```bash
docker load -i ubuntu-latest.tar
```
注意：`docker import` 也可从tar创建镜像，但会丢失原有镜像的历史和标签，一般用于容器快照。

# q
如何启动一个持续运行的容器用于调试或测试？
# a
可以使用 `tail -f /dev/null` 作为前台进程保持容器运行：
```bash
docker run -d ubuntu:latest tail -f /dev/null
```
或同时安装必要工具：
```bash
docker run -d ubuntu:latest sh -c "apt update && apt install -y coreutils && tail -f /dev/null"
```
`-d` 表示后台运行。

# q
如何将运行中的容器修改保存为新镜像并导出？
# a
1. 先对需要修改的容器进行软件安装等操作，例如：
```bash
docker exec -it <容器ID> bash
apt install vim stress-ng -y
```
2. 提交容器为新的镜像：
```bash
docker commit <容器ID> ubuntu:1.0
```
3. 导出新镜像为tar文件：
```bash
docker save -o ubuntu-1.0.tar ubuntu:1.0
```
