# q
如何将 feature/new_feature_F 分支上的所有提交压缩为一个新提交并合并到主分支，同时使用指定的多行提交消息？
# a
切换至主分支后执行压缩合并：
```bash
git checkout main
git merge --squash feature/new_feature_F
git commit -m "添加 loadEntryFile.c

添加 loadEntryFile.h

修复 loadEntryFile.c 中的拼写错误

修复 loadEntryFile.c 中的拼写错误（真的）"
```
这样主分支上只会出现一个组合提交，不会包含该分支的单独提交。

# q
如何从 feature/new_feature_G 分支中只提取消息为 "Added superSecretFeature.c" 的提交合并到主分支？
# a
使用 `git cherry-pick` 命令。首先查看目标分支的提交历史找到该提交的哈希值（例如使用 `git log feature/new_feature_G`），然后在主分支上运行：
```bash
git checkout main
git cherry-pick <commit-hash>
```
这会将所选提交精确应用到主分支。

# q
如何使用 GCC 的 AddressSanitizer 检测 executeCommand.c 中的内存安全漏洞？
# a
在编译时添加 `-fsanitize=address` 标志。如果使用 make 构建，可通过修改环境变量或 Makefile 来实现，例如：
```bash
make CFLAGS="-fsanitize=address -g"
```
或者在 Makefile 中的编译命令中加入该选项。编译后运行程序，AddressSanitizer 会在运行时检测并报告内存越界、释放后使用等错误。

# q
如何根据规范使用 Docker Compose 定义 employee-manager-postgres 和 employee-manager-app 两个服务？
# a
完整的 `docker-compose.yml` 配置：
```yaml
services:
  employee-manager-postgres:
    image: postgres:latest
    container_name: employee-manager-postgres
    volumes:
      - employee-manager-data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=employee-manager
      - POSTGRES_USER=admin
      - POSTGRES_PASSWORD=pass
    ports:
      - "5432:5432"

  employee-manager-app:
    build: .
    container_name: employee-manager-app
    environment:
      - DB_SERVER=employee-manager-postgres
    ports:
      - "8080:8080"

volumes:
  employee-manager-data:
```
