# q
在 gRPC proto 文件中如何定义一个服务端流式 RPC 方法？
# a
在 `service` 块中使用 `rpc` 声明，返回类型前加 `stream` 关键字。例如：
```proto
service WeatherReporter {
  rpc GetCityWeatherMultipleDays(LocationDatePeriod) returns (stream CityWeatherData);
}
```

# q
gRPC 服务端实现服务端流式 RPC 方法时，如何向客户端多次发送数据？
# a
在覆写的方法中，通过 `responseObserver.onNext(data)` 多次发送响应，全部发送完成后调用 `responseObserver.onCompleted()` 结束流。示例：
```java
@Override
public void getCityWeatherMultipleDays(LocationDatePeriod req, StreamObserver<CityWeatherData> responseObserver) {
    for (LocalDate date = req.getStartDate(); !date.isAfter(req.getEndDate()); date = date.plusDays(1)) {
        CityWeatherData data = generateDailyData(date);
        responseObserver.onNext(data);
    }
    responseObserver.onCompleted();
}
```

# q
gRPC 客户端如何使用 BlockingStub 消费服务端流式 RPC 返回的多个消息？
# a
调用流式 RPC 方法会返回一个 `Iterator` 对象，可使用 `forEachRemaining` 遍历所有响应。示例：
```java
blockingStub.getCityWeatherMultipleDays(request)
    .forEachRemaining(data -> System.out.println(data));
```

# q
在 gRPC 服务端启动时，如何注册服务实现并指定监听端口？
# a
使用 `ServerBuilder.forPort(port).addService(new ServiceImpl()).build().start()`。`ServiceImpl` 需要继承自动生成的 `XxxGrpc.XxxImplBase` 并覆写 RPC 方法。
