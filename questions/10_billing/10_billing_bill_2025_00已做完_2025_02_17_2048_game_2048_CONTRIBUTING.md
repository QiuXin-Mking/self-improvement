# q
贡献时，应该基于哪个分支创建新分支？
# a
应基于 `master` 分支创建新分支，不要从 `gh-pages` 分支创建（除非严格必要）。`master` 是游戏的“纯净”版本，`gh-pages` 包含分享、分析等与游戏核心无关的功能。

# q
如何编译项目的 SCSS 文件？
# a
需要先安装 Ruby 和 `sass` gem（`gem install sass`），然后执行命令：
```
sass --unix-newlines --watch style/main.scss
```
SASS 会自动监测并重新编译 CSS。

# q
编辑 JavaScript 时有哪些代码风格要求？
# a
使用 2 空格缩进。项目中有 `.jshintrc` 文件，安装并运行 `jshint` 可帮助代码遵循规范。

# q
哪些修改可能不会被合并到项目中？
# a
以下修改可能会被拒绝或需要仔细评估：
- 撤销/重做功能
- 保存/加载功能
- 格子外观或内容的更改
- 布局更改
- 网格大小更改

# q
哪些类型的贡献是受欢迎的？
# a
欢迎的修改包括：
- Bug 修复
- 兼容性改进
- “底层”优化
- 不影响核心玩法的小改动
