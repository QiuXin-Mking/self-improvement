# q
Git的核心工作流程涉及哪三个区域？它们是如何协作的？
# a
三个区域是工作区（Working Directory）、暂存区（Staging Area/Index）和版本库（Repository）。工作区是直接编辑文件的文件夹；暂存区是中间层，通过 `git add` 将修改从工作区选择性地放入暂存区；版本库通过 `git commit` 将暂存区的修改打包成带备注的提交（commit）永久保存。所有提交集合构成版本库，存储在隐藏的 `.git` 目录中。

# q
如何在一个新文件夹中初始化 Git 仓库？
# a
使用 `cd` 进入目标文件夹后执行：
```bash
git init
```
该命令会在文件夹中创建隐藏的 `.git` 目录，使其成为受 Git 管理的仓库。

# q
将修改保存为一次提交的标准三步流程是什么？给出具体命令和参数。
# a
1. 查看状态：`git status`
2. 添加所有修改到暂存区：`git add .`（或指定单个文件 `git add 文件名`）
3. 提交并附带备注：`git commit -m "备注信息"`
示例：
```bash
git status
git add .
git commit -m "初始化项目，添加基础页面"
```

# q
如何用最简洁的方式查看提交历史？请给出命令及输出示例含义。
# a
使用 `git log --oneline` 可以显示简略的提交历史，每条记录包含提交ID和备注。例如：
```
a1b2c3d (HEAD -> main) 初始化项目，添加基础页面
e4f5g6h 添加用户登录模块
```
这表示当前分支有两次提交，最近的提交是“初始化项目”，其ID为 a1b2c3d，HEAD 指向它。

# q
如果误将文件添加到暂存区，如何将其移出暂存区但保留工作区的修改内容？
# a
使用命令：
```bash
git reset HEAD 文件名
```
该操作将指定文件从暂存区移除，但工作区的修改内容依然保留，不会被删除。
