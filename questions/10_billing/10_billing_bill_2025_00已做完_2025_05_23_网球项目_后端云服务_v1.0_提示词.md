# q
HTTP接口接收的JSON数据包含哪些字段？其含义和取值约束是什么？
# a
JSON结构为 `{"motion":"001","speed":"002","user_id":"9167","user_name":"qx"}`。其中：
- `motion`：动作编码，0 无效状态，1 还原动作，2 捡球动作，3 急停动作，必须校验在 0-3 区间内。
- `speed`：速度值，必须校验在 0-100 区间内。
- `user_id`、`user_name`：用户标识与名称。

# q
处理完HTTP请求后，消息如何转发到MQTT？主题格式是什么？
# a
将原始的JSON消息 `{"motion":"001","speed":"002","user_id":"9167","user_name":"qx"}` 完整转发到MQTT，主题格式为 `wq/v1.0/{user_id}`，其中 `{user_id}` 替换为实际用户ID。

# q
接收到的消息需要如何记录到数据库？数据库连接配置从哪里读取？
# a
记录格式为：`timestamp, motion, speed, user_id, user_name`。数据库连接通过配置文件 `tennis.conf` 中的数据库文件路径指定，默认路径为 `/root/.mycnf`，系统应通过该文件链接数据库。

# q
如何将数据库中的 `timestamp` 字段从整数时间戳类型更改为 `DATETIME` 类型？
# a
使用以下SQL步骤：
```sql
-- 1. 新增临时DATETIME字段
ALTER TABLE tennis_log ADD COLUMN timestamp_dt DATETIME;
-- 2. 用原时间戳填充新字段
UPDATE tennis_log SET timestamp_dt = FROM_UNIXTIME(`timestamp`);
-- 3. 删除原timestamp字段
ALTER TABLE tennis_log DROP COLUMN `timestamp`;
-- 4. 重命名新字段为timestamp
ALTER TABLE tennis_log CHANGE timestamp_dt `timestamp` DATETIME NOT NULL;
```
