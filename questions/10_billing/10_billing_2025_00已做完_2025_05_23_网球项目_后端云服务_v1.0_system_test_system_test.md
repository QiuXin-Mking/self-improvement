# q
如何启动或重启网球助手云服务，并检查其运行状态？
# a
使用以下 systemctl 命令：
- 重启服务：`systemctl restart tennis-assistant.service`
- 查看状态：`systemctl status tennis-assistant.service`

# q
云服务的 HTTP 接口路径及请求格式是什么？
# a
接口路径：`POST /tennis_action`
请求头：`Content-Type: application/json`
请求体示例：
```json
{"motion":2, "speed":85, "user_id":"u123", "user_name":"testuser"}
```

# q
如何通过 MQTT 订阅某个用户从 HTTP 接口接收到的动作消息？
# a
使用如下命令，其中 `{user_id}` 替换为实际用户ID：
```
mosquitto_sub -h localhost -t wq/v1.0/{user_id}
```
例如监听用户 `u123`：`mosquitto_sub -h localhost -t wq/v1.0/u123`

# q
根据测试案例，`speed` 和 `motion` 参数的校验规则是什么？校验失败时系统行为如何？
# a
- `speed` 不能超过 100（如发送 850 会报错）
- `motion` 值不能超过 3（如发送 8 会报错）
- 校验失败时接口返回错误，并且对应的 MQTT topic 上不会收到消息
