# q
在CentOS/RHEL上安装KVM虚拟化环境需要安装哪些核心软件包？
# a
核心软件包包括：
- `qemu-kvm`：KVM内核模块和QEMU虚拟机引擎
- `libvirt`：虚拟化通用管理服务（后端程序）
- `libvirt-client`：提供`virsh`和本地管理命令行工具
- `virt-install`：命令行建虚拟机工具
- `bridge-utils`：网络桥接支持（如需桥接网络让虚拟机互通）

安装命令：
```bash
sudo yum install -y qemu-kvm libvirt libvirt-client virt-install bridge-utils
```

# q
如何启动并设置libvirtd服务开机自启？
# a
使用以下命令：
```bash
sudo systemctl enable --now libvirtd
```
该命令会同时启用并启动服务。之后可以用`sudo systemctl status libvirtd`查看状态。

# q
如何验证KVM内核模块是否已正确加载？
# a
运行以下命令检查：
```bash
lsmod | grep kvm
```
若输出包含`kvm_intel`（Intel CPU）或`kvm_amd`（AMD CPU），则表示KVM模块已加载。再通过`virsh list --all`能正常输出（可能为空列表、显示none或paused）表明虚拟化服务正常。
