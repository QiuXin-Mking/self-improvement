# q
如何检查KVM内核模块是否已加载？
# a
执行以下命令：
```bash
lsmod | grep kvm
```

# q
如何检查CPU是否支持硬件虚拟化（Intel VT-x或AMD-V）？
# a
执行命令，若输出数字大于0则表示支持：
```bash
egrep -c '(vmx|svm)' /proc/cpuinfo
```

# q
如何使用virt-install命令创建虚拟机并导入已有的云镜像？
# a
```bash
sudo virt-install \
    --name my-vm \
    --ram 2048 \
    --vcpus 2 \
    --disk path=/var/lib/libvirt/images/ubuntu-22.04-server-cloudimg-amd64.img,format=raw \
    --os-type linux \
    --network bridge=virbr0 \
    --graphics vnc,listen=0.0.0.0 \
    --noautoconsole \
    --import
```

# q
运行虚拟机时遇到“Could not access KVM kernel module: Permission denied”错误，如何解决？
# a
将qemu用户和当前用户加入kvm组：
```bash
sudo usermod -aG kvm qemu
sudo usermod -aG kvm $(whoami)
```
然后重新登录或重启libvirtd服务使变更生效。

# q
如何配置libvirt虚拟网络，创建默认的NAT网桥（virbr0）并设置为自动启动？
# a
1. 创建网络定义XML文件（例如/tmp/default.xml），内容如下：
```xml
<network>
  <name>default</name>
  <uuid>e61eec6c-024e-41c1-8a7e-b123c5e42ffc</uuid>
  <forward mode='nat'/>
  <bridge name='virbr0' stp='on' delay='0'/>
  <ip address='192.168.122.1' netmask='255.255.255.0'>
    <dhcp>
      <range start='192.168.122.2' end='192.168.122.254'/>
    </dhcp>
  </ip>
</network>
```
2. 依次执行以下命令：
```bash
sudo virsh net-define /tmp/default.xml
sudo virsh net-start default
sudo virsh net-autostart default
```
