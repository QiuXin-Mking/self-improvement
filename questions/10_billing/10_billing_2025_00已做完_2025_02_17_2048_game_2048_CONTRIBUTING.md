# q
在为 2048 项目贡献代码时，应从哪个分支创建新分支？
# a
应基于 `master` 分支创建新分支，不要从 `gh-pages` 分支创建（除非绝对必要）。`master` 是游戏的“纯净”版本，而 `gh-pages` 包含分享功能、分析等与游戏核心无关的内容。

# q
如何正确修改项目的 CSS 样式？
# a
应编辑 `style/` 目录下的 SCSS 文件（如 `main.scss`），而不是直接编辑 `main.css`，因为 `main.css` 是根据 SCSS 自动生成的。修改后需使用 `sass` gem 编译，安装后运行以下命令自动监听变更并重新编译：
```
sass --unix-newlines --watch style/main.scss
```

# q
哪些类型的改动很可能不会被合并到主分支？
# a
以下改动可能不会被接受，或需要仔细评估：撤销/重做功能、保存/加载功能、更改方块外观或内容、更改布局、更改网格大小。这些会直接影响核心游戏玩法。

# q
在 JavaScript 代码中应遵循怎样的缩进规范？
# a
应使用 2 空格缩进。项目中提供了 `.jshintrc` 配置文件，安装并运行 `jshint` 可以帮助代码遵循规范。
