# q
该图像标注应用的整体架构由哪两个主要部分组成？
# a
应用拆分为两个方案：可弹性伸缩的Web前端（以EC2 Auto Scaling Group和Application Load Balancer为核心，实现负载自适应伸缩）和自动化的后端处理（由AWS Lambda无服务器函数驱动，通过事件触发机制自动完成图像描述生成和缩略图生成）。

# q
为提高安全性，应用实例和堡垒机的安全组规则应如何配置？
# a
应用实例安全组仅对ALB开放80/443端口，22端口仅对堡垒机安全组开放；堡垒机安全组仅对指定IP开放22端口，并使用网络ACL白名单保护其公网IP。

# q
使用AWS CLI创建面向互联网的Application Load Balancer的命令及关键参数是什么？
# a
```bash
aws elbv2 create-load-balancer \
  --name webapp-alb \
  --subnets subnet-aaa subnet-bbb \
  --security-groups sg-11112222 \
  --scheme internet-facing \
  --type application \
  --ip-address-type ipv4
```
关键参数：`--scheme internet-facing` 使负载均衡器对外暴露，`--type application` 指定为ALB，`--subnets` 至少传入两个不同可用区的子网。

# q
如何通过AWS CLI创建一个Auto Scaling Group并使其实例自动注册到ALB目标组？
# a
```bash
aws autoscaling create-auto-scaling-group \
  --auto-scaling-group-name webapp-asg \
  --launch-template LaunchTemplateName=webapp-template \
  --min-size 1 \
  --max-size 3 \
  --desired-capacity 1 \
  --vpc-zone-identifier "subnet-...,subnet-..." \
  --target-group-arns arn:aws:elasticloadbalancing:...
```
关键参数：`--launch-template` 指定启动模板，`--vpc-zone-identifier` 指定子网列表，`--target-group-arns` 将ASG实例关联到指定目标组以实现负载均衡和健康检查。

# q
在文档提供的示例中，创建RDS MySQL数据库实例的关键命令配置了哪些安全相关内容？
# a
```bash
aws rds create-db-instance \
  --db-instance-identifier imageappdb \
  --db-instance-class db.t3.micro \
  --engine mysql \
  --master-username admin \
  --master-user-password 'Admin1234!' \
  --allocated-storage 20 \
  --vpc-security-group-ids sg-0783450f5d3042229 \
  --publicly-accessible \
  --backup-retention-period 0
```
数据库实例通过 `--vpc-security-group-ids` 关联到已开放22、80、443端口的安全组，并通过 `--publicly-accessible` 设置为公开访问（不推荐生产环境使用），同时备份保留期设为0（不保留自动备份）。
