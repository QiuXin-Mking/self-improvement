# q
mosquitto_pub 命令中 -h、-p、-t、-m 参数分别代表什么含义？
# a
- `-h`：MQTT 代理服务器的主机名或 IP 地址（如 `101.201.53.141`）。
- `-p`：MQTT 服务的端口号（如 `1883`）。
- `-t`：发布消息的目标主题（Topic），例如 `"wq/v1.0/9167"`。
- `-m`：要发送的消息内容，通常为 JSON 字符串。

# q
示例中使用的 MQTT 主题 "wq/v1.0/9167" 具有怎样的层级结构？
# a
主题分为三个层级，由 `/` 分隔：
- `wq`：顶层命名空间（可能代表项目或产品标识）。
- `v1.0`：版本号。
- `9167`：设备或会话的唯一标识符。

# q
该测试方案发布的 JSON 消息包含哪些字段？其含义是什么？
# a
JSON 报文包含以下字段：
```json
{"motion":"1", "speed":50, "user_id":"U100", "user_name":"Test"}
```
- `motion`：运动模式（"1" 或 "2"），表示不同的工作状态。
- `speed`：速度值，整数（如 50、100）。
- `user_id`：用户标识。
- `user_name`：用户名称。
