# q
在CentOS/RHEL系统上安装KVM虚拟化环境需要安装哪些核心软件包？
# a
```bash
sudo yum install -y qemu-kvm libvirt libvirt-client virt-install bridge-utils
```
- `qemu-kvm`：KVM内核模块和QEMU虚拟机引擎
- `libvirt`：虚拟化通用管理服务（后端程序）
- `libvirt-client`：`virsh`和本地管理命令行工具
- `virt-install`：命令行建虚拟机工具
- `bridge-utils`：网络桥接支持（如需桥网络虚拟机互通）

# q
安装完成后如何启动并设置libvirtd服务自启动？
# a
```bash
sudo systemctl enable --now libvirtd
```
然后可检查状态：
```bash
sudo systemctl status libvirtd
```

# q
如何验证KVM内核模块已正确加载？
# a
```bash
lsmod | grep kvm
```
输出应显示`kvm_intel`或`kvm_amd`，表示模块已加载。

# q
如何确认libvirt守护进程可以正常管理虚拟机？
# a
```bash
virsh list --all
```
如果能列出虚拟机（即使当前没有虚拟机，显示空列表或`no state`等），说明libvirtd服务正常。
