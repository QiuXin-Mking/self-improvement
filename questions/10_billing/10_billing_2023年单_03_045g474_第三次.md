# q
在STM32的待机模式（standby mode）中，如何用VBAT仅保持RTC运行，并实现唤醒？
# a
待机模式下，VDD断电，仅VBAT供电给RTC模块，其他外设关闭。唤醒机制为RTC闹钟中断（Alarm interrupt），通过EXTI线17触发，唤醒后系统会复位并从main函数开头重新执行。

# q
STM32从待机模式唤醒后，程序会直接进入中断回调吗？
# a
不会。从待机模式唤醒时，系统会直接复位，重新从main()第一行代码开始运行，不会调用 `HAL_RTC_AlarmAEventCallback` 等回调，因此通常不会保持之前的程序状态。

# q
Sleep模式与Standby模式唤醒后的程序行为有什么区别？
# a
- Sleep模式唤醒后，程序继续从进入睡眠的位置后执行，可以调用RTC闹钟回调（如 `HAL_RTC_AlarmAEventCallback`）。
- Standby模式唤醒后，MCU复位，程序从头开始执行，不进入中断回调。

# q
能否使用RTC_OUT引脚输出信号直接驱动外部继电器或MOSFET来为系统板供电？
# a
不可行。实测RTC_OUT引脚在正常运行时可能无有效输出信号（脉冲或电平），无法可靠驱动外部开关。需要改用GPIO在唤醒后输出控制信号，或通过其他外设实现电源控制。

# q
RTC的闹钟中断（Alarm interrupt）与事件（event）有何区别？
# a
- 中断依赖中断控制器和EXTI线（如EXTI line 17），需要硬件支持，能唤醒处理器并触发中断服务程序。
- 事件不依赖中断控制器，仅需输入电平变化即可触发，例如按键按下信号，可作为唤醒源但不会进入中断向量。
