# q
如何检查 KVM 内核模块是否已加载？
# a
```bash
lsmod | grep kvm
```
如果输出包含 `kvm` 和 `kvm_intel` 或 `kvm_amd`，说明 KVM 模块正在运行。

# q
如何检查 CPU 是否支持硬件虚拟化（Intel VT-x 或 AMD-V）？
# a
```bash
egrep -c '(vmx|svm)' /proc/cpuinfo
```
如果输出的数字大于 0，则 CPU 支持虚拟化；`vmx` 表示 Intel VT-x，`svm` 表示 AMD-V。

# q
如何使用 `virt-install` 部署一个 KVM 虚拟机？
# a
```bash
sudo virt-install \
    --name my-vm \
    --ram 2048 \
    --vcpus 2 \
    --disk path=/var/lib/libvirt/images/ubuntu-22.04-server-cloudimg-amd64.img,format=raw \
    --os-type linux \
    --network bridge=virbr0 \
    --graphics vnc,listen=0.0.0.0 \
    --noautoconsole \
    --import
```
该命令将创建一个名为 `my-vm`、内存 2 GB、2 个 vCPU 的虚拟机，使用的镜像必须放在 `/var/lib/libvirt/images/` 下，并通过 `virbr0` 桥接网络提供 VNC 访问。

# q
安装虚拟机时若因镜像文件权限导致无法读取（例如镜像在 `/root` 下），应如何解决？
# a
将镜像移动到 libvirt 默认镜像目录，并赋予 `qemu:qemu` 所有权：
```bash
sudo mkdir -p /var/lib/libvirt/images
sudo mv /root/ubuntu-22.04-server-cloudimg-amd64.img /var/lib/libvirt/images/
sudo chown qemu:qemu /var/lib/libvirt/images/ubuntu-22.04-server-cloudimg-amd64.img
```

# q
如何创建并启用 libvirt 的默认虚拟网络 `virbr0`？
# a
1. 创建网络定义 XML 文件（例如 `/tmp/default.xml`）：
```xml
<network>
  <name>default</name>
  <uuid>e61eec6c-024e-41c1-8a7e-b123c5e42ffc</uuid>
  <forward mode='nat'/>
  <bridge name='virbr0' stp='on' delay='0'/>
  <ip address='192.168.122.1' netmask='255.255.255.0'>
    <dhcp>
      <range start='192.168.122.2' end='192.168.122.254'/>
    </dhcp>
  </ip>
</network>
```
2. 执行以下命令定义、启动网络并设为开机自启：
```bash
sudo virsh net-define /tmp/default.xml
sudo virsh net-start default
sudo virsh net-autostart default
```
