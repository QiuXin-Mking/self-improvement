# q
应用使用哪个Web框架处理HTTP请求，接收的JSON消息有哪些字段及motion值的含义？
# a
使用FastAPI框架。接收的JSON格式为 `{"motion":"001","speed":"002","user_id":"9167","user_name":"qx"}`。其中 motion 值含义：0 无效状态，1 还原动作，2 捡球动作，3 急停动作。

# q
tennis.conf配置文件可以配置哪些运行参数？
# a
可通过 tennis.conf 修改：FastAPI 服务的 IP 和端口；数据库连接文件路径（默认 `/root/.mycnf`）；MQTT 的 IP 和端口。

# q
对消息中的 motion 和 speed 字段进行校验的规则是什么？
# a
motion 值必须在 0-3 之间，speed 值必须在 0-100 之间。

# q
接收到HTTP消息后需要转发到MQTT，使用的主题格式是什么？
# a
MQTT 主题为 `wq/v1.0/{user_id}`，`{user_id}` 替换为消息中的实际用户ID。

# q
将 tennis_log 表的 timestamp 字段从整数类型改为 DATETIME 类型的SQL步骤是什么？
# a
```sql
-- 1. 新增一个临时 DATETIME 字段
ALTER TABLE tennis_log ADD COLUMN timestamp_dt DATETIME;

-- 2. 用原时间戳字段填充新字段
UPDATE tennis_log SET timestamp_dt = FROM_UNIXTIME(`timestamp`);

-- 3. 删除原有的 timestamp 字段
ALTER TABLE tennis_log DROP COLUMN `timestamp`;

-- 4. 把新字段重命名为 timestamp
ALTER TABLE tennis_log CHANGE timestamp_dt `timestamp` DATETIME NOT NULL;
```
