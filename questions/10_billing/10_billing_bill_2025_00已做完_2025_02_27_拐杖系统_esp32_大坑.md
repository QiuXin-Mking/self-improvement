# q
在 ESP32 的 Arduino 环境中，使用 `Serial2.begin(9600, SERIAL_8N1, 16, 17);` 初始化串口时，实际定义的 RX 和 TX 引脚分别是哪个？
# a
根据 ESP32 Arduino 核心库的定义，`Serial2.begin(baud, config, rxPin, txPin)` 参数顺序为**波特率、配置、RX引脚、TX引脚**。因此语句 `Serial2.begin(9600, SERIAL_8N1, 16, 17);` 实际指定的 **RX = GPIO16**，**TX = GPIO17**。  
常见注释 `// TX=16, RX=17` 是错误的（参数顺序与直觉相反），这就是典型的大坑。

# q
在 ESP32 上使用 `Serial2` 时，如何通过代码指定自定义的 GPIO 引脚（例如 RX=16, TX=17）并设置 9600-8N1 格式？
# a
使用如下代码：
```cpp
Serial2.begin(9600, SERIAL_8N1, 16, 17); // RX=16, TX=17
```
注意参数位置：第三个是 RX 引脚，第四个是 TX 引脚，切勿与常见的 `(TX, RX)` 顺序混淆。
