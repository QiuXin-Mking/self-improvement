# q
在二层广播域中，两台主机配置了不同的子网掩码（如 `/16` 和 `/24`），为何它们仍能直接 ping 通？
# a
因为双方都认为对方的 IP 地址落在自己的子网范围内：
- 主机 94（`192.168.234.111/16`）网络为 `192.168.0.0/16`，目标 `192.168.234.110` 在该范围内。
- 主机 95（`192.168.234.110/24`）网络为 `192.168.234.0/24`，目标 `192.168.234.111` 也在该范围内。
因此两台主机都会直接发送 ARP 请求解析对方 MAC 地址，实现二层通信，无需经过网关。

# q
主机如何判断目标 IP 是否与自己处于同一子网？
# a
将目标 IP 地址与本机的子网掩码进行按位“与”运算，如果结果等于本机的网络地址，则认为目标在同一子网，直接通过 ARP 获取 MAC 进行通信；否则将数据包发给默认网关。

# q
实验中同网段不同 prefix 能通，路由表中的哪些条目起了关键作用？
# a
两台主机的路由表中都包含直连路由，例如：
- 94：`192.168.0.0/16 dev ens256 proto kernel scope link`
- 95：`192.168.234.0/24 dev ens36 proto kernel scope link`
这些条目让内核知道对应网络通过指定接口直接可达，无需下一跳，因此ARP 直接在该接口上发起。

# q
同网段不同 prefix 的主机能成功通信的必要条件是什么？
# a
必要条件是**双方互相认为对方的 IP 在自己配置的子网内**（即各自的子网掩码与对方 IP 做 AND 运算后能匹配）。如果任何一方认为对方不在子网内，就会将数据包发往默认网关，导致通信可能失败。
