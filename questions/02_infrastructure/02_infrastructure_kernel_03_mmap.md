# q
mmap 为什么不被广泛应用在传统存储系统设计中？
# a
1. 刷盘时机不确定，内核自动控制。  
2. 数据一致性/持久性不如直接 write+fsync 易控制。  
3. 难以获得已刷盘/未刷盘的精确信息。  
4. 压力大时的行为难以预测（如回收、脏页丢失）。

# q
使用 mmap 时，刷盘时机由谁控制？
# a
由内核自动控制，时机不确定。

# q
mmap 相比 write+fsync 在数据持久性方面有什么不足？
# a
数据一致性/持久性不如直接 write+fsync 易控制。

# q
在使用 mmap 的场景中，能否精确知道哪些数据已刷盘？
# a
难以获得已刷盘/未刷盘的精确信息。

# q
mmap 在系统高负载下可能出现哪些问题？
# a
行为难以预测，例如页回收、脏页丢失。
