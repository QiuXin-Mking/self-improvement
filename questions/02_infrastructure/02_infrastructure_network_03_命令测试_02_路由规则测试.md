# q
在一台服务器上，当192.168.234.104网卡故障后，如何通过172.22.251.104接口访问192.168.234.105？
# a
通过添加静态路由将目标流量导向可用的网关，命令：
```bash
sudo ip route add 192.168.234.105 via 172.22.251.105
```

# q
如何查看当前系统的IPv4路由表，以确认新添加的路由是否生效？
# a
使用 `ip route` 命令：
```bash
ip route
```
示例输出中会包含新增路由：
```
192.168.234.105 via 172.22.251.105 dev eno1
```

# q
如何删除之前添加的 `192.168.234.105 via 172.22.251.105` 这条临时路由？
# a
使用 `ip route del` 命令：
```bash
sudo ip route del 192.168.234.105 via 172.22.251.105
```
