# q
如何配置多台服务器之间的SSH免密登录？
# a
通过SSH密钥认证实现。步骤：1）在本地生成RSA密钥对（若不存在）；2）使用 `ssh-copy-id` 将公钥分发到每台服务器；3）将每台服务器的公钥收集到本地 `authorized_keys` 文件；4）将汇总后的 `authorized_keys` 分发到所有服务器，确保任意两台服务器之间可免密通信。具体命令参见提供的 `setup_ssh_trust.sh` 脚本。

# q
脚本中 `ssh-keygen -t rsa -N "" -f ~/.ssh/id_rsa` 这条命令的作用是什么？
# a
用于生成RSA密钥对。`-t rsa` 指定密钥类型为RSA；`-N ""` 表示私钥不设密码（空密码），可实现无交互登录；`-f ~/.ssh/id_rsa` 指定密钥文件路径，生成 `id_rsa`（私钥）和 `id_rsa.pub`（公钥）。

# q
脚本如何将一台服务器的公钥收集到本地，并最终实现所有服务器互信？
# a
通过三个循环完成：
1. 第一个循环：将本机公钥复制到所有服务器（`ssh-copy-id`）。
2. 第二个循环：从每台服务器读取 `~/.ssh/id_rsa.pub`，追加到本机的 `authorized_keys`。
3. 第三个循环：将汇总了所有公钥的 `authorized_keys` 分发到每一台服务器，确保每台服务器都信任其他所有主机。

# q
测试SSH鉴权是否成功的脚本是如何工作的？
# a
定义一个服务器列表（`mds`、`oss`、`mgs`），然后循环执行 `ssh "$ser" date`。如果无需密码即可远程执行 `date` 命令并返回正确时间，说明鉴权配置成功。
