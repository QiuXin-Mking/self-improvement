# q
`ip rule show` 命令的作用是什么？
# a
`ip rule show` 用于显示当前系统中的策略路由规则（Routing Policy Database）。每条规则定义了如何根据条件（如源地址）选择不同的路由表进行数据包转发。

# q
在 `ip rule show` 输出中，每行开头的数字代表什么？优先级如何影响路由决策？
# a
开头的数字（如 `0:`、`32759:`）是规则的优先级，数值越小优先级越高。系统按优先级从小到大顺序匹配规则，当数据包的属性（如源 IP）满足某条规则的条件时，就使用该规则指定的路由表进行查找，后续低优先级规则不再匹配。

# q
如何理解规则 `32759: from 172.250.101.104 lookup 11`？
# a
这条规则表示：优先级为 32759，对于源 IP 地址是 `172.250.101.104` 的数据包，使用编号为 `11` 的自定义路由表进行路由查找，而不是默认的主表（`main`）。

# q
在 `ip rule show` 输出中，`local`、`main`、`default` 路由表分别有什么作用？
# a
- `local`（规则 0）：处理本地和广播流量的专用路由表，优先级最高，包含回环地址和本地接口地址的路由。
- `main`（规则 32766）：默认的主路由表，通过 `ip route` 添加的普通路由通常存放在此表。
- `default`（规则 32767）：系统默认的空路由表，通常作为最后回退处理。
