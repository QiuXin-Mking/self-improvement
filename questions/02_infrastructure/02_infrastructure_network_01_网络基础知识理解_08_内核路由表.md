# q
如何解读 `route -n` 输出的内核路由表各列含义，尤其是 Flags 标志？
# a
- **Destination**：目标网络或 IP 地址。
- **Gateway**：到达目标的网关 IP 地址；`0.0.0.0` 表示直连网络，无需网关。
- **Genmask**：目标网络的子网掩码。
- **Flags**：
  - `U`：路由启用（Up）
  - `G`：需经过网关（Gateway）
  - 示例中 `UG` 表示启用且为网关路由，`U` 表示直连路由。
- **Metric**：度量值，数值越小优先级越高，用于路由选择。
- **Ref**：对此路由的引用次数（示例中均为 0）。
- **Use**：使用此路由发送的数据包计数。
- **Iface**：出口网络接口名称。

# q
路由表中的默认路由（`0.0.0.0`）有什么作用？示例中的默认路由是如何配置的？
# a
默认路由的目标为 `0.0.0.0`，掩码为 `0.0.0.0`，匹配所有未被其他具体路由匹配的数据包。示例中默认路由条目为：
```
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
0.0.0.0         172.22.0.1      0.0.0.0         UG    100    0        0 eno1
```
表示所有不匹配 `10.10.0.0/16`、`172.22.0.0/16`、`192.168.0.0/16` 的流量，均通过网关 `172.22.0.1` 从接口 `eno1` 送出。

# q
为什么路由表中有些条目的 Gateway 是 `0.0.0.0`？这些条目与接口有什么关系？
# a
Gateway 为 `0.0.0.0` 表示目标网络是主机直连的本地子网，数据包直接从该条目指定的网络接口（Iface）发送，不需要经过任何网关。示例中有三条直连路由：

| 目标网络      | 网关     | 接口      | 说明                   |
|---------------|----------|-----------|----------------------|
| `10.10.0.0/16` | `0.0.0.0` | `docker0` | Docker 网桥内部网络     |
| `172.22.0.0/16` | `0.0.0.0` | `eno1`    | 主网卡本地子网          |
| `192.168.0.0/16` | `0.0.0.0` | `enp94s0f0` | 另一物理网卡本地子网    |

这些条目标志均为 `U`（无 `G`），对应接口上配置了 IP 地址（`172.22.251.104/16`、`192.168.234.104/16`、`10.10.0.1/16`），主机可以直接与这些子网内的其他设备通信。
