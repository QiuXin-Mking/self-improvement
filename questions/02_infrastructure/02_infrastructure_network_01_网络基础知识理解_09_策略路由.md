# q
什么是策略路由（Policy Routing）？
# a
策略路由是一种基于特定策略或条件（如源地址、接口、标记等）选择路由的方法，不同于传统的仅基于目的地IP地址的路由。在Linux中可使用`ip`命令实现更灵活的路由控制。

# q
在策略路由配置中，数字200代表什么？
# a
数字200是用户定义的路由表标识符，用于唯一标识一个路由表。用户自定义标识符通常为大于255的整数，以避免与系统保留表（如`255 local`、`254 main`、`253 default`）冲突。“200”仅为示例，实际应以`/etc/iproute2/rt_tables`文件中的记录为准。

# q
如何检查策略路由表标识符是否与其他标识符冲突？
# a
查看`/etc/iproute2/rt_tables`文件内容即可。已使用的标识符和表名均记录在该文件中，选择新标识符时需确保不重复。可使用`cat /etc/iproute2/rt_tables`查看。

# q
命令`ip rule add from 192.168.1.2 table test_table`的作用和组成部分是什么？
# a
该命令为Linux添加一条策略路由规则：当数据包源地址为`192.168.1.2`时，使用名为`test_table`的路由表进行路由。
- `ip rule`：操作策略路由规则。
- `add`：添加新规则。
- `from 192.168.1.2`：规则匹配条件，限定源地址。
- `table test_table`：满足条件时使用的路由表名称（需预先创建）。
