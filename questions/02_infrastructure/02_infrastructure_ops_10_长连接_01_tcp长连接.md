# q
TCP长连接的核心原理是什么？
# a
客户端和服务器建立TCP连接后，通过应用层协议或心跳机制保持连接活跃，避免因长时间无数据而被网络中间设备（如防火墙、NAT）关闭，从而持续进行数据交换。

# q
保持TCP长连接活跃常用的技术手段有哪些？
# a
定期发送心跳包检测连接有效性，同时实现自动重连的逻辑，当检测到连接断开时尝试重新建立连接。

# q
实现TCP长连接服务器端的关键步骤有哪些？
# a
1. 创建socket（AF_INET, SOCK_STREAM）
2. bind绑定监听地址和端口
3. listen设置最大连接队列
4. accept阻塞等待客户端连接
5. 循环使用recv/send或read/write收发数据并处理异常断开

# q
为什么TCP长连接需要心跳机制？
# a
因为网络中间设备（如防火墙、NAT）可能因连接空闲而超时断开，心跳包保持有规律的轻量级数据传输，既能防止空闲断开，又能及时探测对端是否存活，触发重连。
