# q
如何添加一条静态路由，使服务器A（172.22.6.65）通过172.22.102.96访问服务器B的192.168.234.112？
# a
在服务器A上执行以下命令：
```bash
sudo ip route add 192.168.234.112 via 172.22.102.96
```

# q
如何查看当前系统的路由表以确认路由规则是否生效？
# a
使用 `ip route` 命令查看路由表，例如：
```bash
ip route
```
添加路由后，输出中会包含类似 `192.168.234.112 via 172.22.102.96 dev eno1` 的条目。

# q
如何删除已添加的静态路由 `192.168.234.112 via 172.22.102.96`？
# a
执行以下命令：
```bash
sudo ip route del 192.168.234.112 via 172.22.102.96
```
删除后，对应的路由条目将从 `ip route` 的输出中消失。

# q
如何添加基于源地址的策略路由规则，将从172.22.6.65发出的流量引导至指定的路由表？
# a
使用 `ip rule` 命令添加规则：
```bash
ip rule add from 172.22.6.65 table 10
```
该规则要求预先在路由表10中定义实际的路由条目（例如通过 `ip route add ... table 10` 添加）。
