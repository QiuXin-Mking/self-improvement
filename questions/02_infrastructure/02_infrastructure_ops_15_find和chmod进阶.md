# q
`chmod +x` 命令对文件和目录分别有何作用？
# a
- 对于**目录**：添加执行权限后，用户才能进入（`cd`）该目录。
- 对于**文件**：添加执行权限后，用户可以直接执行该文件（若其本身是可执行的）。
- 默认情况下 `chmod +x` 不会递归，只影响当前层级的目录或文件。

# q
如何递归地为整个文件夹及其所有子目录和文件添加执行权限？
# a
不能直接用 `chmod -R +x`（会不加区分地修改所有文件和目录），推荐使用 `find` 分别处理目录和文件：
```bash
# 递归给所有目录添加执行权限（允许进入）
find myfolder -type d -exec chmod +x {} \;
# 递归给所有普通文件添加执行权限
find myfolder -type f -exec chmod +x {} \;
```
其中 `-type d` 匹配目录，`-type f` 匹配普通文件，`-exec` 将匹配到的路径传给 `chmod` 执行。

# q
为什么递归设置执行权限时推荐将目录和文件分开处理？
# a
因为目录的执行权限含义是“可进入”，文件的执行权限含义是“可执行”。对所有内容无差别地添加执行权限会导致所有普通文件都被标记为可执行，存在安全风险且不符合实际需求。分开处理可以精确控制：仅让目录可进入，只对确实需要执行的文件添加执行权限。
