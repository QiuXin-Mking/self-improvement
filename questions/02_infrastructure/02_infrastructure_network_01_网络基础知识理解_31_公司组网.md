# q
在Windows系统中，如何添加一条永久的静态路由，将所有发往172.34.0.0/16网段的流量指向下一跳172.17.0.1？
# a
使用 `route add` 命令并加上 `-p` 参数使其持久化：
```
route add 172.34.0.0 mask 255.255.0.0 172.17.0.1 -p
```
其中：
- `172.34.0.0` 为目标网络
- `mask 255.255.0.0` 指定子网掩码（/16）
- `172.17.0.1` 为下一跳网关地址
- `-p` 表示将该路由永久添加到注册表，重启后仍有效

# q
Windows下查看当前路由表的命令是什么？
# a
```
route print
```
该命令会显示IPv4和IPv6路由表，包括活动路由和永久路由。

# q
`route add` 命令中的 `-p` 参数的作用是什么？
# a
`-p` 表示“持久”（persistent），添加的路由会被写入系统注册表，在计算机重启后依然存在。若不使用 `-p`，该静态路由仅在当前会话中有效，重启后丢失。

# q
用 `ping` 测试连通性时，如何判断目标主机是否可达？
# a
若显示“来自 <IP> 的回复：字节=32 时间<1ms TTL=128”等类似信息，说明连通性正常；若显示“请求超时”或“目标主机不可达”，则说明网络不通或路由配置有问题。本笔记中 `ping 172.22.0.1` 和 `ping 172.34.0.1` 均返回“通”，表示目标可达。
