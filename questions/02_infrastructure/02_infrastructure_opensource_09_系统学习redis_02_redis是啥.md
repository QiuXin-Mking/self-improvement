# q
Redis是什么？如何理解它？
# a
Redis（Remote Dictionary Server）是一个开源的内存数据库，遵守BSD协议，提供高性能的键值（key-value）存储系统。它支持多种数据类型，常用于缓存、消息队列、会话存储等场景，能够支持每秒数十万次的读写操作。

# q
Redis的核心特点有哪些？
# a
- **性能极高**：基于内存，支持每秒数十万次读写操作
- **丰富的数据类型**：字符串、列表、集合、哈希表、有序集合等
- **原子性操作**：所有操作要么完全执行，要么不执行
- **持久化**：可将内存数据保存到磁盘，系统重启后恢复
- **发布/订阅模式**：支持消息传递和实时数据传输
- **单线程模型**：通过事件驱动处理并发，简化并发控制
- **主从复制**：从节点备份数据或分担读请求，提高可用性和伸缩性
- **支持Lua脚本**：可在服务器端执行复杂操作
- **跨平台兼容**：支持Linux、macOS、Windows

# q
Redis与其他key-value存储的主要区别是什么？
# a
- **丰富的数据类型**：除基本key-value外，还提供list、set、zset、hash等数据结构存储
- **高性能读写**：读速度约110000次/s，写速度约81000次/s，数据存储在内存中
- **原子性操作**：保证数据一致性
- **持久化机制**：支持将内存数据持久化到磁盘，避免数据丢失
- **高级特性**：内置发布/订阅、通知、key过期等
- **主从复制和高可用**：支持master-slave备份
- **Lua脚本支持**：提供灵活的服务端编程能力
- **单线程架构**：简化并发设计，仍然保持高吞吐
