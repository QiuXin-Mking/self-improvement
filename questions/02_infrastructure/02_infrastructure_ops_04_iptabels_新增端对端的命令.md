# q
如何新增一条允许特定源IP访问特定目的IP的iptables INPUT规则？
# a
使用以下命令在 INPUT 链中追加一条规则，允许来自 172.16.7.20 且目标为 10.3.41.228 的流量通过：
```bash
sudo iptables -A INPUT -s 172.16.7.20 -d 10.3.41.228 -j ACCEPT
```
其中 `-A INPUT` 表示追加到 INPUT 链，`-s` 指定源IP，`-d` 指定目的IP，`-j ACCEPT` 表示接受匹配的数据包。

# q
命令 `sudo iptables -A INPUT -s 172.16.7.20 -d 10.3.41.228 -j ACCEPT` 的作用是什么？
# a
该命令在 iptables 的 INPUT 链中添加一条规则，作用是允许源IP为 172.16.7.20 且目的IP为 10.3.41.228 的数据包进入本机，即放行从 172.16.7.20 发往 10.3.41.228 的流量（通常目的IP为本机地址）。命令通过 `-j ACCEPT` 直接接受匹配的包，不再继续匹配后续规则。
