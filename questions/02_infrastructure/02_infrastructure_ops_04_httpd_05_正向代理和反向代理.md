# q
正向代理是什么，它通常有哪些用途？
# a
正向代理是一种位于客户端和目标服务器之间的代理服务器，用于代表客户端发出请求并获取资源。它常用于访问控制、缓存加速、隐藏客户端真实IP（匿名性）、内容过滤以及绕过地理限制等场景。

# q
正向代理的核心工作原理是什么？
# a
客户端先被配置为通过代理服务器发送请求，之后的工作流程为：
1. 客户端将请求发送给正向代理服务器。
2. 代理服务器接收后，代替客户端向目标服务器转发请求。
3. 目标服务器将响应返回给代理服务器。
4. 代理服务器再将响应返回给客户端。

示意图：`[客户端] <--> [正向代理服务器] <--> [目标服务器]`

# q
正向代理与反向代理的主要区别是什么？
# a
- 正向代理：位于客户端与目标服务器之间，代理代表客户端向目标服务器发起请求。
- 反向代理：位于目标服务器与客户端之间，代理代表目标服务器接收客户端请求并转发给后端服务器。

两者的代理对象不同：正向代理代理客户端，反向代理代理服务端。

# q
如何为浏览器配置正向代理？
# a
以 Firefox 为例：
1. 打开浏览器设置，找到网络设置或代理设置。
2. 输入正向代理服务器的地址和端口，例如 `proxy.example.com:8080`。
