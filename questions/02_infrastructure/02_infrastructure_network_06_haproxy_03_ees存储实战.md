# q
evip是什么？
# a
evip即External VIP，是控制台接入的伪公网IP。客户端请求从伪公网IP网关进入万兆交换机，经千兆交换机到达节点，通过网桥进入OSPF网络命名空间，该命名空间中的伪公网IP就是evip。

# q
如何理解Linux网络命名空间（netns）？
# a
网络命名空间是Linux内核提供的网络隔离机制，每个命名空间拥有独立的网络栈（包括iptables策略）。可通过`ip netns list`查看。Docker利用该特性实现容器网络隔离，也可用于在本地虚拟化出多个独立网络环境。

# q
管理流量和数据流量的路径有什么区别？
# a
管理流量路径：公网网关IP → 万兆交换机 → 千兆交换机 → 实体网卡 → 虚拟网桥 → evip → haproxy协议 → 请求分发。  
数据流量路径：公网网关IP → 万兆交换机 → 服务器实体网卡 → OSPF协议。

# q
haproxy命名空间的作用是什么？
# a
用于实现负载均衡。在命名空间中运行的haproxy可以对请求进行分发，完成负载均衡。
