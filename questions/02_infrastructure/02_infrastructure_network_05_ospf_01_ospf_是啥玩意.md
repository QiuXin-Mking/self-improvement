# q
OSPF是什么？
# a
OSPF（Open Shortest Path First，开放式最短路径优先）是一种动态路由协议，属于内部网关协议（IGP），基于链路状态算法。它通过接口状态与链路状态数据库来生成最短路径树，并将最佳路径放入路由表。OSPF 分为 OSPFv2（用于 IPv4）和 OSPFv3 两个版本，也称为接口状态路由协议。

# q
OSPF 的运行过程包含哪些核心步骤？
# a
1. 在所有开启 OSPF 的接口上以组播方式发送 Hello 报文，发现 OSPF 邻居；  
2. 建立 OSPF 邻居关系，形成邻居表；  
3. 向各邻居发送 LSA 报文，相互通告路由，形成 LSDB（链路状态数据库）；  
4. 基于 LSDB 通过 SPF（最短路径优先）算法计算最佳路径（cost 度量值最小），并放入路由表。

# q
OSPF 中的 LSDB 是什么？它如何生成？
# a
LSDB（链路状态数据库）是 OSPF 路由器通过相互发送 LSA（链路状态通告）报文通告路由后形成的数据库，包含网络拓扑的链路状态信息，用于 SPF 算法计算最短路径树和最佳路由。
