# q
网络桥是什么，工作在OSI模型的哪一层？
# a
网络桥（Bridge）是一种用于连接并过滤不同网络的设备，工作在数据链路层（第二层），通过MAC地址识别和转发数据帧，将多个网络段连接成一个逻辑上单一的网络。

# q
网络桥如何实现数据转发而不是广播到所有端口？
# a
网络桥通过**学习功能**记录每个端口上设备的MAC地址，形成MAC地址表。转发时根据目标MAC地址查询该表，仅将数据帧发送到目标设备所在的特定网络段，实现过滤和精准转发，而不是泛洪到所有端口。

# q
网络桥在提升网络性能方面有什么作用？
# a
网络桥可以**隔离碰撞域**：通过只向目标网络段转发数据，而不是广播到整个网络，减小了碰撞域的范围，从而减少冲突机会，提高网络整体性能。

# q
为什么说网络桥对上层协议是透明的？
# a
网络桥仅工作在数据链路层，基于MAC地址处理数据帧，不解析或修改上层（网络层及以上）协议的数据。因此只要使用相同的数据链路层协议（如以太网），它就可以连接不同类型的上层网络，对上层完全透明。

# q
网络桥和交换机的关系是什么？
# a
交换机是现代网络中更先进的设备，通常具有更多端口和更强的处理能力，但在核心功能上执行与网桥类似的任务（连接、过滤、基于MAC地址转发）。交换机可视为多端口、高性能的网络桥，在许多场景中已取代传统的网桥。
