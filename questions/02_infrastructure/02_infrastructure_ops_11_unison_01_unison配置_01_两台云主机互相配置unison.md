# q
unison配置文件中的 `batch` 参数有什么作用？
# a
`batch = true` 启用批处理模式，从不提示用户，自动接受默认答案，对无人值守脚本至关重要。

# q
unison 的 `fastcheck` 参数如何影响同步性能？
# a
`fastcheck = true`（默认）时，首先比较文件大小和修改时间，只有不同时才进行内容校验，适合异地低延迟环境；设为 false 会强制对所有文件进行内容校验，速度非常慢，不建议在高延迟网络中使用。

# q
unison 如何自动处理文件冲突，避免交互式确认？
# a
可以在 `.prf` 配置文件中通过 `prefer = newer` 让版本较新的文件获胜，或使用 `force` 指定某一方覆盖另一方（如 `force = ssh://...//path`），配合 `batch = true` 实现无人值守自动解决冲突。
