# q
判断 .back 文件是否有替换价值的 5 条标准是什么？
# a
1. **内容完整性**：.back 文件明显更完整（章节/参数解释/排查流程）
2. **结构化**：.back 文件有清晰标题层级、代码高亮、表格
3. **故障案例**：.back 将流水账重构为完整排查文档
4. **技术细节**：.back 补充函数说明、错误处理、依赖清单
5. **体系化**：.back 按模块组织，形成学习路径

# q
审查结果应输出到哪个文件？命名格式是什么？
# a
输出到 `ai_并行/{agent_id}.md`，其中 `{agent_id}` 是当前 agent 的标识符。

# q
执行一次文件对审查任务的标准流程包含哪些步骤？
# a
1. 读取源文件和 .back 文件内容
2. 对比内容差异
3. 根据 5 条判断标准评估
4. 填写审查结论表格（包括“是否应该替换”等 6 列）
5. 如有需要，输出替换命令
6. 将结果写入 `ai_并行/{agent_id}.md`

# q
审查结果表格中的“是否应该替换”与“是否有替换价值”有何区别？
# a
- **是否应该替换**：最终操作决策，取值为“是”、“否”、“待定”
- **是否有替换价值**：基于 5 条标准的客观判断，取值为“是”、“否”、“未判断”
