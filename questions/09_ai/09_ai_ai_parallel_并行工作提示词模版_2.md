# q
如何使用提示词模板为大批量文件审查分配子任务序号范围？
# a
使用计算公式：总文件数 1154，每 agent 处理 10 个，需要 agent 数 = ceil(1154/10) = 116 个。agent N 的处理范围为：
- 起始序号：(N-1) * 10 + 1
- 结束序号：min(N * 10, 1154)

# q
并行审查任务中判断 .back 文件是否有替换价值的 5 条标准是什么？
# a
满足任一条件即有替换价值：
1. 内容完整性：.back 文件明显更完整（章节/参数解释/排查流程）
2. 结构化：有清晰标题层级、代码高亮、表格
3. 故障案例：将流水账重构为完整排查文档
4. 技术细节：补充函数说明、错误处理、依赖清单
5. 体系化：按模块组织，形成学习路径

# q
审查输出模板中的表格必须包含哪些列？替换命令汇总的加入条件是什么？
# a
表格列：序号、源文件路径、.back 文件路径、是否应该替换、是否有替换价值、是否审查过、理由（20 字内）。替换命令汇总仅当"是否应该替换"="是"时，才列入 ```cp `{.back 文件路径}` `{源文件路径}` ``` 命令。
