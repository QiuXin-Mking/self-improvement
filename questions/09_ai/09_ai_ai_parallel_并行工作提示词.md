# q
并行审查任务中，判断 .back 文件是否有替换价值的五条标准是什么？
# a
满足任一条件即有替换价值：
1. 内容完整性：.back 文件明显更完整（章节/参数解释/排查流程）
2. 结构化：.back 文件有清晰标题层级、代码高亮、表格
3. 故障案例：.back 将流水账重构为完整排查文档
4. 技术细节：.back 补充函数说明、错误处理、依赖清单
5. 体系化：.back 按模块组织，形成学习路径

# q
在并行审查输出模板中，审查结论表格包含哪些列？
# a
表格包含 7 列：序号、源文件路径、.back 文件路径、是否应该替换、是否有替换价值、是否审查过、理由。
其中“理由”字段要求 20 字以内，替换命令汇总仅收录“是否应该替换”为“是”的行。

# q
当总文件数为 N，每个 agent 处理 M 个文件时，如何计算需要的 agent 数以及每个 agent 的处理序号范围？
# a
- 需要的 agent 数 = ceil(N / M)
- 对于第 k 个 agent（k 从 1 开始）：
  - 起始序号 = (k-1) * M + 1
  - 结束序号 = min(k * M, N)

例如 N=1154, M=10，则需要 116 个 agent；agent 1 处理序号 1-10，agent 2 处理 11-20，agent 116 处理 1141-1154。
