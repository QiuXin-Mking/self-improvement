# q
技术文档整理助手在丰富内容单薄或缺失的文档时，遵循的任务流程是什么？
# a
1. 读取原文件，了解现有内容和结构，记录标题与格式。
2. 探索上下文，查看同目录文件与知识库中的参考资料。
3. 分析文档主题，根据标题/文件名识别其所属领域（工具类、概念类、操作类、经验类）。
4. 丰富文档内容，按照不同文档类型的对应结构补充信息。

# q
根据文档标题或文件名，如何识别文档所属领域？有哪些类别？
# a
根据标题/文件名，可分为四类：
- 工具类：介绍工具的用途、特点、使用方法
- 概念类：解释概念的定义、原理、应用场景
- 操作类：提供步骤指南、注意事项、常见问题
- 经验类：记录案例、问题排查、最佳实践

# q
在丰富工具类文档时，应遵循什么样的内容结构？
# a
工具类文档结构包括：
- 工具概述（是什么、谁开发、用途）
- 主要特点/核心功能
- 使用场景/典型用例
- 使用方法/操作步骤
- 参数说明/配置选项
- 输出解读/结果分析
- 与其他工具对比
- 常见问题 (FAQ)
- 相关工具/参考资料

# q
编写补充文档时，有哪些需要遵守的原则？
# a
- 使用清晰的一级/二级/三级标题组织内容
- 适当使用表格进行对比和总结
- 代码示例使用代码块标注
- 保持技术准确性，避免模糊表述
- 添加内部链接指向知识库中的相关文档

# q
技术文档整理助手在输出结果时，对格式有哪些要求？
# a
- 保持 Markdown 格式
- 文档标题使用 `# ` 一级标题
- 章节使用 `## ` 二级标题
- 必要时使用 `### ` 三级标题
