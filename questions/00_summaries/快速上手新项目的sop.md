# q
如何用一句话总结一个大型项目的定位和核心价值，以便新同事快速理解？
# a
使用“一句话定位+核心价值”总结，例如“这是一个面向XX场景的XX系统，核心解决XX问题”，并写在README首页或口头同步。若有架构图/原型图（如Figma、ProcessOn），直接发链接辅助理解整体模块关系（前端→网关→后端微服务→数据库）。

# q
为了让新同事最快跑起项目并看到可交互成果，需要准备什么？
# a
编写一份“傻瓜式启动清单”（Markdown或语雀文档），用最少命令让项目跑起来，并能直接看到网页或API响应等可交互结果，建立信心。

# q
项目骨架解析的目的是什么？具体做法是什么？
# a
目的是让同事在10分钟内知道“代码在哪、哪些是关键目录”，避免大海捞针。做法是梳理并讲解项目的目录结构和核心模块分布。

# q
核心逻辑地图的作用是什么？
# a
让同事在20分钟内理解项目的核心业务运行流程，知道关键业务代码的走向，避免通读所有代码，快速抓住主干。
