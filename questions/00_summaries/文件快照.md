# q
快照机制的基本思想是什么？
# a
在需要快照的瞬间暂停所有IO，复制元数据链作为快照入口；由于元数据链数据规模很小，复制几乎瞬间完成，不会影响应用程序IO。之后通过两种数据分配思想保存物理磁盘数据：RoFW（第一次写时重定向）或 CoFW（第一次写时复制）。

# q
RoFW 与 CoFW 在性能和副作用上的主要区别有哪些？
# a
- RoFW：主系统数据被重定向，需要访问重定向表，更吃计算资源；会影响读性能；删除快照后会产生大量外部碎片，清理困难。
- CoFW：在第一次写时复制快照数据，更吃 IO 资源；删除快照后取消重定向没有后遗症，不会产生碎片。

# q
如何通过技术手段缩短 CoFW 写时复制快照的写响应时间？
# a
1. 预先将存储空间划分为源数据区和快照区，使用快照链表保存快照元数据，用快照索引结构（地址映射表）判断数据块是否已在快照区。
2. 写请求到来时，将数据按起始地址和长度分块，查询索引结构判断每个分块是否已存；若未存则在快照区分配空间（空间不足时删除最旧快照）。
3. 启动快照读进程，将源数据区的数据块读入内存的快照数据写入队列队尾，更新快照区空间指针。
4. 所有分块处理完毕后，允许写请求的数据写入源数据区；满足启动条件时，内存队列中的数据依次出队写入快照区，物理 Copy 过程与系统并发，对应用透明。

# q
快照回滚覆盖源物理卷时，为保证交互友好和数据一致通常采用什么方式？需要注意哪些关键操作？
# a
通常采用后台 Rollback：接收到命令后立即向客户端返回成功，系统可以继续对任意区域进行 IO。需注意：
- 返回成功前必须将 Buffer 中的内容 Flush 到源卷，防止后续覆盖导致数据不一致；
- 不应直接清空 Buffer，以防回滚失败；为确保一致性，也可先 Unmount 源卷，回滚完成后再 Mount。
- 回滚过程中若需保留其他快照，仍要对源卷进行 CoFW 操作；但多数设计认为回滚后后续时刻的快照不应存在，此时可忽略此操作。
