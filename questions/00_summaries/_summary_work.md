# q
52_bill 嵌入式接单项目库的核心业务模式是什么？
# a
通过闲鱼/淘宝等平台接单，为客户（以大学生毕设、课程设计为主）定制基于 STM32/ESP32/树莓派等平台的嵌入式系统方案，提供从需求分析、硬件选型、PCB 设计、代码编写到最终交付的全流程服务，并将每个项目归档为可复用知识库。

# q
接单流程标准化的核心步骤和极限周期是什么？
# a
步骤：需求确认 → 原理图/PCB → 发厂制板 → 硬件采购 → 调试写代码 → 交付；极限周期可压缩至 4-5 天。

# q
36_信鸽计划编码方案中，颜色方案抗水印的关键原理是什么？
# a
浅灰色水印在 RGB 三通道数值接近，而有效色块在两通道高亮、一通道为 0（双原色方案：红/绿/蓝/黄/品红/青），通过三通道分离和通道量化可有效抑制水印干扰，提升识别鲁棒性。

# q
在 53_常规经验总结中，multipathd 无法启动的常见原因是什么？
# a
`multipathd` 必须存在配置文件 `/etc/multipath.conf` 才能启动；否则 `systemctl status` 会显示 `ConditionPathExists=/etc/multipath.conf was not met` 错误，需先创建配置文件。

# q
全局知识积累的核心模式有哪些？
# a
1. “做单即归档”：每完成一个外包项目，将需求、方案、代码、踩坑记录完整归档形成可检索知识库；2. “复刻即学习”：通过复刻开源硬件项目（如小智、ESP32 Spark）快速掌握新技术栈；3. “技能复用最大化”：同一套嵌入式技能同时服务于主业（Ceph 存储）、副业（外包接单）和内容创作（小红书）。
