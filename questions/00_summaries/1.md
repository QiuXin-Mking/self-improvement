# q
如何发现 iSCSI 目标并一次性连接所有发现的节点？
# a
使用 `iscsiadm -m discovery -t st -p <目标IP> -l` 命令，其中 `-t st` 表示 sendtargets 发现类型，`-l` 表示发现后自动登录。或者先执行发现 `iscsiadm -m discovery -t sendtargets -p 172.22.251.60:3260`，再使用 `iscsiadm -m node -L all` 一次性连接指定 Target 的所有路径。

# q
如何删除特定的 iSCSI 节点记录？
# a
使用 `iscsiadm -m node -o delete -T <target_iqn> -p <目标IP:端口>` 可以永久删除某条 target 信息。例如：
```bash
iscsiadm -m node -o delete -T iqn.2010-05.com.macrosan.target:fw1:a6da04d30b3b43ada0a97a7998988ae5:7894 -p 172.22.251.60
```
如果只指定 `-p` 或只指定 `-T`，则会删除匹配的所有节点记录。

# q
iSCSI 客户端断开连接有哪些方式？
# a
- 退出单个 Target 登录：`iscsiadm -m node -T <iqn> -u` 或 `iscsiadm -m node -T <iqn> -p <IP> -u`
- 退出所有登录：`iscsiadm -m node -U all`
- 删除节点记录（永久断开）：`iscsiadm -m node -o delete`（可按 Target 或全部删除）
- 退出全部登录并清除记录并非同一操作，`-u`/`-U all` 仅登出，`-o delete` 会移除发现信息。

# q
配置 iSCSI 启动器需要修改哪些关键文件并重启服务？
# a
1. 编辑 Initiator 名称文件 `/etc/iscsi/initiatorname.iscsi`
2. 配置 iSCSI 守护进程参数 `/etc/iscsi/iscsid.conf`
3. 重启服务：`service iscsi restart`（或 `systemctl restart iscsid`）
4. 查看当前名称：`cat /etc/iscsi/initiatorname.iscsi`
