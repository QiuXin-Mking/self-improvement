# q
什么是巨型帧（Jumbo Frame）？
# a
巨型帧是指数据链路层能传输的最大帧长超过标准以太网1500字节的帧，典型值为9000字节。该概念由Alteon Networks在1998年提出，虽未成为IEEE 802.3标准，但大多数设备厂商已支持。

# q
启用巨型帧的主要好处是什么？
# a
主要好处是提高网络传输效率，尤其在高速网络中。效率定义为实际payload传输时间与发送整个数据包（含头部）时间的比值，更大的帧意味着头部开销占比更低，从而提高有效吞吐量。

# q
标准以太网帧的MTU是多少？巨型帧通常设为多大？
# a
标准以太网MTU为1500字节。巨型帧没有严格的单一标准，常见设置为9000字节，但也有设备支持其他范围的值，实际可用大小取决于厂商实现。

# q
如何从网络效率角度理解巨型帧提升性能的原理？
# a
效率约为payload时间/总传输时间。巨型帧通过增大每帧携带的数据量，减小帧头、帧间隙等固定开销的比例，使更多时间用于传输有效数据，类似无线网络中通过帧聚合（如802.11n）提高效率的思路。
