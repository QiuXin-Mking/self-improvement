# q
如何创建文件的软链接和硬链接？
# a
- 硬链接：`ln source target`（source 只能是文件）
- 软链接：`ln -s source target`（source 可以是文件或目录）

# q
软链接和硬链接在文件系统中的存储原理有何区别？
# a
硬链接是文件的别名，与源文件共享同一个 inode 和数据块，除文件名外属性完全相同。软链接是一个独立的文件，拥有自己的 inode 和数据块，内容是指向目标文件路径的字符串。

# q
软链接相比硬链接有哪些独有的能力？
# a
1. 可以链接目录。
2. 可以链接不存在的文件。
3. 可以跨文件系统。
（这三项硬链接均无法实现）

# q
删除源文件后，软链接和硬链接分别会受到什么影响？
# a
硬链接仍能正常读写文件内容，因为数据块未被删除，硬链接计数仅从原有值减一。软链接会变成“断链”，指向无效路径，无法再访问文件。
