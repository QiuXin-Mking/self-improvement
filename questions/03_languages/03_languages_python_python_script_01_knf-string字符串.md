# q
Python f-string 字符串格式化的核心原理是什么，为什么它不需要 % 运算符？
# a
f-string（格式化字符串字面量）通过在字符串前加 `f` 或 `F` 前缀，允许直接在 `{}` 中嵌入变量或表达式。Python 在运行时会将其中的 `{variable}` 替换为变量值，并可附带格式说明（如 `:03` 补零）。因为变量替换已内建在字符串解析过程中，所以完全不需要再使用传统的 `%` 运算符进行填充。

# q
在 f-string 中使用 `%` 运算符会导致什么问题？
# a
f-string 本身已经完成了变量插入，如果继续使用 `% (var1, var2)` 进行格式化，Python 会尝试将 f-string 的格式化结果作为左侧字符串，再用 `%` 进行第二次格式化，这会导致类型错误或格式化字符串与提供的元组不匹配，从而引发 `TypeError` 或输出非预期的内容。

# q
如何在 f-string 中实现类似 `%03d` 的数字补零效果？
# a
在 f-string 的 `{}` 内，使用冒号 `:` 后接格式说明：`{变量名:03}` 表示将整数变量格式化为至少 3 位，不足前面补零。例如 `f"{mat_cnt:03}"` 对应 `%03d` 的效果。
