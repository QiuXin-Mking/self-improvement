# q
进程是什么？如何控制进程？
# a
进程（process）是正在被执行的程序，每个进程有唯一的进程ID，可通过`getpid()`获得。进程控制使用`fork`创建子进程、`exec`执行新程序、`waitpid`等待子进程结束。

# q
线程与进程的关系及特点是什么？
# a
一个进程可以包含一个或多个线程，它们共享地址空间、文件描述符和进程相关属性，但每个线程拥有独立的栈。线程ID仅在本进程内有效，跨进程无意义，错误处理通常使用`errno`。

# q
什么是信号（signal）？如何发送？
# a
信号是通知进程发生某种情况的技术，可通过键盘操作（如Delete、Ctrl+C）或`kill`函数发送。只有root超级用户才能向其他进程发送信号。

# q
什么是系统调用？
# a
系统调用是程序向内核请求服务的入口函数，例如进程控制相关的`fork`、`exec`、`waitpid`等。
