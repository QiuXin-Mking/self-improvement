# q
结构体和指针为什么不能直接使用 `==` 比较？
# a
因为它们属于不同类型，编译器无法生成合法的比较指令。如 `map_info_t` 是结构体类型，`void*` 是指针类型，直接比较会引发 `invalid operands to binary ==` 错误。

# q
如何理解原子操作 `atomic_dec_and_test()` 的返回值？
# a
`atomic_dec_and_test()` 会先将原子变量减 1，然后判断新值是否为 0。如果结果为 0 则返回真（非零），否则返回假（0）。示例中 `assert_true( atomic_dec_and_test(&msg->ref) == 1 )` 验证先将引用计数从 2 减到 1，再判断返回值应为 0（假），即断言相等。

# q
结构体中柔性数组成员 `segs[]` 的本质是什么？使用它时需要注意什么？
# a
`segs[]` 是 GNU C 的柔性数组成员（空长度数组），不占用结构体本身的空间，本质上相当于一个**指针**，指向紧随该结构体之后的内存区域。实际使用时需要通过额外的内存分配来容纳变长数据，并通过 `->segs` 访问，但不能直接对 `segs` 进行 `sizeof` 操作或赋值。
