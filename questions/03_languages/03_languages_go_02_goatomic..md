# q
`atomic.Uint32` 是什么，在 Go 中如何使用？
# a
`atomic.Uint32` 是 Go 语言 `sync/atomic` 包提供的原子操作类型，用于对 `uint32` 类型的变量进行无锁的线程安全操作。它通过底层的原子指令保证操作的原子性和内存可见性，避免使用互斥锁。常用方法包括 `Store`（写入值）、`Load`（读取值）、`Swap`（交换值并返回旧值）等。

# q
`atomic.Uint32` 的 `Swap` 操作的核心原理是什么？
# a
`Swap` 方法原子地将新值设置到变量中，并返回设置之前的旧值。整个过程在一个原子操作中完成，不会被其他 goroutine 中断，因此在并发环境下能安全地更新共享变量并获取其先前的状态，示例：
```go
previousValue := myUint32.Swap(100)
```

# q
原子操作（atomic）在并发编程中的主要作用是什么？
# a
原子操作用于在多 goroutine 并发访问共享变量时，避免数据竞争（race condition），保证变量读写、交换等操作的原子性和内存可见性，不需要显式的互斥锁，从而简化代码并提升性能。它们尤其适合简单的状态标志、计数器等场景。
