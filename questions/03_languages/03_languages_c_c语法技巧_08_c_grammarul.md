# q
C语言中数字后缀`UL`表示什么？
# a
`U`表示无符号（unsigned），`L`表示长整数（long）。`UL`组合表示`unsigned long`类型。后缀大小写均可，可以单独使用（如`100U`）或组合使用（如`100UL`）。示例：`#define BITMAP_4k_UNIT_EXTENT_CNT (4*1024*8UL)`。

# q
C语言中浮点数后缀`F`的作用是什么？为什么`F`不能与`U`组合？
# a
`F`表示`float`（单精度浮点数），用于显式指明数据类型。不加后缀时，浮点常量默认类型为`double`。`F`不能与`U`组合，因为浮点数没有`unsigned`的概念。

# q
在C语言中，为什么需要使用数字后缀来指明数据类型？试举一个重载函数调用中的例子说明。
# a
后缀用于显式指定常量的类型，避免类型转换歧义。尤其在函数重载时，若有两个重载版本`void fun(int);`和`void fun(float);`，调用`fun(0.5)`时，`0.5`默认是`double`，而`double`向`int`和`float`转换的优先级相同，会产生歧义。改用`fun(0.5F)`则明确调用`float`版本，消除歧义。
