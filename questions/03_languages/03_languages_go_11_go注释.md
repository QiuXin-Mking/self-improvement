# q
Go 代码中常见的特殊注释标记有哪些？它们各自表示什么含义？
# a
常见的特殊注释标记包括：
- `FIXME:` 表示代码需要修复或改进，如添加错误处理等
- `BUG:` 表示已知的代码缺陷，例如在特定输入下返回错误结果
- `NOTE:` 用于强调重要的说明信息，如算法复杂度等

这些标记帮助开发者在代码中快速定位待处理的问题或关键注意事项。

# q
`FIXME` 注释与 `BUG` 注释在使用场景上有什么主要区别？
# a
`FIXME` 通常用于标记需要修复、改进或尚未完成的功能代码（如缺少错误处理），强调“需要做某事”；`BUG` 则用于标记已发现的、会导致错误行为的代码缺陷（如函数在负输入时返回错误结果），强调“已知存在的问题”。两者都是提醒开发者关注的技术债务，但侧重点略有不同。
