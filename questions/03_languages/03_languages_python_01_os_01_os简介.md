# q
os模块如何实现跨平台兼容？
# a
os模块根据当前操作系统自动导入底层模块（posix或nt），并提供统一的接口。所有跨平台代码应使用os模块，而不是直接导入posix或nt，同时路径操作（如分割、拼接）应使用os.path子模块。

# q
os.name是什么？它有哪些可能的值？
# a
os.name是一个字符串常量，用于标识当前操作系统类型。在类UNIX系统上值为`'posix'`，在Windows上为`'nt'`，可用于编写平台相关的代码分支。

# q
os模块中与路径分隔和行分隔相关的常量有哪些？分别代表什么？
# a
- `os.sep`：路径分隔符，`'/'`或`'\\'`
- `os.altsep`：替代路径分隔符，`None`或`'/'`
- `os.pathsep`：环境变量`$PATH`中的分量分隔符，如`':'`或`';'`
- `os.linesep`：文本文件行结束符，`'\n'`、`'\r'`或`'\r\n'`
- `os.extsep`：扩展名分隔符，始终为`'.'`

# q
posix模块和nt模块的区别是什么？
# a
两个都是Python内建模块，用于提供与操作系统的底层接口：
- posix模块：针对类UNIX系统（Linux、macOS等），封装POSIX系统调用
- nt模块：针对Windows系统，封装Windows API

os模块会自动选择其中之一，开发者通常不应直接导入这两个模块。
