# q
shebang是什么，它的作用是什么？
# a
shebang是脚本第一行以 `#!` 开头的代码，其作用是指定由哪个解释器来执行该脚本。例如，当系统中同时安装多个Python版本或多个Shell时，通过shebang可以明确指定解释器，避免因版本不兼容导致的问题。

# q
`#!/usr/bin/python3` 和 `#!/usr/bin/env python3` 两种写法的区别是什么？
# a
- `#!/usr/bin/python3` 是硬编码的绝对路径，直接指定解释器的固定位置，若解释器不在该路径则脚本无法执行。
- `#!/usr/bin/env python3` 从 `PATH` 环境变量中查找 `python3` 解释器的位置，路径没有被写死，因此更灵活通用，推荐使用。

# q
Python脚本中的 `# -*- coding: utf-8 -*-` 声明的作用是什么？它在什么时候起作用？
# a
该声明用于指定源文件的编码格式（如UTF-8）。它在Python解释器将源码文件的二进制数据解码为Python代码时（即读取二进制文件后的第2步）起作用，告诉解释器用指定的编码（如UTF-8）进行解码，避免乱码或解码错误。

# q
Python解释器是如何确定源文件的编码的？
# a
解释器先读取源文件的前两行，用默认编码方式解码，然后使用正则表达式 `coding[:=]\s*([-\w.]+)` 进行匹配，提取匹配到的第一个分组作为编码，再用该编码对整个源码文件的二进制内容进行解码，得到可执行的Python语句。
