# q
在C语言宏定义中使用 `do { ... } while(0)` 包裹语句块的主要作用是什么？
# a
为了让多条语句的宏可以安全地在任何上下文中使用，避免出现语法错误。直接使用 `{ ... }` 包裹会在宏调用后面跟分号时产生多余的语句分隔，而 `{ ... };` 的形式在某些控制流（如 `if-else`）中会引发意外的语法错误。`do { ... } while(0)` 接受一个分号结尾，且只执行一次，是包裹多条语句的惯用法。

# q
宏 `#define rb_set_red(r) do { (r)->rb_parent_color &= ~1; } while (0)` 中 `(r)` 的括号有什么作用？
# a
保护宏参数 `r` 的运算优先级。如果宏传入的实参是一个表达式而非简单变量，没有括号可能会导致运算符优先级错误。例如，如果 `r` 是 `*ptr`，不加括号的 `*ptr->rb_parent_color` 会被解析为 `*(ptr->rb_parent_color)`，导致错误。加上 `(r)` 后可确保先求值再取成员。

# q
为什么 `do { ... } while(0)` 中的 `while(0)` 能让循环体只执行一次？
# a
因为 `while(0)` 的条件为假，循环在执行完 `do` 块后立即退出，所以整个语句块只会被执行一次。这保证了宏的语义与普通语句一致，同时提供了一个完整的语法单元可以安全地以分号结束。
