# q
什么是线程局部存储（TLS），它解决什么问题？
# a
线程局部存储（TLS/TSD）是一种让每个线程维护各自独立数据副本的机制，数据在同一线程内跨函数调用可访问，但其他线程不可见/不可修改。它用于解决全局变量或静态变量在多线程环境下需要加锁而带来的性能问题，让“只能由一个线程访问的数据”无需同步即可安全使用。

# q
`pthread_once` 函数的作用和典型使用场景是什么？
# a
`pthread_once` 保证在多线程环境中，无论被调用多少次，由 `init` 指定的初始化函数只会执行一次。典型场景是线程安全的单例初始化或只需执行一次的全局资源初始化，`once_control` 必须是初始化为 `PTHREAD_ONCE_INIT` 的静态变量。

# q
`pthread_key_create` 函数中的 `destructor` 参数有何作用？调用时机是什么？
# a
`destructor` 是自定义析构函数，当线程终止时，若与该键关联的值不为 `NULL`，系统会自动调用该函数释放线程持有的资源。一个线程有多个 TSD 变量时，各 destructor 的调用顺序不确定，因此设计应相互独立。

# q
线程局部存储的典型实现包含哪些核心数据结构？
# a
典型实现包含：
1. 一个全局（进程级）数组 `pthread_keys`，其索引即 `pthread_key_t` 值，每个元素记录该键是否在用以及关联的 destructor 函数。
2. 每个线程拥有一个指针数组，存放各自通过 `pthread_setspecific` 存储的数据块指针，与全局数组一一对应。

# q
如何减少对有限 `pthread_key_t` 数量的消耗？
# a
可以将多个线程局部变量封装到一个数据结构中，然后只使用一个 `pthread_key_t` 关联该封装对象。系统限制可通过 `PTHREAD_KEY_MAX` 或 `sysconf(_SC_THREAD_KEYS_MAX)` 查询，Linux 默认支持 1024 个键。
