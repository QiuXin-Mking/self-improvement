# q
网络字节序（大端字节序）的存储规则是什么？
# a
网络字节序采用大端字节序，最高有效字节（MSB）存储在最低的内存地址处，最低有效字节（LSB）存储在最高的内存地址处。与之相对，某些体系结构（如x86）通常采用小端字节序（LSB在低地址）。

# q
`htons` 函数的核心作用是什么？
# a
`htons` 代表“host to network short”，用于将16位无符号短整型数从主机字节序转换为网络字节序。在网络编程中，主要用于设置套接字地址结构（如端口号），确保数据在不同主机间传输时字节顺序一致。

# q
手动将16位无符号短整型转换为网络字节序的核心原理是什么？
# a
通过位操作交换高字节和低字节，示例函数实现如下：
```c
unsigned short manual_htons(unsigned short host_value) {
    return ((host_value & 0xFF00) >> 8) | ((host_value & 0x00FF) << 8);
}
```
原理是：用掩码分别保留高8位和低8位，再通过移位交换位置后合并。

# q
为什么在实际网络编程中推荐使用系统提供的 `htons` 而不是手动计算字节序？
# a
系统提供的 `htons` 是平台无关的，能自动根据当前主机的字节序决定是否需要交换字节；手动计算假设主机为小端，在已是大端的主机上会错误交换，导致可移植性问题。
