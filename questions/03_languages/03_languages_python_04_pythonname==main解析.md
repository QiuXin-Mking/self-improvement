# q
Python 模块的内置属性 __name__ 的值是如何确定的？
# a
`__name__` 是每个模块都有的内置属性。当模块被直接运行时，`__name__` 的值为 `"__main__"`；当模块被 `import` 导入时，`__name__` 的值为模块的文件名（不含 `.py`）。

# q
`if __name__ == "__main__":` 的作用是什么？
# a
用于判断当前模块是否被直接运行。只有在直接运行模块时，条件才为真，从而执行其下方的代码块。如果模块是被其他脚本 `import` 的，则该代码块不会执行。

# q
为什么通过 `if __name__ == "__main__":` 可以将模块用作脚本和库而互不影响？
# a
直接运行时 `__name__` 为 `"__main__"`，预定的执行逻辑会运行；而 `import` 时 `__name__` 变为模块文件名，条件为假，执行逻辑被跳过。这样既能将模块作为独立脚本运行，又能安全地导入复用，避免测试代码或副作用在导入时意外执行。
