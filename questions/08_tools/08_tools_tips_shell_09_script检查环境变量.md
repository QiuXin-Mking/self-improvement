# q
如何判断一个环境变量是否为空？
# a
使用 `[ "${变量名}" != "" ]` 进行判断，例如：
```sh
if [ "${http_proxy}" != "" ];then
    echo "http_proxy=${http_proxy}"
    exit -1
fi
```

# q
在检查环境变量的脚本中，`exit -1` 表示什么？
# a
`exit -1` 表示以非零退出码终止脚本，通常代表异常退出。父进程可通过 `$?` 捕获到该值来判断脚本执行失败。

# q
示例脚本的主要作用是什么？一般用在什么场景？
# a
检测 `http_proxy` 和 `https_proxy` 环境变量是否已被设置，若已设置则输出变量内容并以错误码退出。通常用于避免代理环境对某些网络操作（如下载、安装）产生干扰，在脚本开始前强制要求清除代理。
