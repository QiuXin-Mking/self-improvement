# q
在 getopt 短选项中，冒号数量如何表示选项是否必须带参数？
# a
- `a:` 表示选项 `-a` 后面必须跟一个参数值。
- `b::` 表示选项 `-b` 的参数可选；如果有参数，必须紧跟在选项后（例如 `-bvalue`），不能加空格。
- `c` 表示选项 `-c` 后面不能有参数。

# q
getopt 如何定义长选项？`-o` 和 `-l` 分别有什么作用？
# a
使用 `-o` 指定短选项，格式与短选项规则一致；使用 `-l` 指定长选项，多个长选项用逗号分隔，如 `--along:,,blong::,,clong`。命令结构如下：
```bash
getopt -o a:b::c -l along:,blong::,clong -- "$@"
```
`--` 用于分隔选项与参数，避免 getopt 解析错误。

# q
解析 getopt 输出时，为什么需要使用 `eval set -- "${ARGS}"`？
# a
`getopt` 输出的结果中的参数会保留原始参数中的空格，若不使用 `eval` 重新设置位置参数，带空格的值（如 `2020-01-08 15:00:00`）会被 shell 按空格拆分成多个参数。`eval set -- "${ARGS}"` 可正确合并带空格的参数，使其作为一个整体赋值给 `$1`, `$2` 等位置变量。

# q
getopt 输出的双横线 `--` 有什么意义？
# a
`--` 是 getopt 输出中的分隔符：左侧是正确识别和规范化后的选项及其参数，右侧是未被识别为选项的额外参数（无关参数）。在脚本中通常通过循环检测到 `--` 时执行 `shift` 后 `break`，以结束选项解析。
