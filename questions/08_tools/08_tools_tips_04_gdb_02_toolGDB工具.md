# q
如何编译一个可调试的C程序，并以简洁模式启动GDB？
# a
使用 `gcc -g hello.c -o hello` 编译生成带调试信息的可执行文件，然后执行 `gdb -q hello` 启动GDB，`-q` 参数可屏蔽版本等无关信息。

# q
在GDB中如何查看指定行附近的源代码？
# a
使用命令 `l <行号>` 或 `list <行号>`，会输出该行前后共十行代码（前5行后5行，或前10行后10行）。

# q
在GDB中如何运行程序到断点并查看函数调用栈？
# a
使用命令 `r` （run）运行程序直到断点处停止；使用命令 `bt` 或 `backtrace` 可以显示当前的函数调用回溯。

# q
如何打印某个内存地址对应的结构体内容，并让输出更易读？
# a
```c
(gdb) p *(结构体类型 *)0x内存地址
(gdb) set print pretty on
```
先用指针类型转换打印结构体所有成员的值，再通过 `set print pretty on` 开启美化输出，使结构体逐字段缩进显示。
