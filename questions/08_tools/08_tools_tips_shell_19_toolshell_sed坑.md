# q
在Shell脚本中进行算术运算时，为什么 `l1=$l1+1` 和 `l1=$l1 + 1` 都是错误的？正确写法是什么？
# a
Shell 中 `+` 会被当作字符串拼接，不会进行算术运算；必须使用 `expr` 命令，并且操作数和运算符之间必须加空格。正确写法：
```sh
l1=`expr $l1 + 1`
```
或者使用 `$((...))` 算术扩展（不在此文档范围，但也可行）。关键点是 `expr` 要求 `2 + 2` 的格式，而不是 `2+2`。

# q
在 Shell 中将命令输出赋值给变量时，常见的错误有哪些？正确的写法是什么？
# a
常见错误：
- 等号两边有空格（例如 `line1 = ...`）
- 忘记用反引号或 `$()` 包裹命令

正确写法：使用反引号或 `$()` 把命令包起来，且等号两边不能有空格。
```sh
line1=`cat ifcfg-${wk1} | wc -l`   # 反引号
line1=$(cat ifcfg-${wk1} | wc -l)   # $() 方式
```

# q
在 `sed` 命令中如何正确使用 Shell 变量？
# a
`sed` 使用单引号时变量不会展开，需要把变量放在单引号外面进行拼接。典型的正确写法：
```sh
sed -i ''${line1}'aTYPE=Ethernet' ifcfg-${wk1}
```
这里通过 `''` 和 `'` 将变量值 `${line1}` 暴露在单引号字符串之外，确保变量被展开后再拼接成完整的 `sed` 命令。
