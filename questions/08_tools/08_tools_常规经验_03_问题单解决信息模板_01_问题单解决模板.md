# q
OSD快照删除导致内存泄漏的典型根因是什么？
# a
在 segment 删除时未直接释放 block group 内存，且当 block_num 为 0 时未释放 block_mapping 内存，导致持续打流量及快照策略下内存异常增大。

# q
如何从日志或监控中定位 OSD 快照删除内存泄漏问题？
# a
通过命令 `cat /engine-fs/osd/*/osd_info | grep mem` 持续观测 OSD 内存使用；若在起快照策略并持续打流量的场景下，内存使用持续异常增大而非正常波动，则可判定为内存泄漏。旧 lib 版本特性为内存异常增大，新版本应呈现上下波动。

# q
解决 OSD 快照删除内存泄漏的标准流程或代码修改涉及哪些核心动作？
# a
1. 在 segment 删除时直接释放 block group 内存。
2. 在 block_num 为 0 时释放 block_mapping 内存。
3. 修改文件：`src/osd/osd_mdlog.c`、`src/osd/osd.c`、`src/osd/osd.h`（对应 commit `4c673a570cac` 和 `372b8896dd`）。
