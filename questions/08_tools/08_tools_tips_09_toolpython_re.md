# q
Python re模块中，match和search函数有什么区别？
# a
- `match`：从字符串的**开始位置**匹配正则表达式模式，若开头不匹配则失败。
- `search`：**扫描整个字符串**，返回第一个成功匹配的位置。
- 示例：
```python
import re
re.match(r"c", "abcdef")   # None，因为开头不是'c'
re.search(r"c", "abcdef")  # 匹配到索引2的'c'
```

# q
Python正则中 `*?`、`+?`、`??` 的含义是什么？
# a
它们是非贪婪（non-greedy）版本的 `*`、`+`、`?`：
- `*?`：匹配前一个RE的0次或多次重复，但尽可能少地匹配。
- `+?`：匹配前一个RE的1次或多次重复，但尽可能少地匹配。
- `??`：匹配前一个RE的0次或1次重复，但尽可能少地匹配（即优先0次）。
- 对比：贪婪版本 `*`、`+`、`?` 会尽可能多地匹配。

# q
Python re模块中 `(?P<name>...)` 和 `(?P=name)` 的作用是什么？
# a
- `(?P<name>...)`：命名捕获组，匹配的字串可以通过组名 `name` 访问。
- `(?P=name)`：反向引用命名组，匹配之前由名为 `name` 的组匹配到的相同文本。
- 常用于检查重复内容，例如匹配重复单词。

# q
re模块的 DOTALL 和 MULTILINE 标志分别控制什么行为？
# a
- `DOTALL`（`re.S`）：使 `.` 匹配任意字符，包括换行符 `\n`（默认 `.` 不匹配换行符）。
- `MULTILINE`（`re.M`）：改变 `^` 和 `$` 的行为：
  - `^` 不仅匹配字符串开头，还匹配每行的开头（换行符之后）。
  - `$` 不仅匹配字符串末尾，还匹配每行的末尾（换行符之前）。
- 示例：
```python
re.findall(r"^d", "abc\ndef", re.M)  # 匹配第二行开头的'd'
```
