# q
在CentOS系统中，如何为同一物理网卡（如enp175s0f3）配置两个不同的IPv4地址？
# a
通过创建网卡别名配置文件实现。示例：
- 主接口文件 `ifcfg-enp175s0f3` 配置主IP `172.250.101.189/16`
- 创建别名文件 `ifcfg-enp175s0f3:0`，设置 `DEVICE=enp175s0f3:0`、`IPADDR=192.168.77.109`、`PREFIX=16`、`ONBOOT=yes`
注意两个IP应规划在不同子网，避免路由冲突。

# q
网卡别名配置文件 `ifcfg-enp175s0f3:0` 中，为什么没有配置GATEWAY？
# a
为避免与主接口的默认网关冲突。系统中通常只保留一个默认网关，别名接口多用于提供额外服务IP，无需指定网关。示例中主接口设有 `GATEWAY=172.250.0.1`，别名省略该参数。

# q
对比示例中的主接口和别名接口，它们在IPv6配置上有何不同？
# a
主接口 `ifcfg-enp175s0f3` 明确指定了静态IPv6地址 `IPV6ADDR=2001:db8:3902:c2::e109/64`；别名接口仅启用了IPv6 (`IPV6INIT=yes`)，但没有设置 `IPV6ADDR`，因此只依赖SLAAC自动配置，无静态IPv6地址。

# q
如何让单网卡双IP配置在系统重启后依然生效？
# a
在对应的主接口和别名配置文件中均设置 `ONBOOT=yes`，如示例所示，确保启动时自动激活所有IP。
