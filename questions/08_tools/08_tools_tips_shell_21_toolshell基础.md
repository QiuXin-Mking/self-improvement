# q
在Linux中，`sh` 和 `bash` 通常有什么关系？
# a
在许多Linux系统中，`sh` 已经是一个指向 `bash` 的链接，`/bin/sh` 实际指向 `/bin/bash`。

# q
在bash中定义变量时，等号两边可以有空格吗？变量名命名有哪些规则？
# a
等号两边不能有空格。变量名只能使用英文字母、数字和下划线，首个字符不能为数字，中间不能有空格，不能使用标点符号和bash关键字。

# q
在bash中，单引号和双引号对字符串中的变量解析有什么不同？
# a
单引号内的变量不会被解析，原样输出；双引号内的变量会被解析替换成其值。

# q
如何使用echo在终端输出红色前景文字和红色背景文字？
# a
```bash
echo -e "\033[31m 红色前景 \033[0m 缺省颜色" 
echo -e "\033[41m 红色背景 \033[0m 缺省颜色"
```
其中 `\033[31m` 设置红色前景，`\033[41m` 设置红色背景，`\033[0m` 恢复默认颜色。

# q
如何将标准输出和标准错误都重定向到同一个文件，并且丢弃所有输出？
# a
- 合并输出到文件：`command > file 2>&1`
- 全部丢弃：`command > /dev/null 2>&1`
