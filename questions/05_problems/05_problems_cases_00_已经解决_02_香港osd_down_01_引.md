# q
香港OSD down问题的标准排查步骤是什么？
# a
根据问题单，初步排查方向为：
1. 检查端口连通性（可能涉及MON/OSD间通信）
2. 检查osd22的时延情况，其状态为down，需排查高时延根因

# q
如何定位osd22 down的原因？
# a
1. 确认集群版本：`ceph version 10.2.10 (7e238ff0756642117d59c8920fe4ae79cea3fbcc)`
2. 检查osd22所在节点（10.168.24.130 / 10.168.24.129 已授权）与MON节点（例如10.168.24.254）的端口连通性
3. 分析osd22的时延指标，判断是否存在磁盘性能瓶颈或网络问题
