# q
这个项目的核心目标是什么？
# a
开发一个端到端的工具系统：编码端将任意本地目录（含嵌套子目录和文件）转换为一张高鲁棒性的彩色图像；解码端通过手机拍摄该图像（允许存在浅灰色水印、轻微模糊、光照变化等），准确还原原始目录内容。

# q
编码图像使用哪六种颜色？为何避免使用灰色、白色和黑色？
# a
六种颜色为RGB两两原色组合的高饱和度颜色：
- 黄（255, 255, 0）、品红（255, 0, 255）、青（0, 255, 255）
- 红（255, 0, 0）、绿（0, 255, 0）、蓝（0, 0, 255）
避免使用灰色、白色和黑色，因为它们易与浅灰色水印混淆，不利于解码分离。

# q
图像的物理布局有哪些关键特征？
# a
- 色块按行优先顺序规则排列，形成对齐网格；
- 每个色块尺寸建议 ≥ 20×20 像素，确保手机拍摄后可分辨；
- 图像最外层添加 ≥5 像素宽的纯黑边框，用于自动检测感兴趣区域（ROI）；
- 整体图像比例适配常见屏幕显示与打印（如 4:3 或 1:1）。

# q
解码端如何通过多通道滤镜策略抑制浅灰色水印的影响？
# a
分别提取 R、G、B 三个通道，对每个通道进行阈值量化（例如：>200 → 100%，100–200 → 50%，<100 → 0%）；重建 RGB 值时，中性灰水印因三通道数值接近而被抑制，而有效色块在两通道高亮、一通道为0，从而被保留。

# q
项目采用何种打包与分发方式，使最终用户无需安装 Python 环境？
# a
使用 PyInstaller 或 Nuitka 将整个工具打包为单文件可执行程序（single-file executable）。生成三套独立二进制：dir2image-win.exe、dir2image-macos、dir2image-linux，均为静态链接，无需预装 Python 或手动安装依赖。命令行示例：
```bash
./dir2image --encode my_folder output.png
./dir2image --decode captured.jpg restored_folder/
```
并提供 build.sh / build.bat 脚本自动化完成虚拟环境创建、依赖安装、测试和打包。
