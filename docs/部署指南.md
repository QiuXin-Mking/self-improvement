# 服务器部署指南

## 方式一：一键部署（推荐）

使用项目自带的部署脚本，自动完成本地编译、打包上传、服务器编译部署：

```bash
./scripts/deploy_to_huoshan.sh
```

前置条件：
- 本地 Node.js 环境（前端编译）
- 服务器已配置 SSH 免密登录（`~/.ssh/config` 中 `huoshan` 主机）
- 服务器自动安装 Go 1.22（如需）

脚本流程：
1. 本地编译前端 (`npm run build`)
2. 打包源码 + 前端产物上传到服务器
3. 服务器端编译 Go 程序
4. 自动保留用户数据（`data/` 和 `.env`）
5. 启动服务（默认端口 4430）

## 方式二：手动部署

### 本地编译

```bash
# 编译前端
cd frontend && npm run build

# 编译 Go（本地）
./scripts/compile.sh

# 或交叉编译 Linux
BUILD_TARGET=linux ./scripts/compile.sh
```

### 上传到服务器

```bash
scp bin/web_server user@server:/opt/self-improvement/
scp -r frontend/dist user@server:/opt/self-improvement/
scp .env.example user@server:/opt/self-improvement/
```

### 服务器端配置

```bash
cd /opt/self-improvement
cp .env.example .env
# 编辑 .env 设置 JWT_SECRET 和 PORT
chmod +x bin/web_server
./bin/web_server
```

### 使用 systemd 保持后台运行

创建 `/etc/systemd/system/self-improvement.service`:

```ini
[Unit]
Description=Self-Improvement Learning System
After=network.target

[Service]
Type=simple
EnvironmentFile=/opt/self-improvement/.env
WorkingDirectory=/opt/self-improvement
ExecStart=/opt/self-improvement/bin/web_server
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable self-improvement
sudo systemctl start self-improvement
```

## Docker 部署

```bash
docker-compose up -d
```

Docker 配置（`docker-compose.yml`）：
- 应用端口：4430
- Nginx 反向代理：80/443
- 数据卷：`app_data`（持久化数据库）
- 问题目录挂载：`./questions:/app/questions`

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `4430` | 服务端口 |
| `DATABASE_PATH` | `data/app.db` | SQLite 数据库路径 |
| `JWT_SECRET` | 无 | JWT 签名密钥（生产环境必须修改） |
| `STATIC_DIR` | `dist` | 前端静态文件目录 |

## 常见问题

### 端口被占用

修改 `.env` 中的 `PORT` 即可。

### 数据备份

```bash
# 备份数据库
cp data/app.db data/app.db.bak.$(date +%Y%m%d)

# 定时备份（crontab）
0 2 * * * cp /opt/self-improvement/data/app.db /opt/self-improvement/data/app.db.$(date +\%Y\%m\%d)
```

### 重装后恢复

部署脚本会自动保留 `data/` 和 `.env`，重装后用户数据和配置不丢失。
